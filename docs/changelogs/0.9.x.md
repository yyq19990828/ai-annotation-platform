## [0.9.4 phase 3] - 2026-05-08 · mask→polygon 共享化 + simplify tolerance 注入 + SAM E2E（Polished Contour）

主项详情见 [`CHANGELOG.md` → 0.9.4 phase 3](../../CHANGELOG.md)。要点速览：

- **mask_utils 共享包接入**：删 `predictor.py:352-385` 内嵌 `_mask_to_polygon`；改 `from mask_utils import mask_to_polygon` + `normalize_to=(w,h)` 调用。`polygon.py` 补 try/except 拓扑兜底（self-intersecting contour 时降级原始 contour 而非空）；`normalize.py` `round(_, 6)` 6 位精度对齐协议。docker-compose context 升 `apps/`、Dockerfile `COPY _shared/mask_utils + pip install -e`，**`docker build apps/grounded-sam2-backend/` 命令失效**，改走 `docker compose --profile gpu build` 或 `docker build -f apps/grounded-sam2-backend/Dockerfile apps/`。
- **`Context.simplify_tolerance` 注入 + WARN**：`schemas.py` 加字段；`predictor.py` 顶层 `DEFAULT_SIMPLIFY_TOLERANCE = 1.0` + `VERTEX_COUNT_WARN_THRESHOLD = 200` 常量；`predict_*` + `_masks_to_results` 全部加 `simplify_tolerance` 参数转发；`main.py` 读 `ctx.simplify_tolerance` 校验 float ≥ 0 透传，非法值 422。顶点 > 200 记 `logger.warning` 运维信号。
- **`scripts/eval_simplify.py`**：mask png 目录 × 5 档 tolerance 算 IoU + 顶点数 → markdown 报告。`_render_markdown` "数据驱动结论" 三档判定：① ≥95% 完全达标推荐档；② 85%~95% 且 IoU mean ≥ 0.95 → 近达标 + 长尾分析；③ < 85% 且 IoU spread < 0.05 → 结构性根因；④ 其它 → 推荐最优档。
- **fixtures `real_sam_masks/`**：84 张真实 SAM mask（30 帧 cpc0-R 1920×1080 沙盘视频 × DINO+SAM text "car"/"person"/"building" 三 prompts，DINO scores 0.85-0.97）+ 6 张合成占位 = 90 张, 516KB commit 进库 + README 注释采集流程。**采集复盘**：首次用 "中心 60% bbox" prompt（IoU mean 0.54、IoU≥0.95 16.7%）是错的 — 1920×1080 中心 60% 是车+建筑+地面混合，SAM 拿到杂乱物；换 **DINO+SAM text 模式**后数据完全正常，与工作台 `S` 工具 text 模式真实使用对齐。
- **`docs/research/13-simplify-tolerance-eval.md`**：90 张样本评测。tolerance=1.0 IoU mean **0.98**、IoU≥0.95 占比 **92.2%**、顶点中位 **102** — 接近 95% 验收线但未达，但 IoU mean ≥ 0.95，**默认值合理保持不动**。跨 tolerance 趋势健康（tolerance↑ → 顶点数 273→27 大降, IoU mean 仅微降 0.032）。长尾 < 15% 样本 IoU 落 [0.5, 0.95)，根因 multi_polygon / 空洞结构问题 → ROADMAP P2 follow-up。
- **`_test_seed/seed_reset` 加 ml_backend 工厂**：fixture 项目造 `MLBackend(name="E2E SAM Mock", url="http://mock-sam.e2e:9999", state="connected", is_interactive=True)` 绑到项目；`project.ai_enabled=True` + `ml_backend_id=mock_backend.id`。`SeedReset` + 前端 `SeedData` 加 `ml_backend_id`。
- **`annotation.spec.ts` SAM 工具用例**：`page.route(/\/interactive-annotating/)` 拦截返回固定 polygon；验证 `tool-btn-sam` + `sam-subtoolbar` 可见、3 sub 按钮 aria-pressed 互斥、point 模式点击 stage 后 mock 至少命中 1 次。
- **测试**：mask_utils 9/9 + grounded-sam2-backend 23/23（4 新 simplify_tolerance case）+ api seed/ml_backend 16/16 全绿。
- **不在范围内**：① ProjectSettings UI 暴露 simplify_tolerance（YAGNI）；② query string 兼容（schema body 已能表达）；③ sidecar mock backend；④ 自动化 fixture 采集脚本。

- **下一步**：v0.9.x SAM 一期收口，进入 v0.9.5（M4 + M5 合并）`/ai-pre` 文本批量 UI + 运维收口。
