# v0.9.17 — 视频轨迹与关键帧编辑

## Summary

- 范围来自 `ROADMAP/2026-05-11-video-workbench.md` 的 M2 + M3：把 v0.9.16 的逐帧 `video_bbox` 工作台升级为 compact `video_track` 轨迹标注，并补上关键帧编辑、线性插值、目标消失段和基础质量检查。
- 从当前分支 `codex/v0.9.16-video-workbench` 新建开发分支：`codex/v0.9.17-video-track-keyframes`。
- 继续复用现有 `annotations.geometry` JSONB，不新增数据库表；新视频标注默认写 `annotation_type="video_track"`，旧 `video_bbox` 保持可读兼容。

## Key Changes

- 后端 schema：
  - 在 `apps/api/app/schemas/_jsonb_types.py` 新增 `video_track` geometry。
  - `video_track` 结构包含稳定 `track_id`、`keyframes[]`；每个关键帧包含 `frame_index`、`bbox`、`source`，并支持 `absent` / `occluded` 标记。
  - `class_name` 继续作为轨迹类别来源，不新增 `class_id` 字段或迁移。
  - 保留 `video_bbox` union 分支，兼容 v0.9.16 已保存的逐帧框。
  - 更新 OpenAPI snapshot 与前端类型生成。

- 前端工作台：
  - `VideoStage` 从仅显示当前帧 `video_bbox`，升级为显示当前帧上的轨迹结果：手工关键帧优先，其次显示相邻关键帧间的线性插值框。
  - 新建轨迹：未选中轨迹时画框，创建一个 `video_track` annotation，并生成 `track_id`。
  - 延续轨迹：选中已有轨迹后在其它帧画框或移动框，更新同一条 annotation 的 `keyframes[]`，不创建多条逐帧 annotation。
  - 轨迹列表展示 track、类别、显隐、锁定、重命名和当前/最近关键帧状态。
  - 支持在当前帧把选中轨迹标记为 `absent` 或 `occluded`；插值不得跨越 `absent` 段。
  - 旧 `video_bbox` 继续在对应帧展示，作为兼容路径；新 UI 默认不再创建 `video_bbox`。

- 插值与质量检查：
  - bbox 插值使用相邻关键帧之间的线性插值，按 `frame_index` 距离计算 `x/y/w/h`。
  - 只在同一轨迹相邻有效关键帧之间插值，不跨越 `absent`。
  - 编辑时 clamp bbox 到 `[0, 1]` 归一化范围。
  - 在前端提示基础质量问题：关键帧间隔过大、极小框、同帧同类高度重叠框、越界被修正。
  - 审核模式下可把问题定位到 `track_id + frame_index`，并写入退回原因文本。

- 文档与版本记录：
  - 更新 `ROADMAP/2026-05-11-video-workbench.md`，标记 M2 + M3 在 v0.9.17 落地。
  - 更新 `docs-site/dev/concepts/video-annotation-workbench.md`，说明 `video_track` schema、关键帧、插值和 absent/occluded 语义。
  - 更新 `docs-site/user-guide/for-annotators/video-track.md`，补充轨迹标注流程。
  - 更新 `CHANGELOG.md` 的 v0.9.17 条目。
  - 本期不涉及环境变量；无需更新 `.env.example` 或运行 `pnpm docs:gen-env-vars`。

## Test Plan

- 后端：
  - 新增 `video_track` schema 校验测试：合法轨迹、空 keyframes 拒绝、负数 `frame_index` 拒绝、旧 `video_bbox` 仍可解析。
  - 运行 `cd apps/api && uv run pytest apps/api/tests/test_video_workbench.py`。
  - 运行 `pnpm openapi:export` 后再运行 `pnpm openapi:check`。

- 前端：
  - 新增 `VideoStage` 单测：创建轨迹、给同一轨迹添加第二个关键帧、当前帧显示手工关键帧、关键帧之间显示插值框、`absent` 段不插值、锁定/隐藏轨迹生效、旧 `video_bbox` 仍按帧显示。
  - 新增 transform/type 测试覆盖 `video_track` 到当前帧 bbox 的转换。
  - 运行 `pnpm --filter @anno/web test -- VideoStage transforms`。
  - 运行 `pnpm typecheck`。

- 手工验收：
  - 打开一个 `video-track` 任务，在第 0 / 30 / 60 帧调整同一对象，确认轨迹列表只出现 1 条 track。
  - 删除或标记中间段 `absent` 后，确认插值结果重新计算且不会穿过消失段。
  - 在审核模式中确认能定位到某条轨迹的某一帧问题。
  - 回归打开图片工作台，确认 image bbox / polygon / SAM 路径不受影响。

## Assumptions

- v0.9.17 使用 compact JSON 保存轨迹，不逐帧展开写库。
- 插值结果仅用于前端显示和后续导出展开基础，本期不把 interpolated frame 写成独立 annotation。
- track 显隐和锁定是工作台 UI 状态，不持久化到后端。
- `class_name` 继续作为轨迹类别字段；项目类别当前没有稳定 `class_id`，本期不引入类别迁移。
