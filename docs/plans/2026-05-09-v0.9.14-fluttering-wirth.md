## Scope

### #1 — mask→polygon 多连通域 / 空洞支持（P2，主菜，~2 天）

**协议设计**：扩展 `apps/api/app/schemas/_jsonb_types.py:152-180` 的 Geometry discriminated union。

```python
class PolygonGeometry(BaseModel):
    type: Literal["polygon"] = "polygon"
    points: list[list[float]] = Field(min_length=3)
    holes: list[list[list[float]]] = Field(default_factory=list)  # 新增, 默认 [] 向后兼容

class MultiPolygonGeometry(BaseModel):
    type: Literal["multi_polygon"] = "multi_polygon"
    polygons: list[PolygonGeometry] = Field(min_length=1)

Geometry = Annotated[BboxGeometry | PolygonGeometry | MultiPolygonGeometry, Field(discriminator="type")]
```

向后兼容口子：① 老存量 `{type: "polygon", points: [...]}` 解析 holes 默认 `[]`；② `to_internal_shape()` (`apps/api/app/schemas/prediction.py`) 路径不动，新增 multi_polygon 分支，旧路径走原逻辑。

**算法升级**（`apps/_shared/mask_utils/src/mask_utils/polygon.py`）：

新增 `mask_to_multi_polygon(mask, tolerance, normalize_to)` 返回 `list[{exterior: list[[x,y]], holes: list[list[[x,y]]]}]`：
- L34 改 `cv2.findContours(binary, cv2.RETR_CCOMP, cv2.CHAIN_APPROX_NONE)` 抓内外环树；hierarchy 第 4 列 == -1 = 外环, 否则 = hole。
- 每个外环对应 1 个 polygon，挂上属于它的 holes（按 hierarchy parent 索引归属）。
- shapely simplify 同样跑（外环 + 每个 hole 各自 simplify，tolerance 一致）；保留 v0.9.4 拓扑兜底。
- **保留** `mask_to_polygon` 旧函数不动，predictor 视情景调用：单连通域时仍走 mask_to_polygon，多连通域走 mask_to_multi_polygon。
- 形态学闭运算（`cv2.morphologyEx(..., MORPH_CLOSE)`）做可选预处理，default off（避免吞掉小 hole）；ADR-0013 update 章节解释 trade-off。

**Predictor 改造**（`apps/grounded-sam2-backend/predictor.py:280` + `:325-332`）：

```python
multi = mask_to_multi_polygon(mask, tolerance=eff_tol, normalize_to=(w, h))
if len(multi) == 1 and not multi[0]["holes"]:
    # 单连通域无 hole, 输出旧 polygon shape (向后兼容)
    shape = {"type": "polygonlabels", "value": {"points": multi[0]["exterior"], "polygonlabels": [label]}, "score": score}
else:
    # 多连通域或带 hole, 升级 multi_polygon
    shape = {"type": "polygonlabels", "value": {"polygons": [{"points": p["exterior"], "holes": p["holes"]} for p in multi], "polygonlabels": [label]}, "score": score}
```

**前端渲染**（`apps/web/src/pages/Workbench/stage/CanvasDrawingLayer.tsx`）：

- Konva 用 `<Line>` + `closed={true}` + `fillRule="evenodd"`；hole 通过 sceneFunc 自定义路径绘制（外环 moveTo + lineTo, hole 反向 lineTo, fill evenodd 自动镂空）。
- discriminated union 渲染分支：旧 `polygon` 走原路径；新 `polygon`+holes 或 `multi_polygon` 走 sceneFunc 路径。
- `tools/PolygonTool.ts` 编辑工具暂不支持画 hole（v0.9.14 只读渲染兼容；用户手画 polygon 仍走单 ring 路径）。

## 风险与回滚

- **mask 协议 break**：PolygonGeometry 加 `holes: list = []` 字段是**纯加字段，向后兼容**（老数据反序列化默认 []，老前端忽略未知字段）。MultiPolygonGeometry 是新 discriminator 分支，不影响老路径。**回滚策略**：predictor.py 的「单连通域无 hole 输出旧 shape」逻辑保留，永远不输出新 shape 给老前端 — 即使前端没合也不破。
- **Konva fillRule="evenodd" 浏览器兼容**：所有目标浏览器（Chrome/Edge/Firefox/Safari 14+）均原生支持，无需 polyfill。
- **测试覆盖率超目标体量**：4 个文件全测时间可能超 0.5 天；如发现 BatchesSection (951 行) 测试体量过大，**降级策略** = 先合 GeneralSection + DatasetsSection + AuditPage 三个 section 测试（lines 增量约 +3.5pp 到 ~28.5%），BatchesSection 推到 v0.9.15 与 admin-locked UI 测试合并写。
- **v0.9.13 落地遗留 4 张空白态截图 fixture**：本版**不处理**，仍按 v0.9.13 落地经验，决策延到 v0.9.15 或更后（ROADMAP §A 现在可做段已记录）。
