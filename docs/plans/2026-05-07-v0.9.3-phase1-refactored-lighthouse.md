# v0.9.3 Phase 1 — Refactored Lighthouse

> 状态：Plan 阶段 · 2026-05-07
> 范围：ROADMAP §A/§B/§C 中可在 Mac 无 GPU 环境推进的前端杂项 4 项
> SAM 主线（M3 mask→polygon）不在本 phase；待回到 GPU 机器再启动

---

## Context

v0.9.x 主线（Grounded-SAM-2 接入）目前停留在 M2（v0.9.2 工作台 SAM `S` 工具 + 文本入口落地）。M3 起的 mask→polygon 化抽到 `apps/_shared/mask_utils/` 需要 GPU backend 联调，临时手头是 Mac 不便推进。

为不空转，本 phase 1 用 ROADMAP "现在可做 / 杂项 / 纯前端" 的 4 个 maintenance 项填充，每项体量适中，互相独立可并行 review，发版后 SAM 主线无缝接续 M3。

四项的内在共性是"补足残缺感 + 提升日常体验"，不动主架构：
- API 密钥按钮（disabled "规划中"）让管理页有破窗感 → 完整收口
- 登录页相比注册/忘记密码缺 CAPTCHA 防线 → progressive 模式补齐
- IoU 计算虽未到瓶颈但缺空间索引基建 → 为千框场景预热
- 浮层组件第 5/6 次散落自实现 → 通用组件扩展 custom content 槽，统一外观与键盘行为

class ApiKey(Base):
    __tablename__ = "api_keys"

### 2. 登录页 progressive CAPTCHA

**问题**：
v0.8.7 注册 / 忘记密码已用 Cloudflare Turnstile，登录页未加（每天高频，体验代价大）。生产字典攻击场景：同 IP 失败 ≥ 5 次后，下次登录强制弹 widget；正常用户零打扰。

**基础设施已就位**（无需新装）：
- 前端 `<Captcha>` 组件 `apps/web/src/components/Captcha.tsx`，site key 缺省时返 null（dev 零打扰）
- 后端 `verify_turnstile_token` `apps/api/app/services/captcha_service.py:23-62`
- slowapi `apps/api/app/core/ratelimit.py`（登录已 5/min IP 限流）
- Redis aioredis 工厂 `apps/api/app/core/token_blacklist.py:_get_redis()` 可复用

**后端改动**：

新增 `apps/api/app/services/login_failed_counter.py`：
- `incr(ip) -> int` ：`INCR login_failed:{ip}` + `EXPIRE 3600`
- `get(ip) -> int` ：`GET`
- `reset(ip)` ：`DEL`
- 阈值常量 `CAPTCHA_REQUIRED_THRESHOLD = 5`（写到 `app/config.py`，可 env 覆盖）

`apps/api/app/api/v1/auth.py:58-123` login 路由改造：
- 入口先 `count = await counter.get(client_ip)`；若 `count >= 5` 且 `data.captcha_token` 为空或 `verify_turnstile_token(token, ip)` 失败 → 400 `captcha_required`
- 凭据校验失败时 `count = await counter.incr(ip)`；401 response 加 header `X-Login-Failed-Count: {count}`
- 凭据校验成功时 `await counter.reset(ip)` + 200

`LoginRequest` schema 加可选 `captcha_token: str | None = None`。

**前端改动**：

`apps/web/src/api/client.ts`：拦截器在 ApiError 上保留 response headers，至少暴露 `X-Login-Failed-Count`（小写化键），让 LoginPage 能读。当前 `ApiError` 类型扩 `responseHeaders?: Record<string,string>`。

`apps/web/src/pages/Login/LoginPage.tsx`：
- 状态加 `failedCount`（默认 0）
- `useLogin().mutateAsync` onError 时若 status === 401，从 err.responseHeaders 读 count → setFailedCount
- 渲染：`{failedCount >= 5 && <Captcha onChange={setCaptchaToken} />}` 在密码框下
- 提交时 captchaToken 必填校验 + 透传到 `authApi.login`
- 成功登录后清零（mutation onSuccess）

**关键文件**：
- `apps/api/app/services/login_failed_counter.py` (新)
- `apps/api/app/api/v1/auth.py` (login 改造)
- `apps/api/app/schemas/user.py:LoginRequest` (加 captcha_token 字段)
- `apps/api/app/config.py` (加阈值常量)
- `apps/web/src/api/client.ts` (响应头透传)
- `apps/web/src/api/auth.ts` (login 透传 captcha_token)
- `apps/web/src/pages/Login/LoginPage.tsx` (条件渲染 Captcha)

**验证**：
- 后端：pytest 模拟 5 次失败 + 第 6 次无 token → 400；带合法 token → 401（仍是密码错），但不 captcha_required；成功后再试同 IP 计数应清零（fakeredis fixture 已存在，参考 `tests/services/test_token_blacklist.py`）
- 前端：MSW 桩响应头 `X-Login-Failed-Count: 5` → 断言 `<Captcha>` 渲染
- 手测：dev 模式 `TURNSTILE_ENABLED=False`，故意输错 5 次后第 6 次仍能登录（dev 短路）；CI / staging 开启时验证真实流程。

### 3. IoU rbush 空间索引

**问题**：
当前 `apps/web/src/pages/Workbench/shell/WorkbenchShell.tsx:305-313` 用 useMemo + `aiBoxes` × `userBoxes` 同类筛选 + some() 早退。已有 `iouShape`（`apps/web/src/pages/Workbench/stage/iou.ts:79-99`，polygon-clipping 精确）。

调用单点、当前不是瓶颈，但作为千框场景的基础能力先抽出。

**实现**：

依赖：`pnpm --filter web add rbush @types/rbush`。

新增 `apps/web/src/pages/Workbench/stage/iou-index.ts`：
```ts
// 同类分片 rbush：按 cls 字符串分桶，每桶一棵 RBush
// 入口：buildIndex(boxes: ShapeForIoU[] & {id, cls}): IoUIndex
// 查询：indexQuery(idx, query): candidates[]（仅 bbox 有交集的）
// 候选过 iouShape 精确判定
export class IoUIndex {
  private trees = new Map<string, RBush<...>>();
  insert(box) { ... }
  candidatesForBox(box: ShapeForIoU & {cls}): ShapeForIoU[] { ... }
}
export function buildIoUIndex(boxes): IoUIndex { ... }
```

WorkbenchShell.tsx:305-313 改造：
```ts
const userIndex = useMemo(() => buildIoUIndex(userBoxes), [userBoxes]);
const dimmedAiIds = useMemo(() => {
  const out = new Set<string>();
  if (userBoxes.length === 0 || aiBoxes.length === 0) return out;
  for (const a of aiBoxes) {
    const candidates = userIndex.candidatesForBox(a);
    if (candidates.some((u) => iouShape(u, a) > iouDedupThreshold)) out.add(a.id);
  }
  return out;
}, [aiBoxes, userIndex, iouDedupThreshold]);
```

不变：同类预筛 + some() 早退仍保留。增量收益在 cross-box 候选裁剪。

**关键文件**：
- `apps/web/package.json` (依赖)
- `apps/web/src/pages/Workbench/stage/iou-index.ts` (新)
- `apps/web/src/pages/Workbench/shell/WorkbenchShell.tsx:305-313` (替换)

**验证**：
- 单测 `iou-index.test.ts`：跨类不互相干扰；同类 query 命中包围盒重叠者；空索引 / 单元素边界
- 现有 e2e 标注流程跑通无回归（IoU 视觉去重输出与改造前一致）
- 手测：seed 一个含 100+ AI 框的项目，对比性能（chrome devtools profile 单帧 < 4ms）
