# v0.9.16 — Video Workbench MVP

## Summary

- Scope is M0 + M1 from `ROADMAP/2026-05-11-video-workbench.md`: video metadata/manifest plus a usable `VideoStage` for playback, frame stepping, and per-frame bbox annotation.
- Start from branch `codex/v0.9.16-video-workbench`.

## Key Changes

- Backend media foundation:
  - Add `ffmpeg/ffprobe` to the API/Celery image and implement a media worker path for video `DatasetItem`s.
  - Store video metadata in existing `dataset_items.metadata["video"]`: `duration_ms`, `fps`, `frame_count`, `width`, `height`, `codec`, `poster_frame_path`, and probe error fields.
  - Generate a poster thumbnail into the existing `thumbnail_path` flow so task lists can show video previews.
  - Add `GET /tasks/{id}/video/manifest` returning presigned video URL, poster URL, and normalized metadata.
  - Extend `TaskOut`/frontend `TaskResponse` with `video_metadata`.

- Annotation API/schema:
  - Add `video_bbox` geometry: `{ type: "video_bbox", frame_index, x, y, w, h }`.
  - Keep existing image `bbox`/`polygon` behavior unchanged.
  - Use `annotation_type="video_bbox"` for v0.9.16 video boxes; `video_track` remains out of scope for this release.

- Frontend workbench:
  - Add `VideoStage` alongside `ImageStage`; `WorkbenchShell` selects it when `task.file_type === "video"` or project type is `video-track`.
  - `VideoStage` supports play/pause, exact frame seek, previous/next frame, frame number/timecode display, annotated-frame markers, and bbox create/select/move/resize/delete on the paused frame.
  - Playback mode is read-only; editing is enabled only while paused.
  - Disable SAM/polygon/canvas AI tools for video tasks in v0.9.16; keep queue, topbar, submit/review, comments, locks, and offline queue paths intact.

- Docs and release notes:
  - Add `CHANGELOG.md` entry for `0.9.16`.
  - Add user docs for video annotation under `docs-site/user-guide/for-annotators/`.
  - Add developer docs for video metadata + `video_bbox` schema under `docs-site/dev/concepts/`.
  - Add an ADR only if implementation adds persistent schema beyond `metadata["video"]` and `video_bbox`.

## Test Plan

- Backend:
  - Unit-test video probe parsing with mocked `ffprobe`.
  - Unit-test poster generation failure records metadata error without creating broken tasks.
  - API-test `GET /tasks/{id}` exposes `video_metadata`.
  - API-test `GET /tasks/{id}/video/manifest` returns metadata and signed URLs for video, rejects non-video tasks.
  - Schema-test `video_bbox` validation and legacy `bbox` compatibility.

- Frontend:
  - Unit-test geometry transforms for `video_bbox`.
  - Unit-test `VideoStage` frame stepping, paused editing gate, and current-frame annotation filtering.
  - Unit-test `WorkbenchShell` stage selection for image vs video tasks.
  - Run `pnpm --filter @anno/web typecheck` and targeted Vitest suites.

- Integration/manual:
  - Import at least one small `.mp4`, confirm Celery generates metadata and poster.
  - Link dataset to a `video-track` project, open workbench, seek to frame 0/middle/last, create bbox, save, reload, and verify boxes only appear on their saved frames.
  - Confirm existing image workbench still opens and edits normally.
  - Run docs build after docs changes.

## Assumptions

- v0.9.16 intentionally does not implement track IDs, keyframes, interpolation, video AI tracker, SAM video predictor, or expanded export.
- Direct `/files/upload-init` video tasks can fail gracefully if no video metadata exists; production video import path for this release is dataset import/link.
- Adding `ffmpeg` means Docker image rebuild is required for Celery/API containers.
