# v0.8.8 — Sparkling Tome（0.9.x SAM 接入前的清债收尾）

> **代号**: sparkling-tome
> **目标**: 把 ROADMAP 中所有 P3 杂项 + 几个 P2 小动作清干净，0.9.x 才能专注做 Grounded-SAM-2 基座。
> **预估工作量**: 一周量级（17 项，多数小颗粒度）。
> **不做的事**: 大特性（OAuth2 / 邮箱验证 / 系统设置 admin UI / Bug 反馈 LLM 聚类 / 非 image-det 工作台 / OpenSeadragon / 批次状态机 admin-locked 实施 / 截图 14 张回填）—— 它们各自值得独立版本。

---

## Context

ROADMAP 规划 v0.9.x 接 Grounded-SAM-2 + v0.10.x 接 SAM 3，作为首版 AI 基座 epic。**进入 SAM 大改前**，需要先把现存的小颗粒度技术债清空，避免 v0.9.x 一边接 SAM 一边修小问题。v0.8.8 即这个收尾窗口：把所有 P3 + 几个体量适中的 P2 一次性收掉，让 v0.9.x 干净落地 AI epic。

用户决策：
- 范围 = 稳健清债（P3 全打 + 选定 P2 小动作）
- 单测目标 = 推回 25%（v0.8.7 临时降到 22%）
- 截图自动化 14 张 = 推迟到 0.9.x（与 SAM UI 一起做）
- WS reauth 做完整方案（后端 `/auth/refresh` + 前端重连）
- Grafana = 顺手补 docker-compose prom + grafana service
- version 漂移 = 一并修为 0.8.8

## 风险与权衡

- **CSP 太严会破坏前端 dev / docs**：先用宽松版（允许 `'unsafe-inline'` style + `data:` img），生产稳定后再 strict 化。ADR-0010 留 follow-up。
- **`/auth/refresh` 24h grace 折衷**：拒绝 > 24h 过期 token 是基本安全；7 天 inactive 兜底防止账号撤销但 token 仍能刷。
- **Pre-commit hook 体感**：每改 schema 都 regen 可能慢（uv run 冷启动 ~3s）；如吐槽明显，改成 CI gate 而非 pre-commit。
- **Grafana service 在 docker-compose 默认开启**：dev 资源占用 ~200MB；用 `profile: monitoring` 标记成可选启动（`docker compose --profile monitoring up`），默认不拉起。
- **单测从 22% 推到 25%**：~55 case 工作量集中；如发现 ProjectSettingsPage 单测过于复杂（842 行组件），降级为部分覆盖 + 推到下版补足。
