# v0.9.22 · CVAT 对齐的视频渲染分层与时间轴分桶基建

> 日期：2026-05-12
> 范围来源：`ROADMAP/2026-05-12-video-workbench-rendering-optimization.md` Wave 2 前置项：R2 + R3.2

## 1. 背景与假设

v0.9.21 已完成 R1 帧时钟 / frame timetable 第一版，以及 R3.1 / R3.3 的轨迹插值二分查找和 LRU。下一步不能只做“把一个 SVG 拆成两个组件”的浅层重排，而要向 CVAT 的 canvas 模块边界对齐：

- CVAT `cvat-canvas/src/typescript/canvasView.ts` 明确拆出 `background canvas`、`masks canvas`、`bitmap canvas`、`grid SVG`、`content SVG`、`text SVG`、`attachment board`。
- CVAT `canvasModel.ts` 用 mode / busy guard 约束 `setup / draw / edit / interact`，避免切帧、拖拽、编辑并发踩状态。
- CVAT `shared.ts` 统一做 client 坐标与 SVG/canvas 坐标互转，避免事件处理里散落手算。
- CVAT `frames.ts` 用 `latestFrameDecodeRequest` 丢弃过期帧请求；这一点与 v0.9.21 的 FrameClock 请求序列一致。

当前视频工作台剩余的渲染瓶颈主要在前端：

1. `VideoFrameOverlay` 仍是单个 SVG，轨迹形状、选中态、resize handle、draft / ghost 和 pointer handler 混在一起。
2. 每个可见 bbox 节点都带 `pointerdown`，密集轨迹下会扩大 React diff 和事件绑定成本。
3. 时间轴升级 R4 需要 keyframe / absent / source 的分桶数据，但当前只有 `annotatedFrames` 的扁平数组。

本版假设：

1. 只对齐 CVAT 的模块边界和数据流，不照搬 CVAT 的 imperative canvas runtime。
2. 不引入 `fabric`、`svg.js`、`svg.draggable.js` 等依赖，继续使用现有 React + SVG + HTML video。
3. 不改 API 和 `video_track` wire shape，继续沿用 `absent` 单帧语义；`outside` 段协议留给后续 R3.4。

## 2. 目标

成功标准：

1. `VideoFrameOverlay` 升级为 CVAT-aligned stage surface，至少拆出 Media / Bitmap / Grid / Objects / Text / Interaction / Attachment 七个逻辑层。
2. bbox 主体、track path、label、选中态、handle、draft、ghost 分属明确层级，视觉顺序和命中顺序可测试。
3. bbox 主体渲染按 track / annotation 做 `memo`，当前帧不变时避免无意义重渲染。
4. 拖拽、缩放、draft 更新只影响 Interaction 层；Objects 层保留 committed geometry。
5. 新增几何 picker，交互层通过 `pickTopVideoEntryAt(point)` 找到命中的顶层框，减少每个 bbox 的事件绑定。
6. 新增 `VideoStageMode` 与 busy guard，防止 seek/frame setup 与 draw/drag/resize 并发覆盖状态。
7. 新增统一坐标转换 helper，所有 pointer 入口走同一套 `client -> stage -> video` 映射。
8. 新增 frame bucket helper，能从 `video_track` keyframes 生成 `Map<frame, trackId[]>` 和 timeline marker 数据，供 R4 复用。
9. 保持现有视频框绘制、选中、拖拽、缩放、空帧 ghost、播放 overlay 行为不变。
10. 增加针对 layer contract、mode guard、坐标转换、picker、frame bucket 的单元测试。

## 3. 不做范围

- 不做 R4 时间轴 UI 重设计；本版只产出分桶数据和必要的 marker helper。
- 不做 R3.4 `outside: [{ from, to }]` 协议，仍兼容 `keyframes[].absent`。
- 不做 R5 ImageBitmap / poster / chunk 解码。
- 不做 R8 viewport / pan-zoom，只在层容器上保留可同步 transform 的结构入口。
- 不引入 CVAT 的 `fabric` / `svg.js` / plugin 体系，不把当前 React 组件改写为 CVAT 风格 imperative canvas 类。
- 不实现 mask / polygon / polyline track；本版只为这些形态保留层级位置。
- 不改后端、OpenAPI、数据库迁移或导出协议。

## 4. 前端实现

### 4.1 CVAT-aligned layer contract

目标文件：

- `apps/web/src/pages/Workbench/stage/VideoFrameOverlay.tsx`
- 新增 `apps/web/src/pages/Workbench/stage/VideoStageSurface.tsx`
- 新增 `apps/web/src/pages/Workbench/stage/VideoMediaLayer.tsx`
- 新增 `apps/web/src/pages/Workbench/stage/VideoBitmapLayer.tsx`
- 新增 `apps/web/src/pages/Workbench/stage/VideoGridLayer.tsx`
- 新增 `apps/web/src/pages/Workbench/stage/VideoObjectsLayer.tsx`
- 新增 `apps/web/src/pages/Workbench/stage/VideoTextLayer.tsx`
- 新增 `apps/web/src/pages/Workbench/stage/VideoInteractionLayer.tsx`
- 新增 `apps/web/src/pages/Workbench/stage/VideoAttachmentLayer.tsx`

拆分原则：

1. `VideoStageSurface` 只负责共同的尺寸、aspect ratio、`viewBox`、层叠顺序、可选 transform。
2. `VideoMediaLayer` 承载当前 `<video>`，由 `useFrameClock` 驱动，不挂 annotation pointer handler。
3. `VideoBitmapLayer` 本版默认空实现，保留 `ImageBitmap` / canvas 背景层入口，供 R5.2 使用。
4. `VideoGridLayer` 本版默认隐藏，保留 grid / viewport debug 层入口，供 R8 使用。
5. `VideoObjectsLayer` 渲染 committed annotation：
   - track path preview
   - 当前帧 bbox 主体
   - selected / interpolated / occluded 的基础视觉状态
6. `VideoTextLayer` 单独渲染 label，参考 CVAT 的 text SVG 分离做法，避免 label 遮挡 handle 命中。
7. `VideoInteractionLayer` 渲染：
   - 选中框高亮
   - resize hit area / handle
   - selected track ghost
   - draft drawing rect
   - pointer move / up / cancel / leave
8. `VideoAttachmentLayer` 本版仅保留挂载点，用于后续 review issue、comment、hover thumbnail、context menu 等 DOM 锚点。
9. 所有层使用同一套 coordinate space；R8 前 transform 默认为恒等。

层级顺序：

```text
VideoStageSurface
  1. VideoMediaLayer        HTML video
  2. VideoBitmapLayer       canvas, hidden in v0.9.22
  3. VideoGridLayer         SVG, hidden by default
  4. VideoObjectsLayer      SVG shapes / paths
  5. VideoTextLayer         SVG labels
  6. VideoInteractionLayer  SVG selection / handles / draft / ghost
  7. VideoAttachmentLayer   HTML anchors / future overlays
```

### 4.2 Track shape memo

目标文件：

- 新增 `apps/web/src/pages/Workbench/stage/VideoTrackShape.tsx`
- `apps/web/src/pages/Workbench/stage/VideoObjectsLayer.tsx`
- `apps/web/src/pages/Workbench/stage/VideoTextLayer.tsx`

实现策略：

1. 把单个 entry 的 bbox 主体收到 `VideoTrackShape`。
2. 把 label 收到 `VideoTrackLabel`，由 Text 层统一渲染。
3. 使用 `React.memo`，比较 `entry.id`、`geom`、`selected`、`source`、`className`、`occluded`、`readOnly` 和锁定相关输入。
4. 拖拽中的几何只传给 Interaction 层；Objects / Text 层保留 committed geometry，避免拖拽时整批 bbox 变化。
5. 保持 `vectorEffect="non-scaling-stroke"`，为后续 viewport 放大铺路。

### 4.3 坐标转换与几何 picker

目标文件：

- 新增 `apps/web/src/pages/Workbench/stage/videoStageCoordinates.ts`
- 新增 `apps/web/src/pages/Workbench/stage/videoStagePicking.ts`
- `apps/web/src/pages/Workbench/stage/VideoFrameOverlay.tsx`
- `apps/web/src/pages/Workbench/stage/VideoStage.tsx`

实现策略：

1. 新增 `clientPointToVideoPoint(svg, clientPoint, viewBoxHeight)`，对齐 CVAT `translateToSVG` 的职责。
2. 新增 `videoPointToClientPoint(svg, point, viewBoxHeight)`，用于后续 tooltip / attachment 定位。
3. 坐标 helper 内部使用 `getScreenCTM().inverse()`，测试里用可注入 matrix 避免 jsdom 缺失 DOMMatrix 时脆弱。
4. 新增 `pickTopVideoEntryAt(entries, point, options)`：
   - 从数组末尾向前扫描，保持“后渲染的框优先命中”。
   - 支持 bbox 内部命中和可配置 padding。
   - 跳过 locked / non-interactive entry 的逻辑由调用方决定，不塞进 picker。
5. Interaction 层只挂一个 `onPointerDown`：
   - 命中 entry 时调用既有 `onBeginMove`。
   - 未命中时走既有 `onBeginDraw`。
6. resize handle 仍保留独立 hit area，因为它们是选中框的明确编辑 affordance。

### 4.4 Stage mode 与 busy guard

目标文件：

- 新增 `apps/web/src/pages/Workbench/stage/videoStageMode.ts`
- `apps/web/src/pages/Workbench/stage/VideoStage.tsx`
- `apps/web/src/pages/Workbench/stage/useFrameClock.ts`

数据结构：

```ts
type VideoStageMode = "idle" | "draw" | "drag" | "resize" | "interact" | "pan" | "zoom";

interface VideoStageModeGuard {
  mode: VideoStageMode;
  canSetupFrame: boolean;
  canBeginDraw: boolean;
  canBeginDrag: boolean;
  canBeginResize: boolean;
}
```

实现策略：

1. `idle` 允许 seek / setup frame / draw / drag / resize。
2. `draw` 允许同帧 pointer move / finish / cancel，不允许外部 frame setup 覆盖 draft。
3. `drag` / `resize` 不允许切到新 frame；如果 `frameClock` 收到播放 tick，先暂停或丢弃该 tick，保持现有编辑优先级。
4. `interact` 预留给后续 AI/SAM prompt，不在本版接 UI。
5. `pan` / `zoom` 预留给 R8，不在本版暴露入口。
6. busy guard 只约束前端交互，不改变持久化协议。

### 4.5 Internal event boundary

CVAT 对外用 `canvas.drawn / canvas.edited / canvas.moved / canvas.resizeshape / canvas.interacted` 等 DOM event。本项目不直接采用 DOM CustomEvent，但需要建立同样清晰的内部事件边界。

目标文件：

- 新增 `apps/web/src/pages/Workbench/stage/videoStageEvents.ts`
- `apps/web/src/pages/Workbench/stage/VideoStage.tsx`

事件类型：

```ts
type VideoStageEvent =
  | { type: "drawn"; geom: VideoStageGeom; className: string }
  | { type: "moved"; id: string; geom: VideoStageGeom }
  | { type: "resized"; id: string; geom: VideoStageGeom }
  | { type: "selected"; id: string | null }
  | { type: "canceled" };
```

实现策略：

1. Layer 组件只发 `VideoStageEvent`，不直接改 annotation state。
2. `VideoStage` 仍负责把 event 转成现有 `onChangeAnnotations` / history command。
3. 这一层是后续 split / merge / tracker / issue anchoring 的入口，避免功能继续塞进 pointer handler。

### 4.6 Frame bucket 基建

目标文件：

- 新增 `apps/web/src/pages/Workbench/stage/videoFrameBuckets.ts`
- 新增 `apps/web/src/pages/Workbench/stage/videoFrameBuckets.test.ts`
- `apps/web/src/pages/Workbench/stage/VideoPlaybackOverlay.tsx` 仅做类型预留，暂不改 UI 视觉。

数据结构：

```ts
interface VideoFrameBucket {
  frame: number;
  trackIds: string[];
  hasManual: boolean;
  hasPrediction: boolean;
  hasAbsent: boolean;
}
```

实现策略：

1. 从 `VideoTrackGeometry[]` 构建 `Map<number, VideoFrameBucket>`。
2. 同一帧多个 track 合并；同一 track 重复 keyframe 以最后一次为准。
3. `source === "prediction"` 标记 `hasPrediction`，否则标记 `hasManual`。
4. `absent=true` 标记 `hasAbsent`，但仍保留对应 trackId，供后续 R4 outside 可视化升级。
5. helper 输出稳定排序的 marker 数组，避免时间轴组件重复排序。

验收关注点：

1. 绘制新框、选中框、拖拽框、resize handle、空帧 ghost 行为与 v0.9.21 一致。
2. 拖拽或 resize 过程中播放 tick / seek 不能覆盖当前编辑状态。
3. Objects 层没有每个 bbox 的 `onPointerDown`，命中选择走 Interaction 层统一 picker。
4. label 在 Text 层显示，但不吞掉 resize handle 和 bbox 命中。
5. `data-testid` 或层级标识能证明七个逻辑层按顺序挂载。

## 7. 风险

1. Pointer 事件从 bbox 节点迁到 Interaction 层后，最容易回归的是选中、拖拽、缩放和绘制的优先级；需要用现有 `VideoStage` 测试守住。
2. `React.memo` 若比较函数过宽，可能导致 className / selected 状态不刷新；比较字段必须显式覆盖可见状态。
3. 分层后 label 与 bbox 的 z-index 顺序要保持现状，不能让 label 遮挡 handle 命中。
4. mode guard 若过严，可能阻断合法的快速 seek；只在 draw / drag / resize 期间拦截 frame setup。
5. 坐标转换依赖 SVG CTM，测试环境需要隔离 DOMMatrix 缺口，不能让 jsdom 限制反向污染实现。
6. R2.4 只预留 transform 参数，不引入 viewport 状态，避免把 R8 提前混入本版。

## 8. CVAT 对齐映射

| CVAT 概念 | 本版对应 | 本版处理 |
| --- | --- | --- |
| `background canvas` | `VideoMediaLayer` | 当前仍是 `<video>`，不转 canvas |
| `bitmap canvas` | `VideoBitmapLayer` | 占位，R5.2 再接 ImageBitmap |
| `grid SVG` | `VideoGridLayer` | 占位，R8 再接 viewport/grid |
| `content SVG` | `VideoObjectsLayer` | bbox / track path committed 渲染 |
| `text SVG` | `VideoTextLayer` | label 独立渲染 |
| `attachment board` | `VideoAttachmentLayer` | hover / issue / comment anchor 占位 |
| `Mode.IDLE/DRAW/EDIT/...` | `VideoStageMode` | 轻量 busy guard，不引入 CVAT runtime |
| `translateToSVG/translateFromSVG` | `videoStageCoordinates.ts` | 统一坐标转换 |
| `canvas.*` DOM events | `VideoStageEvent` | typed internal event，保持 React 数据流 |
| `latestFrameDecodeRequest` | v0.9.21 FrameClock + mode guard | 过期请求丢弃，编辑态优先 |
