# v0.9.29 · Video J/K/L Playback + Atomic Seek Plan

> 范围来源：`ROADMAP/2026-05-12-video-workbench-rendering-optimization.md` R18 + R6.1。

## 1. 目标

v0.9.29 把视频工作台的播放控制从「Space + 逐帧 seek」扩展到 NLE 风格的 `J / K / L`：

- `L` 正向播放 / 加速。
- `K` 暂停。
- `J` 反向播放 / 减速。
- 所有显式 seek 入口统一走 `seekFrameAsync`，连续 seek 时只采纳最新结果，避免旧 seek 回调覆盖新位置。

## 2. 范围

### 做

1. `useFrameClock` 暴露 `seekToAsync(frame)`，复用现有 `requestVideoFrameCallback` / `seeked` / timeout 降级路径。
2. `VideoStage` 新增 `seekFrameAsync(frame, opts)`，让时间轴 scrub、逐帧、关键帧跳转、bookmark、jump history 和 loop region 都走同一 seek 原语。
3. `VideoStage` 新增 J/K/L 播放状态：
   - 速度档位：`0.25x / 0.5x / 1x / 2x / 4x`。
   - 正向播放复用浏览器视频播放并设置 `playbackRate`。
   - 反向播放不用 `playbackRate = -1`，改为按帧步进调用 `seekFrameAsync`。
4. `useWorkbenchHotkeys` / `hotkeys.ts` 接入 `J / K / L` 视频模式快捷键。
5. `VideoPlaybackOverlay` 显示当前 J/K/L 速度状态。
6. 补单测：hotkey dispatch、VideoStage 控制、loop 边界与反向播放。
7. 更新 `CHANGELOG.md`、concept docs、roadmap outcome。

### 不做

- 不引入 WebCodecs / chunk decode。
- 不做 ImageBitmap 背景缓存。
- 不把播放速度持久化到后端或 sessionStorage。
- 不改变图片工作台已有 `J / K` 框选择快捷键；只在 `videoMode` 下接管。

## 4. Outcome

- 已落地 `useFrameClock.seekToAsync`，连续 seek 被新目标覆盖时旧 promise 返回 stale。
- `VideoStage` 的 timeline scrub、逐帧、关键帧跳转、bookmark、jump history 和 loop 起跳统一走 `seekFrameAsync`。
- 视频模式新增 `J / K / L`：正向播放使用浏览器 `playbackRate`，反向播放按帧调用 `seekFrameAsync`，不设置负 playbackRate。
- `VideoPlaybackOverlay` 在 jog 播放时显示当前速度标签。
- 已补 `hotkeys.test.ts`、`VideoStage.test.tsx` 和 `VideoPlaybackOverlay.test.tsx` 覆盖。
