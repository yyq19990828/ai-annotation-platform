# v0.9.36 · Video Tracker GPU / Model Deepening

## 背景

`ROADMAP/2026-05-12-video-backend-frame-service.md` 中 S4/S5 已经落地：`VideoTrackerJob` 编排壳、GPU queue、`mock_bbox` adapter、事件流和 `video_track` prediction keyframes 写回。当前缺口是 S6：真实 SAM 2 / SAM 3 video predictor 不能通过平台协议发起，长 frame range 也没有后端分窗和 OOM 运维指引。

## 假设

- SAM 系列仍遵循 ADR-0012：模型运行在独立 ML Backend 服务里，平台 API/Celery 不直接加载 torch / CUDA。
- v0.9.36 只定义并实现平台侧协议桥；真实 backend 可以后续按协议补 `/predict` 的 `context.type="video_tracker"` 分支。
- `mock_bbox` 继续作为 contract adapter，用于无 GPU CI 和前端 R10 对接。
- 低置信度结果不覆盖 manual keyframe，按 outside prediction range 写回。

## 范围

1. 增加 `sam2_video` / `sam3_video` tracker adapter model key。
2. adapter 通过项目绑定的 connected ML Backend 调 `/predict`，发送 `context.type="video_tracker"`、frame range、direction、prompt、source geometry。
3. worker 按 `VIDEO_TRACKER_WINDOW_SIZE_FRAMES` 对长区间分窗调用 adapter，避免单次请求撑爆 GPU 显存。
4. 增加低置信度 outside 阈值配置 `VIDEO_TRACKER_LOW_CONFIDENCE_OUTSIDE_THRESHOLD`。
5. 补测试覆盖外部 backend adapter payload、分窗、低置信度 outside 写回。
6. 同步 `.env.example`、env-vars、video-frame-service 协议文档、runbook、roadmap 状态。

## 不做

- 不在 `apps/api` 内直接集成 SAM 2 / SAM 3 Python predictor。
- 不新增数据库表或迁移。
- 不改前端 UI。
- 不实现 SSE facade 或独立 GPU scheduler。

## 成功标准

- `model_key="sam2_video"` / `"sam3_video"` 能路由到项目绑定 ML Backend，且无 backend 时失败原因明确。
- 长 frame range 会拆成多个 backend 请求，事件流仍按整体 job 推送 progress。
- backend 返回低置信度结果时写入 `outside` prediction range，不覆盖人工 keyframe。
- docs/runbook 明确 GPU queue、OOM、分窗与 worker restart 操作。
