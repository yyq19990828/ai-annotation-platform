# v0.9.31 · Video Observability Pack

## 背景

v0.9.27 已接入视频时间轴 hover 缩略图，v0.9.29 已完成 J/K/L 多速率播放和 `seekFrameAsync`。下一步不继续扩张编辑能力，先把视频工作台的性能回归和 BUG 反馈诊断补齐，避免后续 R5.2 / R8 / Review anchor 开发时只能靠手感判断。

## 范围

来源：`ROADMAP/2026-05-12-video-workbench-rendering-optimization.md` 的 R7.2 / R7.3。

1. 固化视频 bench fixture 描述与脚本入口。
   - 覆盖 720p/3min、1080p/5min、4K/30s 三组视频。
   - 覆盖 10 / 100 / 500 tracks 三档 annotation 密度。
   - 先生成本地 run manifest 和 PR 附件路径，不上传 trace。
2. BugReportDrawer 在视频工作台提交反馈时自动附带诊断。
   - `window.__videoFrameClockDiagnostics` / 视频工作台诊断快照。
   - 最近 seek 耗时、long task 计数、frame preview cache 状态。
   - 当前 frame、timeline mode、J/K/L 播放状态。
3. docs-site 增加视频性能回归 how-to，并在视频工作台概念文档中记录诊断链路。

## 不做

- 不引入 WebCodecs / ImageBitmap 背景缓存。
- 不把 trace 上传为长期资产。
- 不新增 bug report 后端字段；本版沿用描述与 JSON console payload。
- 不新增真实 fixture 视频到仓库。

## Outcome

已落地：

- 新增 `apps/web/scripts/video-bench/fixtures.json` 与 `run-video-bench.mjs`，并在 `@anno/web` 暴露 `video:bench` 脚本。
- `useFrameClock` diagnostics 记录最近 seek 样本，`useVideoFramePreview` diagnostics 记录 cache / prefetch / unsupported 状态。

- 更新 `CHANGELOG.md`、本 ROADMAP 相关切片、`docs-site/dev/concepts/video-annotation-workbench.md` 与新增 docs-site how-to。
