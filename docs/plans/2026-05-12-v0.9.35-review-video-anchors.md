# v0.9.35 · Review Video Anchors

## 背景

`ROADMAP/2026-05-12-video-workbench-rendering-optimization.md` 的下一个近期切片是 V4 Review 模式视频差异化。v0.9.34 已把 tracker adapter 后端闭环跑通，review 端还缺少对视频轨迹来源和帧位置的明确锚定，审核员只能对整条 annotation 留言，回放定位成本高。

## 假设

- 评论仍挂在现有 `annotation_comments` 表下，不新增独立 review issue 表。
- 本版只做对象级 / 帧级锚点，不做像素级视频 diff 或正式审核评论线程模型。
- `raw / final / diff` 复用现有 review diff mode；视频侧解释为 prediction/interpolated 来源、manual/legacy 最终结果和叠加视图。
- Track Composition（bbox → track 聚合、split、merge）不在本版范围。

## 范围

1. 评论协议增加可选 `anchor`：
   - 支持 `kind="video_frame"`、`frame_index`、`track_id`、`source`。
   - 创建、列表、分页接口原样透出。
2. Review 视频工作台支持 raw / final / diff：
   - `raw` 只显示 prediction / interpolated 视频轨迹。
   - `final` 只显示 manual / legacy 视频标注。
   - `diff` 保持叠加。
3. 评论面板在视频任务中自动带入当前帧锚点：
   - 选中 `video_track` 时记录当前 frame、track id 和当前帧 source。
   - 评论列表渲染锚点 chip，点击可跳到对应帧。
4. 轨迹侧栏展示当前帧来源，并支持跳到下一条 prediction keyframe。
5. 同步 changelog、roadmap 和视频工作台概念文档。

## 不做

- 不新增后端 review issue 表。
- 不做视频像素级 diff。
- 不做 track split / merge / bbox 聚合。
- 不改变现有图片评论画布批注协议。
