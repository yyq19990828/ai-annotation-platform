# v0.9.23 · 视频 outside 段协议与时间轴语义基建

> 日期：2026-05-12
> 范围来源：`ROADMAP/2026-05-12-video-workbench-rendering-optimization.md` R3.4 + R4 前置数据层

## 1. 背景与假设

v0.9.22 已完成 CVAT-aligned 的视频渲染分层，以及 keyframe bucket 基建。当前剩余的最近缺口是 R3.4：我们仍用 `keyframes[].absent = true` 表达目标消失，这只能覆盖单帧，不适合后续时间轴、AI tracker、review 评论和导出统一语义。

本版假设：

1. 继续保持 `video_track` compact JSON，不逐帧展开写库。
2. `absent` 单帧语义必须向后兼容；旧数据和旧导出结果不能失效。
3. `outside` 本版只表达“目标在某段帧区间不存在”，不把遮挡 `occluded`、锁定、审核状态混进同一个字段。
4. 时间轴不在本版重做成完整多轨 UI；本版只交付 R4 需要的 marker / segment 数据结构和轻量可视化入口。
5. 不引入后端迁移；`geometry` 是 JSONB，新增可选字段通过 Pydantic schema 和服务层归一化处理。

## 2. 目标

成功标准：

1. `video_track` 支持可选 `outside: [{ from, to, source? }]`，帧区间为闭区间，`from <= to`。
2. 前后端都提供统一归一化逻辑：排序、合并重叠或相邻区间、过滤非法区间，并兼容旧 `absent` keyframe。
3. 渲染解析、最近关键帧、时间轴分桶、导出、track -> bbox 转换全部走同一套 effective outside 判断。
4. 用户在当前帧“标记消失”时写入 outside 单帧区间；后续恢复可见或写入可见关键帧时能移除该帧的 outside 覆盖。
5. `videoFrameBuckets` 升级为 timeline marker 基础：能输出 keyframe marker、outside segment、source 标记和全局密度。
6. `VideoPlaybackOverlay` 保持现有布局，只把底部标记升级为有语义的数据源，支持 outside 灰段 / prediction 标记的最小展示。
7. `format=coco` 的 Video Tracks JSON 在 `keyframes` 模式和 `all_frames` 模式下都尊重 outside 段；旧 `absent` keyframe 仍阻断插值。
8. 文档同步说明 `absent` 与 `outside` 的兼容关系，避免后续协议误用。

## 3. 不做范围

- 不做完整 R4 多轨时间轴、loop region、章节、hover thumbnail。
- 不做 range 拖拽编辑 UI；本版只支持当前帧单点 outside 写入和内部 range 语义。
- 不做 V4 review raw/final 视图。
- 不做 R10 AI tracker 自动写 outside；只预留 `source: "manual" | "prediction"`。
- 不做 R9 polygon / mask track；`outside` 先只服务 bbox track。
- 不新增数据库迁移，不重写 annotation 表结构。

## 4. 协议设计

### 4.1 前端类型

目标文件：

- `apps/web/src/types/index.ts`
- `apps/web/src/pages/Workbench/stage/videoStageTypes.ts`

新增类型：

```ts
export type VideoTrackOutsideRange = {
  from: number;
  to: number;
  source?: "manual" | "prediction";
};

export type VideoTrackGeometry = {
  type: "video_track";
  track_id: string;
  keyframes: VideoTrackKeyframe[];
  outside?: VideoTrackOutsideRange[];
};
```

约定：

1. `from` / `to` 都是 frame index，闭区间。
2. 相邻区间 `[3, 5]` + `[6, 9]` 归一化为 `[3, 9]`。
3. `source` 缺省按 `manual` 处理；后续 tracker 可写 `prediction`。
4. `outside` 对渲染和导出有最高优先级：某帧落在 outside 内时，该帧不显示对象、不导出 bbox。
5. 写入可见 keyframe 时，必须从 outside 中移除目标帧，避免同一帧同时“可见”和“消失”。

### 4.2 后端 schema

目标文件：

- `apps/api/app/schemas/_jsonb_types.py`

新增：

```py
class VideoTrackOutsideRange(BaseModel):
    from_: int = Field(alias="from", ge=0)
    to: int = Field(ge=0)
    source: Literal["manual", "prediction"] = "manual"

    model_config = ConfigDict(populate_by_name=True, extra="forbid")
```

并在 `VideoTrackGeometry` 增加：

```py
outside: list[VideoTrackOutsideRange] = Field(default_factory=list)
```

注意：schema 只负责接受字段；排序、合并、legacy absent 兼容放在服务层 helper，避免 Pydantic model 承担业务语义。

## 5. 前端实现

### 5.1 outside helper

新增文件：

- `apps/web/src/pages/Workbench/stage/videoTrackOutside.ts`
- `apps/web/src/pages/Workbench/stage/videoTrackOutside.test.ts`

核心函数：

```ts
normalizeOutsideRanges(ranges): VideoTrackOutsideRange[]
legacyAbsentRanges(track): VideoTrackOutsideRange[]
effectiveOutsideRanges(track): VideoTrackOutsideRange[]
isFrameOutside(track, frameIndex): boolean
addOutsideRange(track, range): VideoTrackGeometry
removeOutsideFrame(track, frameIndex): VideoTrackGeometry
```

实现规则：

1. `effectiveOutsideRanges()` 合并 `geometry.outside` 与旧 `keyframes[].absent` 单帧。
2. `addOutsideRange()` 只更新 `outside`，不删除现有 keyframes；这样不会因为标记消失误删用户画过的关键帧。
3. `removeOutsideFrame()` 支持把一个 range 拆成左右两段，例如 `[3, 8]` 移除 `5` 后得到 `[3, 4]` 和 `[6, 8]`。
4. `upsertKeyframe(..., { absent: false })` 或复制关键帧到当前帧时，调用 `removeOutsideFrame()` 清理当前帧。
5. 旧 `absent` keyframe 暂不自动迁移；文档声明它是兼容输入，新的 UI 写入优先使用 `outside`。

### 5.2 轨迹解析与渲染

目标文件：

- `apps/web/src/pages/Workbench/stage/videoStageGeometry.ts`
- `apps/web/src/pages/Workbench/stage/VideoStage.tsx`
- `apps/web/src/pages/Workbench/stage/VideoTrackSidebar.tsx`
- `apps/web/src/pages/Workbench/stage/VideoTrackPanel.tsx`

改造点：

1. `TrackIndex` 增加 `outsideRanges`，`resolveTrackAtFrame()` 先判断 outside，再处理 exact keyframe 和插值。
2. 插值阻断从“中间是否有 absent keyframe”升级为“两个关键帧之间是否与 outside range 相交”。
3. `nearestTrackKeyframe()` 继续找最近可见 keyframe，outside 帧上只用于 ghost 参考，不表示当前帧存在对象。
4. `标记消失` 改为写入 `outside: [{ from: frameIndex, to: frameIndex }]`。
5. `复制到当前帧`、拖拽 ghost 写入当前帧、调整遮挡状态时清理当前帧 outside。
6. 关键帧列表仍展示旧 `absent` keyframe；如果当前帧只由 `outside` 覆盖，当前状态文案显示“当前消失”。

### 5.3 时间轴 marker 数据

目标文件：

- `apps/web/src/pages/Workbench/stage/videoFrameBuckets.ts`
- `apps/web/src/pages/Workbench/stage/videoFrameBuckets.test.ts`
- 可选新增 `apps/web/src/pages/Workbench/stage/videoTrackTimeline.ts`

数据结构：

```ts
type VideoTimelineMarker =
  | { type: "keyframe"; frame: number; trackIds: string[]; hasManual: boolean; hasPrediction: boolean }
  | { type: "outside"; from: number; to: number; trackIds: string[]; hasPrediction: boolean };
```

实现策略：

1. 保留现有 `VideoFrameBucket`，扩展 marker 输出，不破坏 v0.9.22 使用方。
2. `outside` segment 单独输出，后续 R4 可直接画区间，不需要从散点反推。
3. 全局密度仍按 keyframe 数统计，outside segment 只作为负空间提示，不计入可见对象密度。
4. 选中 track 时输出单轨 markers；未选中时输出全局合并 markers。

### 5.4 Playback overlay 轻量展示

目标文件：

- `apps/web/src/pages/Workbench/stage/VideoPlaybackOverlay.tsx`
- `apps/web/src/pages/Workbench/stage/VideoStage.tsx`

改造策略：

1. 继续使用当前 `range` seekbar，不替换布局。
2. `annotatedFrames` 升级为可选 `timelineMarkers`，旧 prop 可保留兼容测试。
3. keyframe marker 仍是细竖线；prediction 用不同色；outside 用灰色半透明区间。
4. hover tooltip 只展示当前帧号和时间，不做缩略图。
5. 指针 seek 行为不变，避免本版把 timeline UI 变成大范围重构。

## 6. 后端实现

### 6.1 服务层 outside helper

目标文件：

- `apps/api/app/services/video_tracks.py`

新增 helper：

```py
def normalize_outside_ranges(geometry: dict) -> list[dict]: ...
def legacy_absent_ranges(keyframes: list[dict]) -> list[dict]: ...
def effective_outside_ranges(geometry: dict) -> list[dict]: ...
def frame_is_outside(geometry: dict, frame_index: int) -> bool: ...
def range_intersects_outside(ranges: list[dict], from_frame: int, to_frame: int) -> bool: ...
```

改造点：

1. `resolve_track_at_frame()` 接收 `geometry` 或 outside ranges，不再只看 keyframes。
2. `resolved_track_frames(..., frame_mode="keyframes")` 过滤落在 outside 的 keyframe。
3. `resolved_track_frames(..., frame_mode="all_frames")` 遇到 outside 段时不输出 bbox，并阻断两侧插值。
4. 旧 `absent` keyframe 继续进入 effective outside，保证历史数据导出行为不变。

### 6.2 转换与导出

目标文件：

- `apps/api/app/services/export.py`
- `apps/api/app/services/annotation.py`
- `apps/api/tests/test_video_workbench.py`

改造点：

1. Video Tracks JSON 导出保留 `outside` 字段，便于互操作和后续回导。
2. `track -> video_bbox` 的 `all_frames` 展开使用 outside 过滤。
3. `frame` split 模式如果当前帧在 outside 内，仍返回“需要精确可见关键帧”的 400，不生成空 bbox。
4. 测试覆盖：outside 过滤 keyframes、outside 阻断 all_frames 插值、legacy absent 兼容、outside 与 visible keyframe 冲突时 outside 优先。

验收场景：

6. 时间轴：keyframe marker、prediction marker、outside 灰段在现有控制条中可区分。

## 9. 风险与控制

| 风险 | 控制 |
| --- | --- |
| `outside` 与旧 `absent` 同时存在导致语义冲突 | 所有读路径走 `effectiveOutsideRanges()`，outside 优先；写路径逐步停止产生新 `absent` |
| 单帧 outside 写入后用户无法恢复可见 | 写入可见 keyframe / 复制关键帧时自动 `removeOutsideFrame()` |
| 时间轴 UI 范围膨胀 | 本版只做 markers 和现有 seekbar 的轻量展示，不做多轨编辑 |
| 后端导出与前端渲染不一致 | 前后端 helper 都覆盖相同测试矩阵，并在文档中固定闭区间语义 |
| JSONB schema 新字段影响旧数据 | `outside` 可选且默认空数组，不做迁移 |
