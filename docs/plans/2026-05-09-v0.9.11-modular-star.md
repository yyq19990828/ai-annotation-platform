## 模块 ① CSP nonce-based 收紧（script-src only）

### 目标
`script-src` 从 `'self' 'unsafe-inline' https://challenges.cloudflare.com` 改为 `'self' 'nonce-XXX' https://challenges.cloudflare.com`，每次请求生成 16 字节随机 nonce 并注入 `index.html`。`style-src 'unsafe-inline'` 保留（迁移留 v0.10.x）。

### 实施方案：Nginx sub_filter + vite plugin 占位符

**为什么走 Nginx 而不是 FastAPI**：SPA 由 `infra/docker/nginx.conf` serve（FastAPI 不出 HTML），改造点最小。

### 改动清单

**A. Vite plugin 生成 build-time nonce 占位符**
- 新增 `apps/web/vite-plugins/csp-nonce.ts`：自定义 plugin，在 `transformIndexHtml` hook 中给所有 `<script>` 加 `nonce="__CSP_NONCE__"` 属性。
- 修改 `apps/web/vite.config.ts`：注册 plugin。
- 验证：`pnpm build` 后 `apps/web/dist/index.html` 中所有 `<script>` 带 `nonce="__CSP_NONCE__"`。

**B. Nginx sub_filter 替换占位符为 per-request nonce**
- 修改 `infra/docker/nginx.conf`：
  - 用 `$request_id`（Nginx 自动生成的 16 字节 hex）作为 nonce。或加 lua 模块用 `os.urandom`。**优先 `$request_id`**（无需 lua，零运维成本）。
  - HTML 路由（`location /` serve `index.html`）加：
    ```nginx
    sub_filter '__CSP_NONCE__' '$request_id';
    sub_filter_once off;
    sub_filter_types text/html;
    ```
  - **关键**：仅对 `text/html` 启用 sub_filter，`/assets/` 静态资源不动。
  - 设置 `add_header Content-Security-Policy "...; script-src 'self' 'nonce-$request_id' https://challenges.cloudflare.com; ...";`（CSP header 在 Nginx 出站，FastAPI 的 SecurityHeadersMiddleware 仅对 API 响应起作用）。

**C. FastAPI SecurityHeadersMiddleware 对齐**
- 修改 `apps/api/app/middleware/security_headers.py:24-36`：API 响应路径下的 CSP 也改 `script-src 'self' https://challenges.cloudflare.com`（不再要 `'unsafe-inline'`，因为 API 响应不含 HTML），style-src 保持 `'unsafe-inline'`。
- **不在 middleware 里生成 nonce** —— Nginx 已负责，避免双源真相。

**D. Turnstile 脚本兼容验证**
- Turnstile 通过 `apps/web/src/lib/turnstile.ts` 动态注入 `<script src="https://challenges.cloudflare.com/...">`。这种动态注入的 script 默认**不会**继承 nonce。需要显式 `script.nonce = document.currentScript?.nonce || ''` 或读取 meta tag。
- 方案：在 `index.html` `<head>` 加 `<meta name="csp-nonce" content="__CSP_NONCE__">`，前端 `turnstile.ts` 注入时读取此 meta 并设 `script.nonce`。Nginx sub_filter 同样替换 meta 中的占位符。

**E. ADR-0010 更新**
- 修改 `docs/adr/0010-security-headers-middleware.md`：标记 script-src nonce 已落地（Status: Accepted → Updated），style-src 推迟到 v0.10.x 注明。

**F. 测试基线**
- 后端：`apps/api/app/middleware/__tests__/test_security_headers.py`（如不存在则新建），断言 API 响应 CSP header 不含 `'unsafe-inline'` in script-src。
- 前端：`apps/web/src/lib/__tests__/turnstile.test.ts` 加用例验证 script 注入时 `nonce` 属性正确读取 meta。
- 手测：`docker compose up`，浏览器 DevTools Console 不应有 CSP 违规警告。

### 改动清单

**A. PredictionShape Pydantic 模型**
- 修改 `apps/api/app/schemas/prediction.py:13`：
  ```python
  class BboxGeometry(BaseModel):
      type: Literal["bbox"]
      x: float; y: float; w: float; h: float

  class PolygonGeometry(BaseModel):
      type: Literal["polygon"]
      points: list[tuple[float, float]]

  class PredictionShape(BaseModel):
      type: str               # rectanglelabels|polygonlabels|...（保持 LabelStudio 兼容）
      class_name: str
      geometry: BboxGeometry | PolygonGeometry = Field(discriminator="type")

  class PredictionOut(BaseModel):
      result: list[PredictionShape]   # 替换原 list[dict]
      ...
  ```
  注意：实际数据流 `to_internal_shape()` 已转成内部格式（`apps/api/app/services/prediction.py:11-64`），所以 `PredictionOut.result` 用内部 shape 而非 LabelStudio 原始格式。

**C. prediction_jobs.total_cost 接通**
- 修改 `apps/api/app/workers/tasks.py:39-241` `_run_batch()`：
  - 现状：`pred_svc.create_from_ml_result()` 调用时未传 `token_meta`（line 185-193），也无 cost 累加。
  - 改动：
    1. 从 ML backend response 提取 `meta.total_cost`（如有 — grounded-sam2 当前无；占位逻辑 `cost = response.meta.get("total_cost", 0.0) if response.meta else 0.0`）。
    2. 在循环内累加 `running_total_cost += cost`。
    3. job 完成时 `job.total_cost = Decimal(running_total_cost)` 写入（line 220-224 区域）。
- **注意**：grounded-sam2-backend 目前不返回 cost，`total_cost` 累加值会是 0；保留逻辑路径，等 LLM-based backend（v0.10.x sam3 含语义提示）接入时数据自然到位。
- 单测：`apps/api/app/workers/__tests__/test_tasks.py` 加 `test_run_batch_accumulates_total_cost` 用例（mock prediction response with meta.total_cost = 0.001 × 5 → expect job.total_cost == 0.005）。
