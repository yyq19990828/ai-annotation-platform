# v0.9.28 · Video Segment Collaboration MVP

## 背景

`ROADMAP/2026-05-12-video-backend-frame-service.md` 已完成 B1/B2/B3/B6/B7 第一版，下一条后端主线是 B4 segment 协同。0.9.28 只落 segment 的最小可用后端能力，不进入 B5 tracker job，也不重写 task scheduler。

## 目标

1. 新增 `video_segments` 数据模型，按 `DatasetItem` 表达视频内可协作的 frame range。
2. 短视频或 metadata 未完整时懒生成单 segment；metadata 有 `frame_count` 时按 `VIDEO_SEGMENT_SIZE_FRAMES` 切分。
3. `manifest-v2` 返回 `segments`，旧前端可忽略该字段。
4. 新增 segment 查询与锁协议：
   - `GET /api/v1/tasks/{task_id}/video/segments`
   - `POST /api/v1/tasks/{task_id}/video/segments/{segment_id}:claim`
   - `POST /api/v1/tasks/{task_id}/video/segments/{segment_id}:heartbeat`
   - `POST /api/v1/tasks/{task_id}/video/segments/{segment_id}:release`
   - `GET /api/v1/videos/{dataset_item_id}/segments`
5. 权限复用 task 可见性；annotator 只能 claim 未分配或分配给自己的 segment，project_admin / super_admin 可释放任意锁。

## 不做

- 不拆 `Task`，不把 scheduler 改成 segment 调度器。
- 不做 presence WebSocket。
- 不做 B5 AI tracker job。
- 不新增外部服务。

## 实施步骤

1. 数据层：新增 `VideoSegment` 模型、`0058_video_segments.py` 迁移、`VIDEO_SEGMENT_SIZE_FRAMES` / `VIDEO_SEGMENT_LOCK_TTL_SECONDS` 配置。
2. 服务层：新增 segment 懒生成、查询、claim、heartbeat、release。
3. API 层：接入 task 兼容入口和 videos facade；manifest v2 加 `segments`。
4. 文档：同步 `docs-site/dev/reference/video-frame-service.md` 与 runbook。
5. 验证：运行 `pytest apps/api/tests/test_video_frame_service.py`、`pytest apps/api/tests/test_alembic_drift.py`，并尝试 `pnpm --filter @anno/docs-site build`。

## 成功标准

- 新视频访问 manifest 或 segments 时能得到稳定 segment 列表。
- claim 后锁定用户与过期时间可见；heartbeat 仅锁持有人或管理员可续约；release 可幂等清锁。
- 非分配 annotator claim 已分配给他人的 segment 返回 403。
- Alembic drift 测试通过。
