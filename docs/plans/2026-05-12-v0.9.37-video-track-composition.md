# v0.9.37 · Video Track Composition

## Summary

v0.9.37 implements the V6 Track Composition slice from `ROADMAP/2026-05-12-video-workbench-rendering-optimization.md`: manually compose legacy `video_bbox` annotations into compact `video_track` data, split a track at the current frame, and merge two compatible tracks.

## Key Changes

- Add `POST /api/v1/tasks/{task_id}/annotations/video/track-compositions` as the transactional composition endpoint.
- Support `aggregate_bboxes` for same-task, same-class `video_bbox[]` with one bbox per frame; the default behavior removes source bboxes after creating one `video_track`.
- Support `split_track` for a visible current frame; the source annotation keeps the earlier segment and a new annotation stores the later segment.
- Support `merge_tracks` for exactly two same-class, non-overlapping tracks; the survivor keeps merged keyframes and receives an `outside` gap between segments.
- Add video sidebar controls for bbox aggregation, current-track split, and two-track merge without changing the existing JSONB schema.
- Keep undo/redo working through history batch commands that restore created, deleted, and updated annotations.
- Update changelog, roadmap, video concept docs, API guide, and OpenAPI snapshot.

## Assumptions

- This release is manual composition only: no Re-ID, AI merge suggestions, multi-user conflict resolution, ImageBitmap caching, WebCodecs, viewport, or minimap.
- Bbox aggregation requires all source boxes to share one class and one task; duplicate frame indexes are rejected instead of guessed.
- Merge accepts exactly two tracks, requires one class, and rejects overlapping visible frame ranges.
- No migration is added; old `video_bbox` rows remain readable.
