# v0.9.30 · Video Timetable / Frame Cache Repair

## 背景

`ROADMAP/2026-05-12-video-backend-frame-service.md` 的 S3 目标是补齐 B6.2 与 B3 的运维闭环：旧视频可以重建 `video_frame_indices`，失败的单帧缓存可以通过 API 重新投递，poster 不再走一套独立抽帧路径。

## 范围

1. 新增 CLI：`python -m app.cli.video.rebuild_timetable`。
   - 支持按 `--dataset-item-id`、`--dataset-id` 或 `--all` 选择视频。
   - 下载源视频或 playback 视频，用现有 `probe_video_frame_timetable()` 重建 `VideoFrameIndex`。
   - 更新 `metadata.video.frame_timetable_frame_count` / `frame_timetable_error`。
2. 单帧缓存修复 API。
   - task 入口与 `/videos/{dataset_item_id}` facade 都支持 `POST .../frames:retry`。
   - 只重试 `failed` 行；`force=true` 时可重置指定帧并重新投递。
3. poster 复用 B3。
   - 视频 metadata 任务用 `VideoFrameCache(frame_index=0,width=512,format=webp)` 作为 poster 产物。
   - `DatasetItem.thumbnail_path` 与 `metadata.video.poster_frame_path` 指向同一 cache key。

## 不做

- 不实现 B1 compact / sparse timetable。
- 不改导出数据结构，只提供 rebuild 命令让旧视频与 B1 表对齐。
- 不引入 tracker job、GPU queue 或模型 adapter。
