# v0.9.4 phase 3 — Polished Contour

> 实施落地时 rename 为 `2026-05-08-v0.9.4-phase3-polished-contour.md`（CLAUDE.md plan 文件命名规范，当前路径为 plan mode 默认 scratch path，唯一可编辑文件约束）。

## Context

v0.9.4 phase 1 + phase 2 已落地（后端 file_path → SAM URL 解析 + 子工具栏拆分 + text 模式 box/mask/both）。phase 3 是 v0.9.x SAM 一期的最后一刀，目标三件事：

1. **mask→polygon 共享化** —— `apps/_shared/mask_utils/` 包**已存在且实现完整**（`polygon.py:15-62` `mask_to_polygon` + tests 7 个 case），但 `apps/grounded-sam2-backend/predictor.py:352-385` 仍用一个内嵌的 `_mask_to_polygon`。这意味着 v0.10.x sam3-backend 接入时要么重新写一遍要么大动 backend，phase 3 完成"装上共享包 + 删内嵌"才算真正解锁复用。
2. **simplify tolerance 调参 + 注入路径** —— 当前 tolerance 硬编码 1.0（像素），不同业务图（精细器件 / 大物体）合适值不同。先用 `scripts/eval_simplify.py` 跑 5 档 tolerance × N 张样本 mask 选定默认值；同时让前端 / 平台 API 能按需 override（Context body 新字段）。复杂形状（顶点 > 200）后端 WARN 日志便于运维发现"shapely simplify 没收敛到合理顶点数"的退化样本。
3. **SAM E2E 完整路径** —— v0.9.2 的 `S` 工具 + v0.9.4 phase 2 的子工具栏切换目前只有 vitest 单测保护，annotation.spec.ts 仅覆盖 bbox 拖框。phase 3 加 SAM 端到端 spec + `_test_seed` 加 ml_backend 工厂；后端响应用 `page.route()` 拦截平台 `/interactive-annotating` 端点 mock，避免 CI 起 GPU 容器。

phase 3 不引入用户可见新功能，但是 v0.10.x sam3-backend 落地的硬前置（共享包）+ 后续 SAM 体验调优的基础设施（tolerance 可调 + E2E 守卫）。

### M1 · mask_utils 共享包真接入（~0.5 天）

**问题**：`apps/_shared/mask_utils/` 已就位（`pyproject.toml` + `src/mask_utils/polygon.py` 完整 + `tests/test_polygon.py`），但 grounded-sam2-backend Dockerfile (`apps/grounded-sam2-backend/Dockerfile:64-74`) 没 COPY 它，predictor.py L352 还用内嵌版。

**关键差异**（迁移时要保留）：
- 共享包用 `cv2.CHAIN_APPROX_NONE` 保留所有顶点 + simplify 前置；内嵌版用 `cv2.CHAIN_APPROX_SIMPLE`（cv2 自带粗简化）。**保留共享包行为**（更可控）。
- 内嵌版 L367-381 有 `try/except BLE001` 兜底（shapely 偶发拓扑错误降级回原始 contour）；共享包没有。**给共享包补上**：在 `polygon.py` `simplify` 调用周围加 try 块，失败时返回未简化 contour 而非空。
- 内嵌版 L385 强制 `round(x/w, 6)` 归一化；共享包归一化由 `normalize_to=(w,h)` 选择 + `normalize.py` 的 `normalize_coords` 不一定 round(6)。**对齐 round(6) 行为**进 `normalize_coords`。

**关键文件**：
- `docker-compose.yml`（grounded-sam2-backend service）：build context 从 `./apps/grounded-sam2-backend` 升到 `./apps`，加 `dockerfile: grounded-sam2-backend/Dockerfile`
- `apps/grounded-sam2-backend/Dockerfile`：所有 COPY 路径加 `grounded-sam2-backend/` 前缀；新增 `COPY _shared/mask_utils/ /app/mask_utils/` + `RUN pip install -e /app/mask_utils`

- `apps/_shared/mask_utils/tests/test_polygon.py`：加 1 个 case 验证拓扑兜底（self-intersecting contour 输入 → 不抛异常返回非空）

## 风险

| 风险 | 缓解 |
|---|---|
| docker-compose context 升级踩坑（其它依赖 build context 的工作流断）| 提交前本地完整 `docker compose build` 全部 service 跑一遍；commit message 注明 backend 构建命令变化 |
| 共享包 CHAIN_APPROX_NONE → 顶点更多 → simplify 后形状细节变化 | M3 eval 用真实 SAM mask 跑 IoU 验证，肉眼回归在 phase 2 已落的几张样本图上做对比 |
| E2E `page.route()` mock 与 `useInteractiveAI` 的 inflight / 防抖逻辑冲突（80ms debounce）| spec 用 `await page.waitForTimeout(150)` 等防抖；mock 响应直接同步返回，不模拟延迟 |
| _test_seed ml_backend 工厂被现有 backend 单测意外依赖 | 仅在 `reset` endpoint 主动调用，不进默认 fixture 路径；现有测试用独立 mock backend 不受影响 |
