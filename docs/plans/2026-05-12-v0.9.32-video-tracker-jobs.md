# v0.9.32 · Video Tracker Job Shell

## 背景

`ROADMAP/2026-05-12-video-backend-frame-service.md` 的当前基线已经完成 B1/B2/B3/B4/B6/B7 第一版。0.9.32 只落 B5 的 job 编排壳，让前端 R10 能先对接创建、查询、取消 tracker job；真实 SAM/KCF adapter 和逐帧写回留到后续切片。

## 目标

1. 新增独立 `video_tracker_jobs` 表，不复用 `prediction_jobs`。
2. 支持状态机 `queued -> running -> completed | failed | cancelled` 的持久化字段与取消请求字段。
3. 新增 API：
   - `POST /api/v1/tasks/{task_id}/video/tracks/{annotation_id}:propagate`
   - `GET /api/v1/video-tracker-jobs/{job_id}`
   - `DELETE /api/v1/video-tracker-jobs/{job_id}`
4. 创建 job 时校验 video task、annotation 归属、frame range、可选 segment 范围与锁。
5. 暴露 Redis/WebSocket 后续可订阅的 `event_channel`，但本版不实现真实模型输出。

## 不做

- 不接 SAM 2 / SAM 3 / KCF adapter。
- 不写 prediction keyframes 或覆盖 manual annotation。
- 不新增 GPU worker 或外部服务。
- 不实现 SSE facade。

## 实施步骤

1. 数据层：新增 `VideoTrackerJob` 模型、状态枚举与 `0059_video_tracker_jobs.py` 迁移。
2. Schema/服务层：新增 create/get/cancel 请求响应、状态校验、segment lock 校验。
3. API 层：接入 task 创建入口和 `/video-tracker-jobs/{job_id}` 查询/取消入口。
4. 文档：同步 `docs-site/dev/reference/video-frame-service.md`、runbook 中 tracker job 的壳能力，并在 roadmap 标注 S4 完成。
5. 验证：运行 tracker 相关测试、`pytest apps/api/tests/test_video_frame_service.py`、`pytest apps/api/tests/test_alembic_drift.py`，必要时更新 OpenAPI snapshot。

## 成功标准

- 有锁 segment 内能创建 queued tracker job，并返回 `job_id` 与 `event_channel`。
- 无锁或跨 segment 的请求被拒绝。
- `GET` 只能读取当前用户可见 task 的 job。
- `DELETE` 对 queued/running job 幂等设置 `cancel_requested_at` 并返回 `cancelled`；对 terminal job 保持原状态。
