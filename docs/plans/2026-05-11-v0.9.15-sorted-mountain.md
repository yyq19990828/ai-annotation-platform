# v0.9.15 — Sorted Mountain (批次状态机二阶段)

## Context

v0.9.14 收尾后，0.9.x 主线（Grounded-SAM-2 接入）功能面封口。v0.9.15 是 0.9.x 终版，目标是清最后一个可现做的 P2 长尾：**实施 ADR-0008 batch admin-lock + bulk-approve/reject**。完成后 ROADMAP P2 段只剩「等外部依赖 / 客户驱动」类，立即开 v0.10.0 M0 sam3-backend。

ADR-0008 设计为 **soft hold**（不是 hard pause）：
1. 冻结 batch 自动状态推进（`check_auto_transitions` 短路）
2. 阻断 `/tasks/next` 新派单（`get_next_task` 排除 locked batch）
3. 暴露可审计的锁元数据（owner/super_admin 锁定/解锁）

## Phase 3: Bulk Approve/Reject（~1d）

### 3.1 Schema

**文件**: `apps/api/app/schemas/batch.py`

```python
class BulkBatchApprove(BaseModel):
    batch_ids: list[UUID] = Field(..., min_length=1, max_length=200)

class BulkBatchReject(BaseModel):
    batch_ids: list[UUID] = Field(..., min_length=1, max_length=200)
    feedback: str = Field(..., min_length=1, max_length=500)
```

### 3.2 Service

**文件**: `apps/api/app/services/batch.py`

- `bulk_approve(project_id, batch_ids)`: reviewing → approved，非 reviewing fail gracefully
- `bulk_reject(project_id, batch_ids, feedback, reviewer_id)`: reviewing → rejected + soft-reset (review/completed tasks → pending) + recalculate counters

### 3.3 Audit

```python
BULK_BATCH_APPROVE = "batch.bulk_approve"
BULK_BATCH_REJECT = "batch.bulk_reject"
```

### 3.4 API Endpoints

**文件**: `apps/api/app/api/v1/batches.py`

- `POST /bulk-approve` — `require_roles(*_REVIEWERS)` + `BulkBatchApprove` + audit
- `POST /bulk-reject` — `require_roles(*_REVIEWERS)` + `BulkBatchReject` + audit + notification (annotator)

**权限**：reviewer/project_admin/super_admin（与单批次 reject/approve 一致）。bulk-approve 不绕过 admin_locked（locked batch 在 `bulk_approve` service 中 fail gracefully：`cannot approve locked batch`）。

### 3.5 Tests

**文件**: `apps/api/tests/test_batch_lifecycle.py`

新增 `TestBulkApproveReject` class (8 cases):
- bulk approve reviewing → 200
- skips already approved
- fails non-reviewing
- bulk reject with feedback → tasks reset to pending
- reject sends notification to annotator
- annotator cannot bulk approve (403)
- reviewer can bulk approve (200)
- audit log created

### 3.6 Frontend

**3.6.1 API** (`apps/web/src/api/batches.ts`):

- `bulkReject(projectId, batchIds, feedback)`

**3.6.2 Hooks** (`apps/web/src/hooks/useBatches.ts`):

- `useBulkRejectBatches(projectId)`

**3.6.3 UI** (`apps/web/src/pages/Projects/sections/BatchesSection.tsx`):

- Bulk action bar 追加「通过」(green) + 「驳回」(red) 按钮
- Bulk approve: inline confirm modal（Pattern A，无 textarea）
- Bulk reject: 带 feedback textarea 的 modal

**3.6.4 BulkRejectModal** (`apps/web/src/pages/Projects/sections/BulkRejectModal.tsx`):

## 风险与规避

1. **Router 路径冲突**：`/bulk-approve`, `/bulk-reject` 必须注册在 `/{batch_id}/admin-lock` 之前，避免 FastAPI 把 "bulk-approve" 当作 batch_id UUID 解析。现有代码已遵循此模式。
2. **Scheduler 回归**：Phase 1 测试覆盖确保 Phase 2 变更不破坏现有自动推进逻辑。
3. **Bulk reject 性能**：per-batch `UPDATE tasks` 在 200 批次 × 1000 tasks 场景有压力。v0.9.15 可接受；如遇性能问题后续合并为单条 UPDATE。
4. **前端兼容**：`admin_locked` 默认 `false`，旧缓存 `BatchResponse` 不会 break。
