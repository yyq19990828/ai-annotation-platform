## Context

v0.9.5 (Async Oasis) 主线（`/ai-pre` 文本批量预标 UI + 类别 alias + Batch `pre_annotated` 状态机 + ADR-0012/0013）已收尾。真用起来后暴露三类非长尾欠账：

1. **工作台工具栏可用性**：v0.9.5 phase 4 收口图标语义（P2-a），但 38×38 主工具按钮仍仅 native `title` 提示、无 hotkey 角标、无分组分隔、SAM 子工具栏垂直挤主按钮下方易混淆主子层级 → ROADMAP §C.2 P2-b 主体
2. **AI 预标管理员闭环缺失**：admin 跑完 `/ai-pre` 后只有"已跑完"纯文本，无"打开标注工作台"接管按钮、无历史 job 列表、无失败重试入口；ML backend 注册要先存才能 health check（手敲 `http://172.17.0.1:8001` 摩擦大）；RegisteredBackendsTab 只显示 state/last_checked，backend `/health` v0.9.5 已扩 `gpu_info`+`cache`+`model_version` 但前端未消费
3. **截图自动化欠账**：v0.8.7 落地 14 个场景配置但 0 张 PNG；v0.9.5 又新增 4 个未配置场景（ai-pre / cost / alias-chips / pre-annotated 队列卡）

v0.9.6 一次性清这三类欠账，让 v0.10.x sam3-backend 接入时 UI/UX 基础线齐全，不再被工具栏 P2-b / 截图实跑 / SAM 注册体验等同期分心。

**不在本版本范围**（明确推迟）：
- mask→polygon 多连通域/空洞支持（P2 长尾，v0.9.4 phase 3 评测 IoU≥0.95 占比 92.2% 接近验收线但未达 → 留 v0.10.x 与 sam3-backend 共做，复用同一 mask_utils）
- ML backend storage endpoint 生产化策略（ADR-0012 已写决策框架；首次生产部署遇到再扩策略表）
- AIPreAnnotatePage 类别 alias chips 频率排序（需后端 GROUP BY predictions 聚合，留 v0.9.7 chip）
- 真实 SAM mask 50 张 simplify tolerance 验收（已跑 84 张真实 + 6 合成 = 90 样本，结论"接近验收线、保持默认值不动"已写入 `docs/research/13-simplify-tolerance-eval.md`，**v0.9.6 不必再跑**）

    @classmethod
    def _normalize_alias(cls, v: str | None) -> str | None:
        if not v or not isinstance(v, str): return v
        # lower + trim + 折叠多重空格 + 折叠多重逗号
        s = re.sub(r'\s+', ' ', v.lower().strip())
