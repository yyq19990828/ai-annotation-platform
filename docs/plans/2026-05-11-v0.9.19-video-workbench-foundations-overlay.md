# v0.9.19 视频工作台基础设施 + 悬浮时间轴

> 日期：2026-05-11
> 范围来源：`ROADMAP/2026-05-11-video-workbench-vs-image-gap.md` O2/O3 + `ROADMAP.md` C.3 视频时间轴悬浮化

## 1. 目标

把 `video-track` 工作台从可用 MVP 推到更可靠的日常标注基线，并收掉底部时间轴占用画布高度的问题。

成功标准：

1. 视频关键帧新增、修改、消失/遮挡标记支持 keyframe 级撤销/重做。
2. 视频创建、更新、重命名在网络断开或 5xx 时进入现有离线队列，恢复连接后可重放。
3. 视频更新遇到后端版本冲突时能打开现有 `ConflictModal`，重载路径可恢复。
4. `VideoStage` 底部固定 playback bar 改为画布内悬浮 overlay，保留播放、逐帧、scrubber、关键帧 tick 和帧信息。
5. 新增 `,` / `.` 作为视频上一帧 / 下一帧快捷键，并同步帮助面板。
6. 视频数据集文件列表展示媒体信息，并把图片专属“回填维度”替换为视频“补生成元数据”。
7. 已关联项目的数据集追加上传 / ZIP 导入 / 扫描导入后，新增文件立即同步为项目任务，工作台无需解绑重连。
8. 选中轨迹跳到无框帧时显示最近关键帧参考框，并支持拖动或复制成当前帧关键帧。
9. 文档、CHANGELOG、Roadmap 同步更新。

## 2. 不做范围

- 不改后端 schema、数据库迁移或 annotation API wire shape。
- 不做视频 keyframe diff UI；冲突弹窗第一版复用现有通用文案。
- 不做多选、复制粘贴、属性面板、视频 AI tracker、SAM video、Minimap 或视频 viewport 缩放。
- 不合并 `ImageStage` 和 `VideoStage`。
- 不改变数据集与项目的关联模型；只补齐关联后追加文件的任务同步。

## 3. 前端实现

### 3.1 Keyframe-aware history

- 扩展 `useAnnotationHistory` 命令类型，新增 `videoKeyframe`：
  - `annotationId`
  - `frameIndex`
  - `before: VideoTrackKeyframe | null`
  - `after: VideoTrackKeyframe | null`
- `undo` 使用 `before`，`redo` 使用 `after`。
- 应用命令时读取当前 `video_track` geometry，仅替换目标 `frame_index` 的 keyframe，保留同一 track 的其它 keyframes。
- 如果 `after` 或 `before` 为 `null`，表示删除目标帧 keyframe。
- 创建/删除整条 track、改类别仍使用现有 `create` / `delete` / `update` 命令。

### 3.2 Video mutation fallback

- `handleVideoCreate` 增加 `onError -> enqueueOnError(... optimisticEnqueueCreate(payload))`。
- `handleVideoUpdate` 增加网络/5xx fallback：
  - 乐观写入 annotations cache。
  - push `videoKeyframe` 或整条 `update` history。
  - enqueue `update` op。
- `handleVideoRename` 增加网络/5xx fallback：
  - 乐观写入 `class_name`。
  - push 现有 `update` history。
  - enqueue `update` op。
- 409 冲突不进 offline queue，继续由 `useUpdateAnnotation` 的 conflict callback 打开 `ConflictModal`。

### 3.3 Floating playback overlay

- 新增 `VideoPlaybackOverlay` 子组件，接收：
  - `frameIndex`
  - `maxFrame`
  - `fps`
  - `isPlaying`
  - `annotatedFrames`
  - `currentFrameEntryCount`
  - `visible`
  - `highlightAction`
  - `onSeek`
  - `onSeekByFrames`
  - `onTogglePlay`
- 删除 `VideoStage` 底部固定 grid bar，把 overlay 绝对定位到视频画布底部。
- stage hover 时显示；鼠标离开 2 秒后淡出；scrubber focus/drag 时保持显示；绘制或拖动 bbox 时隐藏。
- scrubber hover tooltip 显示目标帧号和时间。

### 3.4 Empty-frame selected-track affordance

- 当 `selectedTrack` 在当前帧没有 exact / interpolated bbox，且当前帧不是 `absent`，渲染最近非 `absent` 关键帧的虚线参考框。
- 参考框可拖动；拖动结束后复用 `upsertKeyframe(selectedTrack.geometry, frameIndex, geom)` 创建当前帧关键帧。
- 右侧当前轨迹面板新增「复制到当前帧」，用参考框 bbox 直接创建当前帧关键帧，方便用户先复制再微调。

### 3.5 Hotkeys

- `hotkeys.ts` video mode 下新增：
  - `,` -> `videoSeek(-1)`
  - `.` -> `videoSeek(1)`
- `HOTKEYS` 增加对应视频帮助条目。
- `VideoStageControls` 扩展为支持可选 action hint，使键盘触发后 overlay 按钮短暂高亮。

## 4. 数据集追加文件同步

- 新增 `DatasetService.create_tasks_for_items(dataset_id, item_ids)`：
  - 查找当前 dataset 已关联的所有 project。
  - 对每个 `(project_id, dataset_item_id)` 幂等创建 `Task`，跳过已存在任务。
  - 新任务保持 `batch_id = NULL`，沿用未归类任务语义。
  - 同步累加对应 `Project.total_tasks`。
- 在 `upload_complete`、`upload_zip`、`scan_items` 事务内调用该方法，确保新增 item 与任务扇出一起提交。
- 前端上传 / 扫描成功后失效 `datasets`、`dataset-items`、`tasks`、`projects`、`project-stats` 查询。
