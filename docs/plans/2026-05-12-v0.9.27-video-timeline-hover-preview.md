# v0.9.27 · 视频时间轴 Hover 缩略图预览

> 日期：2026-05-12
> 范围来源：`ROADMAP/2026-05-12-video-workbench-rendering-optimization.md` R17.1 + R5.1
> 前置依赖：v0.9.25 后端帧服务已提供 task 级 `frames/{frame_index}` 与 `frames:prefetch` 接口；v0.9.26 已提供 loop region、bookmark 和显式 seek history。

## 1. 背景与边界

v0.9.24-v0.9.26 已把视频时间轴从普通 seekbar 升级为轨迹 keyframe / outside / prediction / density / loop / bookmark 的轻量导航面板。当前缺口是：用户 hover 或拖动时间轴时仍只能看到帧号和时间，无法预判目标画面；跨段 seek 仍需要真实跳转后才知道画面内容。

后端 v0.9.25 已完成单帧缓存服务，因此本版不再新增后端能力，只在前端消费既有接口，把“帧作为资源”的第一层收益暴露到工作台。

## 2. 目标

1. 时间轴 hover 时显示缩略图 popover，包含 frame index、格式化时间和当前帧预览图。
2. hover 请求按 frame 节流与去重，避免鼠标移动时打爆 `frames/{frame_index}`。
3. 缓存状态明确：
   - `ready`：显示 signed URL 图片。
   - `pending`：显示轻量 loading，并在短延迟后重试一次。
   - `error`：显示 frame/time fallback，不阻断时间轴交互。
4. 对高价值帧主动 `frames:prefetch`：
   - 当前选中 track 的 keyframes。
   - 当前 task 的 bookmark frames。
   - loop region 的 start/end frame。
5. 预览 UI 不改变当前 seek 语义：hover 只看图，不移动当前帧；拖动 range 仍按现有 `onSeek` 行为。
6. 文档同步：
   - ROADMAP 标记 v0.9.27 切片。
   - `docs-site/dev/concepts/video-annotation-workbench.md` 增加 frame cache / hover preview 行为。
   - `CHANGELOG.md` 增加 0.9.27 条目。

## 3. 不做范围

- 不做 chapter 数据模型和时间轴章节色带（R13 / R17.2）。
- 不做已下载范围可视化（R17.3），因为前端尚未接 ImageBitmap / chunk 命中状态。
- 不做 ImageBitmap 背景缓存（R5.2）。
- 不接 WebCodecs / chunk 解码（R5.3）。
- 不做 J/K/L 多速率播放；留给 v0.9.28 与 `seekFrameAsync` 一起处理。
- 不把书签注释或侧栏列表纳入本版。

## 4. 实现方案

### 4.1 API 客户端

在前端 API 层补 task video frame helpers：

```ts
getVideoFrame(taskId, frameIndex, { format = "webp", width = 320 })
prefetchVideoFrames(taskId, frameIndices, { format = "webp", width = 320 })
```

接口沿用后端 v0.9.25：

```http
GET /api/v1/tasks/{task_id}/video/frames/{frame_index}?format=webp&w=320
POST /api/v1/tasks/{task_id}/video/frames:prefetch
```

注意点：

- `202` / pending 不能当作错误 toast；只进入 preview pending 状态。
- signed URL 过期由下一次 hover 重新拉取，不做长期持久化。
- frame index 先 clamp 到 `[0, maxFrame]`。

### 4.2 `useVideoFramePreview` hook

新增 `apps/web/src/pages/Workbench/stage/useVideoFramePreview.ts`：

- 输入：`taskId`, `maxFrame`, `enabled`, `width`, `format`。
- 暴露：
  - `previewFor(frameIndex)`：请求或返回缓存状态。
  - `prefetch(frames)`：去重后调用 `frames:prefetch`。
  - `clear()`：task 切换时清理内存缓存。
- 内存缓存 key：`${taskId}:${frameIndex}:${width}:${format}`。
- 状态上限：最多保留 120 个 preview 结果，避免长视频 scrub 后无限增长。
- 重试策略：pending 后 800ms 重试一次；仍 pending 就保留 pending，下一次 hover 再触发。

### 4.3 Overlay UI

改造 `VideoPlaybackOverlay`：

- `hoverFrame` 仍由 pointer position 推导。
- 新增 `hoverPreview` prop，或传入 `renderHoverPreview(frame)` callback，避免 overlay 内直接知道 API。
- 在 range 上方显示 popover：
  - 宽度固定 180-220px，图片使用 16:9 容器。
  - ready 状态显示 `<img>`。
  - pending 状态显示骨架和 `F {frame}`。
  - error 状态显示 frame/time 文案。
- popover 位置根据 frame 百分比 clamp 到容器内，不溢出左右边界。
- 现有 keyframe / loop / bookmark marker 不改变层级，popover 不阻挡点击。

### 4.4 Prefetch 触发

在 `VideoStage` 中统一触发预取：

- 选中 track 变化时，取 `sortedKeyframes(selectedTrack.geometry)` 的 frame index。
- bookmarks 变化时预取 bookmark frames。
- loopRegion 变化时预取 `startFrame` / `endFrame`。
- 每次触发最多提交 50 帧，超出时优先当前选中 track 附近帧。

### 4.5 失败与降级

- 如果 task 没有 manifest 或 task id，hover preview 完全禁用，只显示现有 frame/time tooltip。
- 如果 API 404/400，认为当前视频不支持 frame service，不再反复请求同一 task。
- 如果网络失败，当前 frame 记 error，不影响 seek/playback。
