# v0.9.4 Phase 2 — SAM 子工具栏拆分 + text 模式 box/mask/both 输出选择

> 计划文件命名遵循项目 CLAUDE.md 约定（`yyyy-mm-dd-vx.y.z` 前缀）。harness 默认建议名 `roadmap-md-0-9-3-phase-happy-meadow.md` 已弃用。

## Context

v0.9.2 落地 SAM 工具后两个 UX 痛点 v0.9.4 phase 1 真接通后立刻暴露：

1. **`S` 工具点击 / 拖动隐式分流 prompt 类型**：`SamTool.ts:24` 单一 `samProbe` DragInit，`ImageStage.tsx:546-558` 按拖动距离 < 0.5% 短边切 `point` vs `bbox`；新人不可见 Alt+click negative point；文本 prompt 走 `AIInspectorPanel.tsx:216-221` 的 `<SamTextPanel>` 完全脱节。
2. **text 模式永远输出 polygon**：`predictor.py:177-244` `predict_text()` 走 `DINO → SAM image embedding → set_image → predict → mask → polygon` 全链路，对 image-det 项目（标注员要 bbox）反而是负担，SAM 步骤 GPU 时间 200-500ms，box 模式跳过 set_image / cache.put / sam.predict / mask_to_polygon 能省 50-80% 推理时间。

本 phase 同时落两件事，**共用 T 文本子工具的 UI 改动**（segmented control 加在 `<SamTextPanel>` 顶部）。

ROADMAP 详见 `ROADMAP/[archived]0.9.x.md` v0.9.4 phase 2 段。

### 1. 数据模型 · 父工具 + 子态（不扩 Tool 联合）

**改动**：`apps/web/src/pages/Workbench/state/useWorkbenchState.ts`

```ts
// 保留 Tool = "box" | "hand" | "polygon" | "canvas" | "sam" 不变

// useWorkbenchState 内
```

### 9. ImageStage 候选叠加层按 type 分发

**改动**：`apps/web/src/pages/Workbench/stage/ImageStage.tsx:932-948`（候选渲染段）

```tsx
// 当前: candidates.map(c => <Line points={...} stroke="#a855f7" dash={...} />)

candidates.map((c, i) => {
```

### 11. 后端 · `predict_text` 三分支

**改动**：`apps/grounded-sam2-backend/predictor.py:177-244` `predict_text()` 拆为：

```python
def predict_text(self, np_img, text, *, output="mask", box_threshold=0.35, text_threshold=0.25, cache_key=None):
    # 1. DINO → boxes (xyxy 像素)（保留现有 L188-211）
    boxes_xyxy = self._dino_to_boxes(np_img, text, box_threshold, text_threshold)
    if not boxes_xyxy: return []

    # 2. 按 output 分支
    h, w = np_img.shape[:2]
    if output == "box":
        # 跳过 set_image / cache.put / sam.predict / mask_to_polygon
        return [self._box_to_rect_label(b, w, h) for b in boxes_xyxy]

    # 3. mask / both 共享 SAM image embedding 路径（保留 L216-225）
    self._restore_or_set_image(np_img, cache_key)
    masks = self._sam_predictor.predict(box=np.array(boxes_xyxy))
    polygons = [self._mask_to_polygon(m, w, h) for m in masks]

    if output == "mask":
        return [self._poly_to_polygon_label(p) for p in polygons]
    if output == "both":
        # 配对返回, 同 instance 一对 (rect, poly)
        return [
            label
            for box, poly in zip(boxes_xyxy, polygons)
            for label in (self._box_to_rect_label(box, w, h), self._poly_to_polygon_label(poly))
        ]
```

新增 helper：`_box_to_rect_label(box_xyxy, w, h)` 归一化为 `{type: "rectanglelabels", value: {x, y, width, height, rectanglelabels: [...]}, score: float}`；`_poly_to_polygon_label(poly)` 提取自现有 polygon 简化路径。

调用方 `_run_prompt()`（`predictor.py:131` 附近）text 分支把 `output` 参数从 `ctx.output` 透传。

## 验证方案

### 端到端冒烟（按本 phase 13 节顺序对应）

**子工具栏 + 热键（§1-§5）**：
1. 起栈，进 image-det 项目工作台
2. 按 `S` → ToolDock 工具按下 SAM + 子工具栏弹出 + 默认子工具 `point` active
3. 单击画布 → 紫虚线 polygon 候选（v0.9.2 行为，保留）
4. 再按 `S` → 子工具切到 `bbox`，单击画布无反应 / 拖框出 polygon 候选
5. 按 `+/-` → 仅 point 子工具下生效，子按钮 `[+] / [-]` 跟随切换；切到 bbox 时 +/- 不响应
6. 再按 `S` → 子工具切到 `text`，画布点击不响应；右栏 SamTextPanel 输入框获焦
7. 再按 `S` → 退出 SAM，回到上一个工具（box / polygon 等）

**text 三模式 + smart default + sessionStorage（§6-§7）**：
8. image-det 项目首次进 → SamTextPanel TabRow 默认 `[□ 框]` 高亮
9. 输文本 `person` 回车 → 紫虚线**矩形**候选（不是 polygon），Enter 接受 → 创建 BboxAnnotation
10. 切到 `[○ 掩膜]` → 同文本 → polygon 候选（验证 sessionStorage 写入）
11. 切到 `[⊕ 全部]` → 同文本 → 同 instance 同时有 box + polygon，`Tab` 切活跃几何（紫色描边粗细变化），Enter 接受当前活跃形态
12. 关掉浏览器重开同 project → SamTextPanel 默认从 sessionStorage 读，仍为 `both`（最后一次选择）
13. 切 image-seg 项目 → 默认 `[○ 掩膜]`（type_key 不同）

**后端三分支性能（§10-§11）**：
14. 用 `time curl -X POST /predict -d '{"task":..., "context":{"type":"text","text":"person","output":"box"}}'`：
    - `output=box`：~50-100ms（4060 / tiny）
    - `output=mask`：~200-500ms
    - `output=both`：~250-550ms（box 0 成本，复用 SAM mask 计算）
15. `curl /cache/stats` 验证：连续跑 box 模式不写入 cache（hits/misses 不动）；mask / both 模式正常写入

## 实施顺序建议

按风险递减、依赖清晰排：

1. **后端先行（§10-§11 + §13 后端测试）**：协议字段 + predict_text 三分支 + 单测；改完 backend 镜像 rebuild 一次（注意 vendor 不变不会 reload，要重启 container）
2. **PendingCandidate 加 type discriminator + ImageStage 候选渲染（§8-§9）**：纯前端类型扩展，不破老行为
3. **`useInteractiveAI` 加 outputMode 参数（§7）+ SamTextPanel TabRow（§6）+ samTextOutput.ts**：UI 层接通
4. **数据模型 + Tool 拆分 + ToolDock 子工具栏（§1-§3）**：重构 SamTool，注意 ImageStage:546-558 距离分流删除
5. **`S` 循环 + `+/-` polarity（§4-§5）**：hotkey 收尾
6. **截图 + 文档（§12 + §14）**：最后批量收口
7. **CHANGELOG / ROADMAP 标已落地**：commit message 引本 plan

按此顺序，前 3 步完成时已经能"text 三模式"端到端跑通（旧 SAM 工具行为不变），后 3 步是 UX 包装。可以中段提早 commit 第一个 batch 验通。
