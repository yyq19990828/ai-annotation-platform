# v0.10.2 — Prompt-first ToolDock + Exemplar 入口

> Roadmap: [ROADMAP/0.10.x.md](../../ROADMAP/0.10.x.md) §4 · v0.10.2
> 前置：v0.10.1 已落地 `useMLCapabilities` + `/setup` JSON Schema 协商。

---

## Context

v0.10.1 已把后端能力以 JSON Schema 形式吐出来（`/setup.supported_prompts` / `params`），前端 `useMLCapabilities` hook 已能拉到这份数据，但工作台 ToolDock **还没消费它**：

- 当前 `SamTool` 是单一工具 + `SamSubToolbar`（point / bbox / text 三按钮 + polarity 切换），用户不知道当前 backend 支不支持该 sub-tool —— SAM 3 项目里点 "point" 会发请求后端 400。
- 没有 exemplar 入口。SAM 3 后端的 `predict_exemplar` 通路 v0.10.0 已经端到端跑通（协议、API proxy、predictor 全部就绪），缺的只是前端工具 + Shift+拖框手势。
- 后端参数（`box_threshold` / `text_threshold` / `model_variant` 等）目前是写死/项目级配置，没法在工作台动态调。

本切片目标：
1. **拆 SamTool**：拆成 4 个独立工具 `SmartPoint` / `SmartBox` / `TextPrompt` / `Exemplar`，每个声明 `requiredPrompt`。
2. **能力联动置灰**：基于 `useMLCapabilities.isPromptSupported(t.requiredPrompt)` 决定工具按钮置灰 + tooltip。
3. **AIToolDrawer 替代 SamSubToolbar**：AI 工具激活时右侧固定抽屉，含后端选择器 + Schema-form 参数面板 + 工具特定输入（如 Text prompt 的输入框、Exemplar 的提示）。
4. **Schema-form 自研最小实现**：~150 行，支持 number / boolean / enum / string，从 `paramsSchema.properties` 自动渲染。
5. **Exemplar 入口**：新增工具 + `runExemplar` dispatcher；与 `SmartBox` 用同样的拖框手势但走 `type=exemplar`。
6. **E2E 覆盖**：annotation.spec 新增 Prompt-first 切换 + Exemplar 路径。

## 设计要点

### 1. ToolId 扩展 & 工具拆分

[apps/web/src/pages/Workbench/stage/tools/index.ts](../../apps/web/src/pages/Workbench/stage/tools/index.ts)：

```ts
// 旧：'box' | 'hand' | 'polygon' | 'canvas' | 'sam'
export type ToolId = 'box' | 'hand' | 'polygon' | 'canvas'
  | 'smart-point' | 'smart-box' | 'text-prompt' | 'exemplar';

export interface CanvasTool extends ToolMeta {
  id: ToolId;
  hotkey: string;
  label: string;
  icon: string;
  cursor: 'crosshair' | 'grab' | 'default';
  /** 新增：能力协商 key，缺省=非 AI 工具（永远启用） */
  requiredPrompt?: 'point' | 'bbox' | 'text' | 'exemplar';
  onPointerDown?: (ctx: ToolPointerContext) => DragInit | null;
}
```

新建 4 个文件，逻辑直接搬自 `SamTool.ts` 现有 `samSubTool` 分支：

- `SmartPointTool.ts` —— `requiredPrompt: 'point'`，复制原 `mode === 'point'` 分支（含 Alt 反向）。极性切换从全局状态移到本工具自身（保留 `samPolarity` 字段但作用域限定）。
- `SmartBoxTool.ts` —— `requiredPrompt: 'bbox'`，复制原 `mode === 'bbox'` 分支。
- `TextPromptTool.ts` —— `requiredPrompt: 'text'`，`onPointerDown` 返回 `null`（不响应画布）；交互在 AIToolDrawer 内的文本输入完成。
- `ExemplarTool.ts` —— `requiredPrompt: 'exemplar'`，行为=同 `SmartBox` 拖框但 dispatch 时走 `runExemplar`。**沿用普通拖框手势**（用户激活该工具后画框即调用 exemplar；不强求 Shift 修饰键）。

`SamTool.ts` 删除；`TOOL_REGISTRY` 移除 `sam`。

### 2. ToolDock 置灰 + 工具组分隔

[apps/web/src/pages/Workbench/shell/ToolDock.tsx](../../apps/web/src/pages/Workbench/shell/ToolDock.tsx) 改造：

```tsx
const { isPromptSupported, isLoading } = useMLCapabilities(projectId, mlBackendId);

const isDisabled = (t: CanvasTool) =>
  t.requiredPrompt ? !isLoading && !isPromptSupported(t.requiredPrompt) : false;
```

- 删除原 `isSamActive && <SamSubToolbar/>` 内联块。
- 不支持的 AI 工具按钮：`disabled` + opacity 0.4 + tooltip "当前后端不支持此交互模式"；点击不切换。
- ToolDock 不再持有 `samSubTool` / `samPolarity` props。

按钮分组顺序：`box, polygon`（分隔线）→ `smart-point, smart-box, text-prompt, exemplar`（分隔线）→ `hand`。

### 3. AIToolDrawer（新组件）

新建 [apps/web/src/pages/Workbench/shell/AIToolDrawer.tsx](../../apps/web/src/pages/Workbench/shell/AIToolDrawer.tsx)：

- **挂载条件**：`tool` 是 4 个 AI 工具之一 → 渲染右侧固定抽屉（宽 280px，绝对定位，紧贴右侧 LabelSidebar 左边）。
- **结构**：
  ```
  ┌────────────────────────────────┐
  │ <ToolLabel> · ⚙              │
  │ ──────────────────────────── │
  │ 后端: [grounded-sam2 ▼]      │   ← 单一项时仍渲染（稳定 UI 形态）
  │ ──────────────────────────── │
  │ <ToolSpecificControls>       │   ← Text: 输入框；Point: 极性 +/−；Box/Exemplar: 提示文案
  │ ──────────────────────────── │
  │ 参数面板（来自 paramsSchema）│
  │   置信度阈值: [0.50 ──●────] │
  │   ☐ 转为多边形                │
  │ ──────────────────────────── │
  │ 状态: ● Healthy              │
  └────────────────────────────────┘
  ```
- 后端选择器：从 `currentProject.ml_backend_id` 取唯一一项，控件保留（disabled 单选 dropdown）。
- 参数面板调 `<SchemaForm schema={paramsSchema} value={params} onChange={setParams} />`，结果通过 props 透传给 `useInteractiveAI` 注入到 context。

### 4. SchemaForm 最小自研实现

新建 [apps/web/src/pages/Workbench/components/SchemaForm/index.tsx](../../apps/web/src/pages/Workbench/components/SchemaForm/index.tsx) 约 ~150 行：

```ts
interface Props {
  schema: { type: 'object'; properties: Record<string, JsonSchemaField> };
  value: Record<string, unknown>;
  onChange: (next: Record<string, unknown>) => void;
}
```

支持类型：
- `number` + `minimum/maximum/default` → slider + 数值输入
- `boolean` + `default` → 复选框
- `string` + `enum` → 下拉选择
- `string` (无 enum) → 文本输入
- 顶层无 `properties` 或缺字段时渲染空白（兜底）

不引入 `@rjsf/core`（依赖 ~50KB gzipped + AJV，超出本期需求）。后续如需 array / nested object 再扩展。

### 5. 状态迁移

### 6. useInteractiveAI 扩展

[apps/web/src/pages/Workbench/state/useInteractiveAI.ts](../../apps/web/src/pages/Workbench/state/useInteractiveAI.ts)：

interface UseInteractiveAIReturn {

## 风险 & 兜底

| 风险 | 缓解 |
|---|---|
| SchemaForm 自研覆盖不全 | 先支持 number/boolean/enum/string 四种；遇到 array/nested 再扩 |
| 拆分后 hotkey `S` 行为变化让老用户困惑 | 文档明示 `S` 现在循环 4 个 AI 工具（跳过置灰） |
| `useMLCapabilities` 加载期间所有 AI 工具显示为「可用」误导用户 | 加载期间整组按钮 `aria-busy="true"` + 半透明，点击 no-op |
| Exemplar 与 SmartBox 手势相同，用户分不清差别 | 工具按钮 tooltip + Drawer 顶部明确标注「画框寻找全图相似实例」 |
