# v0.9.33 · Video Frame Asset Retry

## 背景

v0.9.25 已落地后端帧服务，v0.9.27 / v0.9.29 / v0.9.31 已分别完成 hover 缩略图、J/K/L 播放和视频观测包。当前视频 probe、poster、frame timetable、chunk / frame cache 已有失败状态，但管理侧缺少统一入口，失败后只能看数据库或日志。

本版收口 `ROADMAP/2026-05-12-video-workbench-rendering-optimization.md` 的候选切片 **Probe / Poster / Frame Asset Retry** 第一版。

## 范围

1. 后端提供视频资产失败列表。
   - 汇总 `probe_error` / `poster_error` / `frame_timetable_error`。
   - 汇总 chunk / frame cache 最近失败状态。
   - 返回 task、dataset item、project、失败类型、错误信息和更新时间。
2. 后端提供手动 retry 入口。
   - `probe` / `poster` / `frame_timetable` 重试复用 media metadata task。
   - `chunk` / `frame` 重试复用现有 frame asset Celery task。
   - 返回 `202 Accepted`，不在请求线程里跑 ffmpeg。
3. 管理侧增加可见入口。
   - 在系统监控或模型市场相近位置展示视频资产失败列表。
   - 支持按失败类型展示状态、错误摘要和重试按钮。
4. 同步文档。
   - `CHANGELOG.md`。
   - roadmap 状态。
   - docs-site 视频工作台 / frame service 文档。

## 不做

- 不引入 WebCodecs / ImageBitmap 背景缓存。
- 不新增独立 media service。
- 不做失败资产自动重试策略；本版只提供人工入口。
- 不改变视频上传、probe、poster 的既有成功路径。
- 不做 chunk 解码前端 UI。
