# v0.9.3 Phase 2 — Merged Market

> 状态：Plan 阶段 · 2026-05-07
> 范围：把两个孤儿管理页（`/admin/ml-integrations`、`/admin/failed-predictions`）合并到侧边栏占位页 `/model-market`；删除三页内容重复与导航孤儿
> SAM 主线（M3 mask→polygon）仍待 GPU 机；本 phase 与 phase 1 同样不依赖 backend 实跑

---

## Context

侧边栏「智能」组下 `/model-market` 自始是 `PlaceholderPage`，无内容。同时 ML 相关已落地的两个页面是孤儿：

| 路由 | 入口 | 与现有页面重复内容 |
|---|---|---|
| `/admin/ml-integrations` | 仅 `AdminDashboard.tsx:424` 一个按钮 | StatCard + Bucket 表 100% 与 `/storage` 重复 |
| `/admin/failed-predictions` | 仅 `AdminDashboard.tsx:147` 一张卡片 | 完整分页页面，但失败条目通常 < 10，单独路由超规格 |

`/storage` 自身已是合格的存储管理页（Bucket + 数据集），不应被 ml-integrations 重复。「模型市场」原计划等 v0.10.x sam3-backend 落地后才开工，但当前已有现成 `MLIntegrationsPage` 的项目级 ML Backend 表 + `FailedPredictionsPage` 的失败列表，这两块拼起来就是一个最小可行的「模型市场 v0」，正好替占位。

## Decisions

1. **`/storage` 不动**：它的角色权限广（admin / annotator / viewer 都能看），不适合塞 ML Backend 这类管理视角内容。
2. **`/admin/ml-integrations` 拆**：bucket / storage 部分本就在 `/storage`，删；项目级 ML Backend 部分搬到 `/model-market` 第一个 tab。
3. **`/admin/failed-predictions` 整体折成 `/model-market` 第二个 tab**。失败预测条数少时占位极小，与 backend 列表共页符合「ML 集成监控」语义。
4. **Tab 切换通过 URL `?tab=backends|failed`** 同步；AdminDashboard 跳转用 `?tab=failed` 直达。
5. **Catalog tab（v0.10.x）暂不做**，留 ModelMarketPage 内 TODO 注释占位。

## Scope · 页面与组件

```

  ModelMarketPage.tsx          # tabs 容器；URL 同步；权限同 model-market PageKey
  RegisteredBackendsTab.tsx    # 搬 MLIntegrationsPage 的 ProjectGroup 逻辑，去 storage 部分
  FailedPredictionsTab.tsx     # 搬 FailedPredictionsPage 主体（list/retry/dismiss/restore）
```

### 删除

- `apps/web/src/pages/Admin/MLIntegrationsPage.tsx`
- `apps/web/src/pages/Admin/FailedPredictionsPage.tsx`
- `apps/web/src/pages/Admin/__tests__/FailedPredictionsPage.test.tsx`（断言对象切换到 Tab 后另写或合并到现有 hook 测试中）
- `App.tsx` 路由 `/admin/ml-integrations` 与 `/admin/failed-predictions`
- `apps/web/src/constants/permissions.ts` `admin-ml-integrations` / `admin-failed-predictions` 两个 PageKey
- `apps/web/src/types/index.ts` 里 PageKey 联合的对应键

### 改造

- `App.tsx`：`<PlaceholderPage title="模型市场" />` → `<ModelMarketPage />`
- `Dashboard/AdminDashboard.tsx:147` 失败预测卡片 → `navigate("/model-market?tab=failed")`
- `Dashboard/AdminDashboard.tsx:424` ML 集成按钮 → `navigate("/model-market")`
- `components/shell/Sidebar.tsx:36` 「3 运行中」mock 徽章 → 改为读 `useFailedPredictions(1, 1, false).data?.total`，> 0 时以 `danger` 变体显示「N 失败」，否则不显示徽章

### API 不动

- `adminMlIntegrationsApi.overview()` 后端不动；前端只读 `projects` / `total_backends` / `connected_backends`，不再渲染 `storage` 字段
- `useFailedPredictions` / `useRetry…` / `useDismiss…` / `useRestore…` hook 全部沿用

## Permissions

- 删两个 PageKey 后，`super_admin` / `project_admin` 列表对应键移除
- `model-market` PageKey 已在两角色列表中存在，无需再加
- RegisteredBackendsTab 全局只读，沿用 `model-market` 页级权限即可
- FailedPredictionsTab 内的 retry/dismiss/restore 走后端权限校验，前端不再单独门禁

## Risk

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 失败预测 tab 内置入页面后宽度收窄，分页/操作列拥挤 | 中 | UX | 表格列改为响应式：窄屏隐藏 prediction.id 与 model_version 列 |
| URL 同步 `?tab=` 与 react-router v6 useSearchParams 在历史栈互动出 bug | 低 | 切换抖动 | 用 `replace: true`，避免每次切 tab 进入历史栈 |
| Sidebar 徽章绑真数据后，每次渲染都查一次 `/admin/failed-predictions` | 中 | 接口 QPS | 复用既有 useFailedPredictions hook 的 react-query 缓存（refetchInterval 已存在）；只查 `total`，pageSize=1 |
| 反向引用未清完（route 反链 / E2E） | 中 | 跳转 404 | 第 6 步 grep 全库 `admin/ml-integrations` `admin/failed-predictions` 字符串 |
