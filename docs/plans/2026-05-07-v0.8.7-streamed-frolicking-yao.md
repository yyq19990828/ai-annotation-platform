## Context

v0.7.7 已落开放注册基座 + 3/min rate-limit；production 放量前剩下两块短板：① 单 IP rate-limit 不防分布式刷号；② Celery / ML Backend 调用延迟仅在内存（`PredictionResult.inference_time_ms`），无法算 P95/P99 SLO。v0.8.5 已把 E2E 写实化到 bbox 拖框 + 多角色串联，但「reviewer 反馈环（reject 后 annotator 看到反馈）+ bbox 落库后画布刷新断言」留作续作；v0.8.0 在 6 类用户手册页面里留 16 个截图占位（PNG 1×1 透明）至今未填。标注工作台另有三处低工时高价值改进未落（resize 修饰符 / 任务跳过 / history 持久化）。

v0.8.7 把这 8 件可独立推进、互不依赖的事一次性收口，让 v0.9.x Grounded-SAM-2 主轴启动前的最后一个准备版彻底干净。

**用户决策（已确认）：**
- 范围：8 项全收（不拆 v0.8.7 + v0.8.8）
- CAPTCHA provider：**Cloudflare Turnstile**（免费 / privacy-friendly / 无图形挑战 / siteverify.api.cloudflare.com）
- 截图自动化：**Playwright 脚本 + 手动触发 + 提交 PNG 到 git**（不进 CI，避免 baseline drift）
- 任务跳过 UX：**Topbar「跳过」按钮 + Modal 选原因枚举**

## F8 · C.3 History 持久化（sessionStorage 5min TTL）

> 当前 `useAnnotationHistory` 命令栈纯内存，刷新即清；与 `replaceAnnotationId` 协同，prediction 临时 id 已稳定可序列化。

### 文件清单

| 路径 | 修改 |
|---|---|
| `apps/web/src/pages/Workbench/state/useAnnotationHistory.ts` | 加 `persist(taskId)` / `restore(taskId)` 内部方法；存 key `wb:hist:{taskId}`，包含 `{stack, cursor, ts}`；`ts > now - 5min` 才 restore；写时机：每次 push command 后 setTimeout 50ms throttle |
| `apps/web/src/pages/Workbench/state/__tests__/useAnnotationHistory.test.ts` | 加 4 例：persist round-trip；TTL 过期不 restore；taskId 切换清旧；redo 命令引用已替换 id 时安全跳过（与 `replaceAnnotationId` 协同） |

### 风险点

- redo 命令引用 `predictionId`，若 prediction 已被 `replaceAnnotationId` 替换，restore 后 redo 可能引用陈旧 id → 在 restore 时全量扫一遍 stack，对照当前 `annotations` map 跳过引用不到的命令（不抛错）
- sessionStorage quota（~5MB）：单 task 命令栈最大 200 条 × ~500B = ~100KB，单 tab 不会超
- 跨 tab：sessionStorage 本身就是 tab 隔离，无需额外处理

### 手测金线（golden path）

1. **CAPTCHA**：`TURNSTILE_ENABLED=true` 启 dev → 注册页加载 Turnstile widget → 不勾 widget 提交 → 后端 400 `captcha_failed`；勾选后正常注册成功

4. **Resize 修饰键**：选中 bbox → 拖右下角 + Shift → 锁定纵横比；Alt → 从中心扩展
5. **Reviewer mini panel**：reviewer 进 ReviewWorkbench → 右侧栏看到 3 个 mini-stat；通过 1 个任务后 20s 内数字 +1
6. **截图自动化**：`pnpm screenshots` 跑完 → `docs-site/user-guide/images/` 下 14 个 PNG 非空 + 内容符合 IMAGE_CHECKLIST 描述

## 风险 / 取舍

1. **F1 dev 体验**：`TURNSTILE_ENABLED=False` 缺省关闭，本地 `.env.example` 不强制；CI 不开 → 不影响 pytest / vitest / playwright。production 部署需在 nginx 层 / docker env 显式 set
2. **F2 Histogram 内存**：`ml_backend_request_duration_seconds` 标签基数 = backends × 2 outcomes，当前 ≤ 10 backend 内存可控；超过 50 backend 后需要降级标签
3. **F4 keypoint 两张占位**：keypoint 工作台未实装时无法截图；IMAGE_CHECKLIST 标 `[skip-v0.8.7]` 注释，等非 image-det 工作台版本一起补
4. **F8 sessionStorage TTL 5min**：选 5min 是与 prediction id 替换窗口对齐；如果 5min 内任务切换频繁，旧 task 命令栈会在 sessionStorage 留垃圾，taskId 切换时手动清旧
5. **范围 ~10.5d 偏满**：若 F4 截图调试遇到 flaky，优先把 14 张降到 10 张核心场景 + 4 张占位
