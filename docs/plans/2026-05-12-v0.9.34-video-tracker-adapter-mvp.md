# v0.9.34 · Video Tracker Adapter MVP

## 背景

`ROADMAP/2026-05-12-video-backend-frame-service.md` 中 S4 已在 v0.9.32 落地 `VideoTrackerJob` 编排壳：独立 job 表、创建 / 查询 / 取消 API、frame range 与 segment lock 校验。当前缺口是 job 创建后不会被 worker 消费，也没有统一 tracker adapter 契约或结果写回路径。

本版本实现 S5 的最小后端闭环：先用 `mock_bbox` adapter 跑通状态机、事件流和 `video_track` keyframe 写回。真实 SAM 2 / SAM 3、GPU 队列容量控制和长视频性能优化留到 S6。

## 假设

- `mock_bbox` 是 contract adapter，不代表最终模型质量；它复用输入 bbox 并逐帧输出，方便前端 R10 对接和 worker eager 测试。
- tracker 输出先写回源 annotation 的 `geometry.type="video_track"`，不新增 prediction 表结构。
- 低置信度 / outside 语义保留在 adapter result 契约里，本版只实现不覆盖 manual keyframe 的合并规则。
- Redis pub/sub 事件是 best effort，发布失败不能让 job 失败。

## 范围

1. 新增 tracker adapter registry：
   - `TrackerAdapter.propagate(ctx) -> Iterator[TrackerFrameResult]`
   - 内置 `mock_bbox`
   - 未知 `model_key` 报错并把 job 标记为 failed
2. 新增 tracker worker：
   - 消费 `queued` job，进入 `running`
   - 逐帧检查取消请求
   - 写回 prediction keyframes / outside ranges
   - 更新 `completed/failed/cancelled`
   - 发布 `job_started/frame_result/job_progress/job_completed/job_failed/job_cancelled`
3. 创建 job 后投递 Celery task，并保存 `celery_task_id`。
4. 补协议文档、runbook、changelog 和 roadmap 状态。

## 不做

- 不接 SAM 2 / SAM 3 / Cutie / DEVA。
- 不新增 GPU worker compose service。
- 不改前端 UI。
- 不改 `PredictionJob` / `Prediction` 表结构。
