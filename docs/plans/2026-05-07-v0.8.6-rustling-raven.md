### Task 实现要点

```python
# apps/api/app/workers/ml_health.py
@celery_app.task
def check_ml_backends_health() -> dict:
    return asyncio.run(_check_all())

async def _check_all():
    async with SessionLocal() as db:
        backends = (await db.execute(select(MLBackend))).scalars().all()
    results = []
    for b in backends:
        await asyncio.sleep(random.uniform(0, 3))  # 错峰抖动
        async with SessionLocal() as db:
            svc = MLBackendService(db)
            await svc.check_health(b.id)  # 复用既有 check_health 逻辑
            b_fresh = await db.get(MLBackend, b.id)
            b_fresh.last_checked_at = datetime.now(UTC)
            await db.commit()
            results.append({"id": str(b.id), "state": b_fresh.state})
    return {"checked": len(results), "results": results}
```

**抖动 0-3s（不是 0-5s）**：5s + 5s timeout = worst-case 50s 临近 60s 边界；3s 留 margin 防与下一轮启动重叠。

关键聚合 SQL：

```sql
-- 主聚合
SELECT
  COUNT(p.id) AS total,
  AVG(pm.inference_time_ms) AS avg_ms,
  COALESCE(SUM(pm.total_cost), 0) AS total_cost,
  COALESCE(SUM(pm.total_tokens), 0) AS total_tokens
FROM predictions p
LEFT JOIN prediction_metas pm ON pm.prediction_id = p.id
WHERE p.created_at >= NOW() - :range::interval;

-- failure_rate 单查
SELECT COUNT(*) FROM failed_predictions
WHERE created_at >= NOW() - :range::interval;

-- by_backend 维度
SELECT p.ml_backend_id, mb.name,
       COUNT(p.id),
       COALESCE(SUM(pm.total_cost), 0),
       AVG(pm.inference_time_ms)
FROM predictions p
LEFT JOIN prediction_metas pm ON pm.prediction_id = p.id
LEFT JOIN ml_backends mb ON mb.id = p.ml_backend_id
WHERE p.created_at >= NOW() - :range::interval
GROUP BY p.ml_backend_id, mb.name;
```

**索引**：`predictions.created_at` 当前无索引，30d 数据量级 < 10K 不需加；CHANGELOG 留 TODO「数据量上 10 万后给 `predictions.created_at` 加 BRIN 索引」。

**异常处理**：聚合查询全包 try/except 返回 0 而非 500（dashboard 不能因聚合挂掉黑屏）。

### 核心 API

```python
# apps/_shared/mask_utils/src/mask_utils/polygon.py
import cv2
import numpy as np
from shapely.geometry import Polygon

def mask_to_polygon(
    mask: np.ndarray,
    tolerance: float = 1.0,
    normalize_to: tuple[int, int] | None = None,
) -> list[list[float]]:
    """Convert binary mask (H, W) to a simplified polygon.

    Args:
        mask: uint8 / bool array, nonzero = foreground.
        tolerance: shapely.simplify tolerance（像素单位）。
        normalize_to: (W, H) 用于归一化坐标到 [0, 1]；None 返回像素坐标。

    Returns:
        [[x1, y1], ...]；多 contour 取面积最大者；空 mask 返回 []。
    """
    contours, _ = cv2.findContours(
        mask.astype("uint8"), cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_NONE
    )
    if not contours:
        return []
    biggest = max(contours, key=cv2.contourArea)
    points = biggest.reshape(-1, 2)
    poly = Polygon(points).simplify(tolerance, preserve_topology=True)
    coords = list(poly.exterior.coords)
    if normalize_to:
        W, H = normalize_to
        coords = [[x / W, y / H] for x, y in coords]
    return [list(c) for c in coords]
```

retry 路由：

```python
@router.post("/admin/failed-predictions/{failed_id}/retry", status_code=202)
async def retry_failed(failed_id, db, user=Depends(require_admin())):
    fp = await db.get(FailedPrediction, failed_id)
    if not fp:
        raise HTTPException(404)
    if fp.retry_count >= 3:
        raise HTTPException(409, "Max retries (3) exceeded")
    retry_failed_prediction.delay(str(failed_id), str(user.id))
    return {"status": "queued", "failed_id": failed_id}
```

Celery task 链路：
1. 读 `failed_predictions` 行 → 取 task_id + ml_backend_id
2. ws 推 `retry.started`
3. 调 `MLBackendClient.predict(...)` 重跑
4. 成功 → 插 `predictions` 行 + 删（或标记 resolved）`failed_predictions` + ws 推 `retry.succeeded`
5. 失败 → `retry_count += 1` + `last_retry_at` + ws 推 `retry.failed`

`max_retries=3` 是 service 层软上限（不是 Celery `task.max_retries`）—— 让管理员看到错误后主动重试三次仍败放弃。

## 风险 / 注意事项

| 风险 | 缓解 |
|---|---|
| ProjectSettings 改造涉及向导 + 通用面板两处，可能漏改 | E2E 必须覆盖：创建项目（绑 backend）→ 进设置看到绑定 → 解绑 → 重绑 |
| `ai_model` 与 backend 同时被填且不一致时显示哪个？ | 保存时 backend 优先：`ai_model = backend.name`（覆盖手填）+ UI hint「将以 backend 名称显示」 |
| Celery beat 进程未启动 → 健康检查全失效 | `docs-site/dev/deploy.md` 加 beat 启动校验；前端 ProjectSettings 显示 `last_checked_at > 5min` 时给橙色 banner「上次心跳超过 5 分钟，请检查 Celery beat 进程」 |
| `mask_utils` 拉 opencv-python-headless ~50MB | 本期不进 api 镜像；v0.9.x backend 镜像本来就有 cv2，可复用 |
| `failed_predictions.retry_count` 加列对老数据 | `server_default="0"` + `nullable=False`，迁移自动 backfill |
| WebSocket retry 进度推送如果 ws 断了 | 前端有 reconnect；列表 useFailedPredictions 同时也会被 ws 触发 invalidate，下次拉取拿到最新状态 |
