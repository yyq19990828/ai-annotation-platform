## Context

**为什么做**：v0.9.5 (Async Oasis) 把 `/ai-pre` 文本批量预标 UI 搭起来；v0.9.6 (Agile Kernighan) 把跑完闭环（CTA + 历史表 + 重试链接）+ alias chips 滚动 + ML backend 注册体验/健康聚合都收齐。但页面本身 UX 仍处在「能跑」而非「好用」阶段：

- **478 行单文件 inline style**：`labelStyle / selectStyle / aliasChipStyle` 散落、硬编码 `maxHeight: 96px / maxWidth: 180px` 等魔数、卡片头无 borderBottom 分隔，与 AdminDashboard / ProjectSettingsPage 等成熟页面的视觉成熟度有明显差距（`apps/web/src/pages/AIPreAnnotate/AIPreAnnotatePage.tsx:448-477`）。
- **无步骤引导**：4 张卡片纯纵向排列，admin 第一次进入不知道流程从哪一步开始；与 CreateProjectWizard 的 stepper 体验断层。
- **交互空白**：无 Ctrl+Enter 提交、无 prompt localStorage 草稿、空 alias 不提示、历史表无搜索/排序/分页（`limit=50` 硬编码）、切项目时 prompt 不重置可能误操作。
- **alias chips 排序粗糙**：当前按 `Object.entries(classes_config)` 的 key 顺序排，30+ 类别全配 alias 时高频类别埋在中间，需要 prediction 频率聚合让常用 alias 浮上来。
- **Wizard step 4 backend 绑定缺失**：v0.9.6 Known Gap，wizard 第 4 步「AI 接入」只能 toggle `ai_enabled`，新建项目仍是「AI 启用 + 未绑定」状态，需进入项目设置二次配置。
- **文档/截图欠账**：v0.9.6 标注的 `sam-tool.md` / `ai-preannotate.md` 两篇用户手册未提交；`apps/web/e2e/screenshots/scenes.ts` 18 个场景配置就位但 PNG 文件 0 张。

**目标**：把 AI 预标注页面提升到与平台其它成熟页面一致的视觉/交互水准，同时清掉 v0.9.6 4 项遗留，让 v0.10.x sam3-backend 主线接入时以一个 UX 完整、文档闭环、截图齐全的基线起步。

**不在本版本范围**（明确推迟）：
- mask→polygon 多连通域/空洞支持（P2 长尾，v0.10.x 与 sam3-backend 共做）
- ML backend storage endpoint 生产化策略（首次生产部署再扩 ADR-0012）
- 完整预标 job 历史追踪（含已结束/已重置批次的 prompt/cost/duration）— 需新建 `prediction_jobs` 表 + worker 写入，留 v0.10.x
- 成本预估 / 单张样例 dry-run preview（feature-dev 范畴，不属于 UX 打磨）

### C.1 后端：新增 alias frequency 端点

**新建** `apps/api/app/api/v1/admin_alias_freq.py`：

```python
@router.get(
    "/admin/projects/{project_id}/alias-frequency",
    response_model=AliasFrequencyResponse,
)
async def get_alias_frequency(
    project_id: uuid.UUID,
    db: AsyncSession = Depends(get_db),
    _admin: User = Depends(require_roles(UserRole.PROJECT_ADMIN, UserRole.SUPER_ADMIN)),
) -> AliasFrequencyResponse:
    """统计本项目所有 predictions 中各 detected label 的出现次数.

    实现：JOIN predictions × tasks WHERE project_id, 用 PG `jsonb_array_elements`
    展开 result JSONB, 提取 value→labels[0] 或 value→class, GROUP BY 计数.
    """
```

返回 schema：
```python
class AliasFrequencyResponse(BaseModel):
    project_id: uuid.UUID
    total_predictions: int
    frequency: dict[str, int]  # {"person": 142, "car": 38, ...}
    last_computed_at: datetime
```

**实现要点**：
- 用 SQL 直接 GROUP BY 比应用层 Python 循环快 10x（参考 `admin_preannotate.py:87-100` 的 GROUP BY 风格）
- 查询：
  ```sql
  SELECT
    jsonb_path_query(p.result, '$[*].value.labels[0]') #>> '{}' AS label,
    COUNT(*) AS cnt
  FROM predictions p
  JOIN tasks t ON t.id = p.task_id
  WHERE t.project_id = :project_id
  GROUP BY label
  ORDER BY cnt DESC
  LIMIT 200;
  ```
- 因 `result` 同时支持 `polygonlabels`（`value.labels[]`）和 `rectanglelabels`（`value.labels[]` 或 `value.class`），需要 fallback 二选一
- 加 5 分钟应用层 cache（用 `apps/api/app/cache.py` 现有 `cached(ttl=300)` 装饰器，参考 `admin_overview.py` 风格）

**单测**：`apps/api/tests/test_admin_alias_freq.py`
- `test_no_predictions_returns_empty`
- `test_polygon_labels_aggregated`
- `test_rectangle_labels_aggregated`

```ts
const freqQ = useQuery({
  queryKey: ["alias-freq", projectId],
  queryFn: () => aliasFrequencyApi.byProject(projectId),
  enabled: !!projectId,
  staleTime: 1000 * 60 * 5,
});

const aliases = useMemo(() => {
  const freq = freqQ.data?.frequency ?? {};
  return Object.entries(project?.classes_config ?? {})
    .map(([name, entry]) => ({
      name,
      alias: entry?.alias ?? null,
      count: freq[entry?.alias ?? ""] ?? 0,
    }))
    .filter((e): e is { name: string; alias: string; count: number } => !!e.alias)
    .sort((a, b) => b.count - a.count || a.alias.localeCompare(b.alias));
}, [project, freqQ.data]);
```

```markdown
- 按 S 键进入 SAM 模式
- 子工具：1=点 / 2=框 / 3=文本
- 切换 polarity：`+` / `-`
- Tab 切候选 / Enter 接受 / Esc 退出

## 三种 prompt 适用场景
- 点 prompt：精修小物体边缘
- 框 prompt：框定大致区域 + SAM 收紧
- 文本 prompt：未知形状对象（参见 [AI 预标注](../projects/ai-preannotate.md)）

## 速度对比表
| 模式 | DINO | SAM | 总耗时 (4060/tiny) |
|---|---|---|---|
| 点 | — | ~80ms | ~80ms |
| 框 | — | ~80ms | ~80ms |
| 文本 box | ~60ms | — | ~60ms |
| 文本 mask | ~60ms | ~80ms | ~140ms |

## 常见问题
- mask 边缘锯齿 → 调整 simplify tolerance（项目设置）
- 召回不准 → 检查 prompt 英文 alias 配置
- backend disconnected → 检查 ML Backend 注册
```

```markdown
# AI 文本批量预标

## 适用场景
- 批量素材冷启动：跑一遍 AI 后再人工 review

## 完整流程（5 步）
1. **类别 alias 配置**（前置准备）
2. **进入 `/ai-pre`**
3. **选项目 + 批次**（仅 active 状态可预标）
4. **输入 prompt + 选输出形态**
   - alias chips 按预标频率排序，常用类别在前
   - 输出形态：image-det 项目用「框」最快；image-seg 项目用「掩膜」
5. **跑预标 + 接管 review**

## 历史与重试
- 历史表显示 pre_annotated 状态批次
- 失败重试：跳到 [模型市场失败列表](../../user-guide/admin/model-market.md)

## 故障排查
- 「未绑定 ML Backend」→ 项目设置 → ML 模型 tab
- 「无 active 批次」→ 数据集页面把 batch 从 draft 转 active
- DINO 阈值：项目设置 → AI 接入 → box/text threshold
```

## 风险与缓解

| 风险 | 缓解 |
|---|---|
| `predictions.result` JSONB GROUP BY 在大表上慢 | 加 `cached(ttl=300)` + 监控 P95；超 500ms 时考虑加 `predictions.detected_label VARCHAR(100)` 索引列 |
| Wizard backend 复制时 auth_token 泄漏跨项目 | 复制后立即 `state="disconnected"` 强制重新 health check + 显式 toast 告知"已复制凭据，请在新项目验证可用性" |
| 拆子组件后 props drilling 太深 | 仅 1 层深度（外壳 → 子组件），不引入 context；超过则 v0.10.x 再用 zustand store |
| stepper anchor 滚动在小屏被 sticky topbar 遮挡 | `scrollIntoView` 加 `block: "start"` + CSS `scroll-margin-top: 64px` |
| 截图实跑环境不一致（不同 GPU / 字体） | 截图运行明确指定 docker compose profile + chromium pinned version |
