# v0.9.26 · 视频播放范围与跳转书签导航

> 日期：2026-05-12
> 范围来源：`ROADMAP/2026-05-12-video-workbench-rendering-optimization.md` R4.4 + R19
> 并行约束：`ROADMAP/2026-05-12-video-backend-frame-service.md` 正在以 v0.9.25 并行开发，本版不占用后端帧服务接口。

## 1. 背景与边界

v0.9.24 已完成 R4.1-R4.3：选中轨迹时间轴、全局密度条和可见 keyframe 跳转。R4 剩余的第一优先级是 loop region：用户能在时间轴上限定一段反复播放，便于检查某个动作片段、outside 边界和 prediction 段。

后端 v0.9.25 正在并行推进视频帧服务，因此 v0.9.26 只做前端本地状态和 UI，不新增 API，不依赖 chunk、thumbnail、frame cache、segment 或 tracker job。

本版同时接入 R19 的轻量导航基础：跳转历史和临时书签。二者都围绕“快速回到某段视频时间”展开，能复用 `VideoPlaybackOverlay`、`VideoStageControls` 和 `useWorkbenchHotkeys`。

## 2. 目标

1. 时间轴支持拖选 loop region，保存 `{ startFrame, endFrame }` 到当前视频 task 的前端会话状态。
2. 播放时如果设置了 loop region，到达 `endFrame` 后自动 seek 回 `startFrame` 并继续播放；暂停、逐帧、手动 seek 不被强行限制。
3. Overlay 展示 loop region 色带、清除按钮和当前范围文案；拖选时显示临时 preview。
4. 书签支持当前 task 内新增/删除/跳转，最小字段为 `{ id, frameIndex, label?, createdAt }`。
5. 跳转历史记录最近 50 次显式 seek 位置，支持后退/前进，不把播放 tick 每帧写入历史。
6. 快捷键补齐：
   - `Ctrl+M`：当前帧加/删书签。
   - `Ctrl+[` / `Ctrl+]`：跳转历史后退/前进。
   - `Alt+L`：清除 loop region。
7. 文档和 Roadmap 标记 R4.4 第一版完成，R19 标记轻量基础完成；R17/R18/R5 仍保持未做。

## 3. 不做范围

- 不做 hover thumbnail、chapter、已下载范围可视化。
- 不接后端帧服务 v0.9.25 的 B2/B3/B4/B5 接口。
- 不做 loop region 持久化到后端；刷新页面后只通过 sessionStorage 尝试恢复当前 task 的本地状态。
- 不做带正文的书签评论；`label` 先用默认 `F {frame}`，后续可接评论系统。
- 不改 `video_track` / `video_bbox` 协议，不改导出。
- 不做 J/K/L 多速率播放；R18 单独排期。

## 4. 实现方案

### 4.1 状态模型

在 `VideoStage` 增加视频导航状态，公共归一化和持久化解析放到 `videoNavigationState.ts`：

```ts
type VideoLoopRegion = { startFrame: number; endFrame: number } | null;
type VideoBookmark = { id: string; frameIndex: number; label?: string; createdAt: number };
type VideoJumpHistory = { entries: number[]; cursor: number };
```

状态按 `currentTaskId` 做 sessionStorage key 隔离，例如：

- `workbench.video.loop.{taskId}`
- `workbench.video.bookmarks.{taskId}`
- `workbench.video.history.{taskId}`

只在 `VideoStage` 的显式用户动作写历史：range seek、按钮逐帧、keyframe 跳转、bookmark 跳转。历史栈自身前进 / 后退和播放 tick、`useFrameClock` 的自动 frame change 不写历史。

### 4.2 Overlay 交互

改造 `VideoPlaybackOverlay`：

- 在现有 range 上方增加 loop region layer，显示已选范围和拖选 preview。
- `Shift + drag` 时间轴定义 loop region；普通 drag / range change 仍是 seek。
- `onLoopRegionChange(region)` 把合法范围 normalize 为 `start <= end`，且 clamp 到 `[0, maxFrame]`。
- 新增清除按钮，只有存在 loop region 时显示，调用 `onClearLoopRegion()`。
- 书签显示为小三角 marker，点击跳转；与 keyframe/outside marker 使用不同垂直位置，避免重叠。

### 4.3 播放循环

改造 `VideoStage`：

- `handleFrameClockChange(nextFrame)` 中，如果 `isPlaying && loopRegion && nextFrame > endFrame`，调用 `frameClock.seekTo(startFrame)` 并保持播放。
- 如果用户从 loop region 外点击播放，先 seek 到 `startFrame` 再播放。
- 逐帧和手动 seek 可到 region 外；region 是播放循环范围，不是编辑限制。
- task 切换时重置内存状态并按 sessionStorage 恢复当前 task 的 loop/bookmark/history。

### 4.4 快捷键与控制接口

扩展 `VideoStageControls`：

```ts
seekByFrames(delta, options?: { recordHistory?: boolean })
seekToKeyframe(dir, options?: { recordHistory?: boolean })
seekToFrame(frame, options?: { recordHistory?: boolean })
toggleBookmark()
jumpHistory(dir)
clearLoopRegion()
```

扩展 `dispatchKey`：

- `Ctrl+M` → `videoToggleBookmark`
- `Ctrl+[` / `Ctrl+]` → `videoJumpHistory`
- `Alt+L` → `videoClearLoopRegion`

`useWorkbenchHotkeys` 只在 `videoMode` 下消费这些快捷键，避免影响图片工作台。
