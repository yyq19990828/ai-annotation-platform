# v0.9.21 · 视频帧时钟与时间表

> 日期：2026-05-12
> 范围来源：`ROADMAP/2026-05-12-video-workbench-rendering-optimization.md` Wave 1 + `ROADMAP/2026-05-12-video-backend-frame-service.md` B1

## 1. 目标

把视频工作台的帧定位和轨迹插值从“可用但粗糙”推进到后续渲染优化的基础层。

成功标准：

1. 后端在视频 probe 阶段生成 `frame_index -> pts_ms` 时间表，并提供按任务读取的 API。
2. 旧视频没有时间表时继续使用 `fps / frame_count` 估算，不阻断工作台打开。
3. 前端 seek / playback 统一走 `useFrameClock`，优先使用 `requestVideoFrameCallback`，不支持时降级到现有事件模型。
4. 快速连续 scrub 时旧回调不能覆盖最新目标帧。
5. `resolveTrackAtFrame` 使用索引缓存和二分查找，保持现有 `absent=true` 阻断插值语义。
6. 开发环境可以看到最近 seek / frame-ready / long task 诊断信息。
7. 文档、CHANGELOG、Roadmap 和 OpenAPI/前端类型同步更新。

## 2. 不做范围

- 不做 R2 渲染分层、R4 时间轴重设计、R5 chunk/WebCodecs 或 poster hover。
- 不新增 `outside` 段协议；`video_track.keyframes[].absent` 继续作为本期消失语义。
- 不改 `video_track` / `video_bbox` annotation wire shape。
- 不为存量视频强制跑迁移；无时间表时按估算路径降级。
- 不引入新服务，仍走现有 FastAPI + Celery media worker。

## 3. 后端实现

- 新增 `VideoFrameIndex` 模型和 `0056_video_frame_indices.py` 迁移：
  - `dataset_item_id`
  - `frame_index`
  - `pts_ms`
  - `is_keyframe`
  - `pict_type`
  - `byte_offset`
  - 唯一约束：`dataset_item_id + frame_index`
- 扩展 media worker：
  - `probe_video_file()` 保持返回轻量视频 metadata。
  - 新增 `probe_video_frame_timetable()`，用 `ffprobe -show_frames` 读取帧级时间戳。
  - `generate_video_metadata` 在 probe 成功后 upsert 时间表；失败时写 `metadata["video"]["frame_timetable_error"]`。
- 新增 schema 与 API：
  - `GET /api/v1/tasks/{task_id}/video/frame-timetable?from=&to=`
  - 返回 `source="ffprobe"` 时带真实帧表；没有数据时返回 `source="estimated"` 和空 `frames`。
  - 复用视频 manifest 的 task 可见性和非视频 400 校验。

## 4. 前端实现

- 新增 `frameTimebase.ts`：
  - `frameToTime(frameIndex, timebase)`
  - `timeToFrame(mediaTime, timebase)`
  - `buildTimebase(metadata, timetable?)`
  - 有 timetable 用二分；无 timetable 用 `fps` 估算。
- 新增 `useFrameClock.ts`：
  - 暴露 `currentFrame`、`isSeeking`、`seekTo(frameIndex)`、`diagnostics`。
  - `requestVideoFrameCallback` 可用时用 `metadata.mediaTime` 更新帧号。
  - 回退路径使用 `seeked/timeupdate` 和 `requestAnimationFrame`。
  - 使用单调 request id 丢弃过期 seek 回调。
- `VideoStage` 改为：
  - 接入 `frameTimetable` 数据。
  - `seekFrame`、逐帧、播放 tick 和受控 `frameIndex` 同步都走 `useFrameClock`。
  - 保持现有 `VideoStageControls` 和 `VideoPlaybackOverlay` 对外行为。
- `videoStageGeometry.ts`：
  - 对每条 track 通过 WeakMap 缓存 sorted keyframes、non-absent keyframes 和 absent frame indexes。
  - `resolveTrackAtFrame` 用二分查找前后关键帧。
  - 插值结果 LRU 上限 1000 条，避免暂停 scrub 重复计算。
- 开发诊断：
  - 开发环境用 `PerformanceObserver("longtask")` 统计视频工作台长任务。
  - `VideoStage` 把最近帧时钟诊断写到 console / window 诊断对象。
