# v0.10.3 — 1:N 后端管理 UI + 收口（M3）

> 对应 roadmap：[ROADMAP/0.10.x.md](../../ROADMAP/0.10.x.md) §4 v0.10.3 / M3。

## Context

v0.10.0 ~ v0.10.2 已经把后端容器化、`/setup` Capability 协商基础设施、Prompt-first ToolDock 全部落地。运行时由 `MAX_ML_BACKENDS_PER_PROJECT` env 锁 1:1，但 schema/API 已按 1:N 设计：

- 后端 `POST /projects/{id}/ml-backends` 已在 [apps/api/app/api/v1/ml_backends.py:64-84](../../apps/api/app/api/v1/ml_backends.py) 校验上限，返 `409` + `detail{code:"ML_BACKEND_LIMIT_REACHED", message, limit, current}`。
- `GET /projects/{id}` 已透出 `ml_backend_limit`（[apps/api/app/schemas/project.py:76](../../apps/api/app/schemas/project.py)）。
- ProjectSettings 已有列表形态（[MlBackendsSection.tsx](../../apps/web/src/pages/Projects/sections/MlBackendsSection.tsx)），但创建按钮路径在超限时**只会以通用 toast 报错**，没有按 roadmap §3.4 形态弹「暂未支持」专用 Modal；也没有显式显示 limit 配额与 capability 摘要。

M3 的目标是把这块 UI 收口到 roadmap §3.4 的形态、补两条 ADR、补一篇管理员向的 ml-backends.md，并让 CHANGELOG / 版本号过版。本阶段**不引入**真正的多后端运行（路由 / fallback / AB 推迟到 v0.11+），仅让 UI 形态稳定到未来放开 env 即可零代码改动。

## 复用决策

- **Capability 拉取**：不要在 MlBackendsSection 里每行 N 个独立 hook（这是管理面板，不是工作台高频路径）。直接 `useQueries` 批量调 `mlBackendsApi.setup(projectId, backendId)`（[ml-backends.ts](../../apps/web/src/api/ml-backends.ts)），失败容忍。`useMLCapabilities` 是工作台单一后端场景的封装，复用其类型 `MLBackendCapability` 即可，不要在 ProjectSettings 里调用它（语义不同：那里假设单一绑定 backend）。
- **Modal**：复用 [components/ui/Modal.tsx](../../apps/web/src/components/ui/Modal.tsx) 即可，不自创 Dialog 抽象。
- **错误码处理**：apps/web 已有 axios 拦截器把 detail 透传成 Error.message；在 `onError` 里 try-parse JSON 取 `code === "ML_BACKEND_LIMIT_REACHED"`（如不可解析则走默认 toast）。检查现有 [api/ml-backends.ts](../../apps/web/src/api/ml-backends.ts) 错误形状再决定 parse 策略。

## 验收（端到端）

1. 测试环境 `MAX_ML_BACKENDS_PER_PROJECT=1`：项目已绑 1 个 backend → 「注册 backend」按钮置灰，hover 显示 fallback 文案；强行点击 → LimitModal 弹出，标题为「🚧 多后端共存暂未支持」。
2. 用 `curl` 直接 `POST /projects/{id}/ml-backends`（绕过前端按钮禁用）→ 409；前端如果通过其他路径触发也能优雅显示 Modal（验证 onError 分支）。
3. `MAX_ML_BACKENDS_PER_PROJECT=2` 重启 api → 同 UI 形态下可绑第二个，配额角标显示 `已用 2 / 2`，按钮再次置灰。
4. 表格能力列在 grounded-sam2 行显示 `point` `bbox` `text`；sam3-backend 行显示 `bbox` `text` `exemplar`。
5. `docs-site` 本地预览：`pnpm docs:dev` → 管理员侧栏出现 `ML 后端` 入口；ADR-0019/0020 在 `docs/adr/` 可见。
6. `pnpm test -- MlBackend` 全绿；`docker exec ... pytest tests/api/test_ml_backends.py` 不回归。

## 风险

- LimitModal 文案与 server `detail.message` 双源 → 若服务器后续调整文案而 fallback 没跟，会显示不一致。**缓解**：fallback 文案只在 message 缺失时显示，并在 ADR-0019 里写明「文案以服务器为准」。
- 批量调 `/setup` 在 backend 多时会放大请求数。当前 limit=1 不构成问题，未来 N>5 再做合并端点。
