# 0.9.40 · 视频工作台 R10 AI Tracker 前端 + R13 Chapter 系统

> Roadmap 来源：[`ROADMAP/2026-05-12-video-workbench-rendering-optimization.md`](../../ROADMAP/2026-05-12-video-workbench-rendering-optimization.md) Wave 4 / Wave 5。
> 基础版本：v0.9.39（ImageBitmap 缓存 + Viewport / Minimap 已落地）。

## 1. Context

- v0.9.32–v0.9.36 已经把 tracker 后端跑通：`POST /tasks/{task_id}/video/tracks/{annotation_id}:propagate` 创建 job、Celery worker 执行 `mock_bbox` / `sam2_video` / `sam3_video` adapter、Redis pub/sub + WS `/ws/video-tracker-jobs/{job_id}` 推送增量事件、`apply_tracker_results` 把 prediction keyframes 写回 `video_track`。**但前端还没有任何入口**：无法发起 propagate、看不到 job 状态、无法接受/拒绝 prediction keyframe。这一版补齐前端，把已有后端能力变成可交付功能。SAM3 真实视频 backend 尚未实装，前端走 `mock_bbox` 即可端到端联调，后续 SAM 容器上线只需切 `model_key`。
- 同时长视频缺少分段语义：R4 时间轴只画了 keyframe / outside / interpolated / prediction marker，没有"章节"概念。客户在 5 分钟以上视频中难以快速定位场景边界。本版引入轻量 `VideoChapter` 模型（独立于 `video_segments` 的协作分片）和 PageUp/PageDown 跳转，为后续 shot detection 自动分章打底。

## 2. 范围

### 2.1 R10 AI Tracker 前端

**后端 API（已就绪，不改动）**
- `POST /api/v1/tasks/{task_id}/video/tracks/{annotation_id}:propagate` → `VideoTrackerJobOut`，请求体 `VideoTrackerPropagateRequest { from_frame, to_frame, model_key, direction: "forward"|"backward"|"bidirectional", segment_id?, prompt }`。位置 `apps/api/app/api/v1/tasks.py:692`，schema `apps/api/app/schemas/video_tracker_job.py:14`。
- `GET /api/v1/video-tracker-jobs/{job_id}`、`DELETE /api/v1/video-tracker-jobs/{job_id}`（cancel）。位置 `apps/api/app/api/v1/video_tracker_jobs.py:60-91`。
- `WS /api/v1/ws/video-tracker-jobs/{job_id}`。位置 `apps/api/app/api/v1/ws.py:192`。事件 type 由 `_event()` 构造，包含 `job.created` / `job.started` / `job.window_progress` / `job.window_completed` / `job.completed` / `job.failed` / `job.cancelled`（详见 `apps/api/app/services/video_tracker_runner.py:47`）。

**前端新增**
1. **API client**（新文件 `apps/web/src/api/videoTracker.ts`）
   - `propagateVideoTrack(taskId, annotationId, payload)` → `VideoTrackerJobOut`
   - `getVideoTrackerJob(jobId)` / `cancelVideoTrackerJob(jobId)`
   - 类型定义沿用后端 schema 字段。

2. **状态 hook**（新文件 `apps/web/src/hooks/useVideoTrackerJobs.ts`）
   - 维护 `Record<jobId, VideoTrackerJobState>`，结构参考 `useGlobalPreannotationJobs`（`apps/web/src/hooks/useGlobalPreannotationJobs.ts:36`）。
   - 单 job 用 `useReconnectingWebSocket`（`apps/web/src/hooks/useReconnectingWebSocket.ts:27`）订阅 `/ws/video-tracker-jobs/{job_id}`；多 job 同时活跃时为每个 job 单独建连，job 终止后 1500ms 移除。
   - 收到 `job.window_completed` / `job.completed` 时使 React Query annotation 查询失效（`["task-annotations", taskId]`），让新的 prediction keyframes 自动出现在 `VideoTrackPanel`。

3. **Propagate 入口**（改动 `apps/web/src/pages/Workbench/stage/VideoTrackPanel.tsx`）
   - 当选中的 annotation 是 `video_track`（或单帧 `video_bbox`）时，在轨迹卡片工具栏新增 "AI 传播" 按钮 + `Shift+T` 快捷键（`hotkeys.ts` 视频组中 `T` 已用于创建/延续轨迹，`Shift+T` 未占用）。
   - 弹出 `VideoTrackerPropagateDialog`（新文件 `apps/web/src/pages/Workbench/stage/VideoTrackerPropagateDialog.tsx`），字段：方向 (forward/backward/bidirectional)、目标范围（"到下一个 keyframe" / "N 帧" / "到结尾" 三种 preset，自动算 from/to）、`model_key` 下拉（mock_bbox / sam2_video / sam3_video，后两者带 "需要项目绑定 ML backend" 提示）。
   - 提交后调用 `propagateVideoTrack` 并把返回 jobId 写入 `useVideoTrackerJobs`。

4. **进度可视化**（改动 `VideoTrackPanel.tsx` + 新增 `VideoTrackerJobBadge.tsx`）
   - 在轨迹卡片右上角显示活跃 job 状态：queued (灰)、running + 进度百分比 (蓝)、completed (绿，3s 后淡出)、failed (红，带 error_message tooltip)、cancelled (灰)。
   - badge 上有 "取消" 按钮，调用 `cancelVideoTrackerJob`。

5. **Prediction 接受/拒绝**（改动 `VideoTrackPanel.tsx`，复用现有 `nextPredictionFrame` 导航 L147–151）
   - 当前帧是 `source: "prediction"` 时，关键帧详情区显示「接受」「拒绝」「下一条预测」按钮：
     - 接受 = 把 keyframe 的 `source` 改为 `"manual"`，复用现有 annotation 更新通道。
     - 拒绝 = 把该 frame 加入 `outside` 段（保留现有 outside 协议，`source: "rejected_prediction"`）。
     - 下一条预测 = 调用 `nextPredictionFrame()`。

**不做**
- Re-ID join 自动建议（R16）、tracker registry UI（R23）—— 留到后续切片。
- Bidirectional 的 UI 仅做 payload 通道，效果取决于后端 adapter 支持度。

### 2.2 R13 Chapter 系统

**后端新增**
1. **模型** `apps/api/app/db/models/video_chapter.py`（新文件）
   ```python
   class VideoChapter(Base):
       __tablename__ = "video_chapters"
       id: UUID (pk)
       dataset_item_id: UUID (FK dataset_items.id, indexed)
       start_frame: int
       end_frame: int
       title: str (max 200)
       color: str | None  # oklch 或 hex
       metadata: JSONB (default {})
       created_by: UUID | None
       created_at / updated_at
       # constraints: end_frame >= start_frame; (dataset_item_id, start_frame, end_frame) 唯一
   ```
   实现风格参考 `apps/api/app/db/models/dataset.py:205`（`VideoSegment`）。

2. **Alembic 迁移** `apps/api/alembic/versions/0062_video_chapters.py`，参考 `0058_video_segments.py:19-112` 的 table + index 范式。

3. **Pydantic schema** `apps/api/app/schemas/video_chapter.py`：`VideoChapterCreate` / `VideoChapterUpdate` / `VideoChapterOut`，参考 `apps/api/app/schemas/video_tracker_job.py:28`。

4. **CRUD endpoints**（追加到 `apps/api/app/api/v1/videos.py`，与 segments 同文件）
   - `GET /api/v1/videos/{dataset_item_id}/chapters` → list
   - `POST /api/v1/videos/{dataset_item_id}/chapters` → create（require `PROJECT_ADMIN` / `SUPER_ADMIN` / 项目内 `REVIEWER`，仿 `_assert_can_cancel` 范式 `video_tracker_jobs.py:44`）
   - `PATCH /api/v1/videos/{dataset_item_id}/chapters/{chapter_id}`
   - `DELETE /api/v1/videos/{dataset_item_id}/chapters/{chapter_id}`
   - 所有 endpoint 走 `assert_project_visible` + role check；annotator 只能 list。

5. **审计**：`AuditAction` 加 `VIDEO_CHAPTER_CREATE/UPDATE/DELETE`（`apps/api/app/services/audit.py`）。

**前端新增**
1. **API client + hook**（新文件 `apps/web/src/api/videoChapters.ts` + `apps/web/src/hooks/useVideoChapters.ts`，React Query），参考现有 `useVideoSegments` pattern。

2. **时间轴 chapter 色带**（改动 `apps/web/src/pages/Workbench/stage/VideoPlaybackOverlay.tsx`）
   - 在 `globalTimelineDensity` 层之上、`bookmarks` 之下加一条 chapter 色带轨：每个 chapter 渲染为彩色段 + 标题 hover tooltip。
   - 色带高度 ≤6px，不抢占现有 keyframe / outside / loop 渲染区。

3. **章节侧栏**（新文件 `apps/web/src/pages/Workbench/stage/VideoChapterSidebar.tsx`）
   - 在视频工作台右侧轨迹侧栏旁新增 collapsible 区段（或紧邻 `VideoTrackPanel` 之下）。
   - 显示章节列表：序号 / 标题 / `start–end` / 时长。点击 = 跳到 `start_frame`。
   - 项目管理员看到「新增」「编辑」「删除」按钮，弹出简单表单（title + color picker + start/end，可用「以当前帧为起点/终点」快捷按钮）。

4. **PageUp / PageDown 跳转**（改动 `apps/web/src/pages/Workbench/state/hotkeys.ts`，视频组追加）
   - `PageDown` → 跳到下一个 chapter 的 `start_frame`
   - `PageUp` → 跳到上一个 chapter 的 `start_frame`（如果当前帧已在某 chapter 内，先回到该 chapter 起点）。
   - 已确认 `PageUp/PageDown` 未被其他视频快捷键占用。

5. **颜色 palette** 复用 `apps/web/src/pages/Workbench/stage/colors.ts:13` 的 `CLASS_COLORS`，章节创建时若 `color` 为空按 `chapterIndex % palette.length` 派生默认色。

**不做**
- Shot detection / 自动分章 worker —— 留作后续。
- 章节与 `video_segments`（协作分片）合并 —— 两者语义不同，分开。
- 章节级评论锚点 —— 现有 `frame` 锚点已够用。

## 4. 复用现有工具 / 不要重造

- `useReconnectingWebSocket`（`apps/web/src/hooks/useReconnectingWebSocket.ts:27`）—— tracker WS 直接用，不要写新 reconnect 逻辑。
- `useGlobalPreannotationJobs` 的 `Record<id, state>` + 1500ms 终止后清理（`apps/web/src/hooks/useGlobalPreannotationJobs.ts:36`）—— tracker job 状态机仿此模式。
- `nextPredictionFrame()`（`VideoTrackPanel.tsx:147`）—— 接受/拒绝 prediction 后导航复用。
- `CLASS_COLORS` palette（`colors.ts:13`）—— chapter 默认色。
- `require_roles` + `is_privileged_for_project`（`apps/api/app/deps.py:82` / `scheduler.py`）—— chapter 写权限校验。
- alembic 模板参考 `0058_video_segments.py`。

## 5. 风险与暂缓

- **WS 鉴权**：现有 `/ws/video-tracker-jobs/{job_id}` 未鉴权（与同文件 `/ws/prediction-jobs` 一致），channel 名含 job UUID 自带不可猜性，沿用即可。如果安全审计要求收紧，本版不做，记入后续。
- **Bidirectional propagate**：mock_bbox adapter 支持有限，UI 提供选项但实际效果以 adapter 为准；不在此版校验。
- **章节边界与 segment 锁的交互**：章节是纯展示语义，不影响 segment claim/lock。
- **导出**：章节信息暂不进 COCO / MOT 导出，等 R22 视频导出统一处理。

## 6. 验证步骤

### 6.1 R10 端到端
1. 起 dev 栈：`docker compose up -d`（已起则跳过），`pnpm --filter web dev`。
2. 在视频任务里用 `T` 创建一条 `video_track`，关键帧只有 1 个（frame 0）。
3. 选中轨迹 → 工具栏 "AI 传播" 或 `Shift+T` → 选 forward / model_key=mock_bbox / "向后 30 帧" → 提交。
4. 预期：badge 显示 queued→running→completed；mock_bbox 在 frame 1..30 写入 prediction keyframes；时间轴 prediction marker 出现；侧栏 `nextPredictionFrame` 能逐帧导航；接受按钮把单帧 source 改为 manual，拒绝按钮把单帧并入 outside。
5. 再发一个 long-range job 立刻点取消 → badge 变 cancelled，WS 收到 `job.cancelled`。
6. 失败路径：临时把 `model_key` 改成 `sam2_video` 但项目未绑定 ML backend → adapter 返回 unavailable → badge 显示 failed + error_message。

### 6.2 R13 端到端
1. `docker exec ... alembic upgrade head` 应用 0062 迁移。
2. 以项目管理员身份在章节侧栏创建 2 个 chapter：`Intro 0–60` / `Action 61–200`。
3. 预期：时间轴出现两条色带；annotator 切角色登录只能看不能改；`PageDown` 从 frame 0 跳到 61；再 `PageDown` 到 200（或卡在最后一个 chapter 起点）；`PageUp` 反向。
4. 编辑章节 title / 范围 → 时间轴和侧栏实时更新。
5. 删除章节 → 时间轴色带消失。
6. Audit log 包含 `VIDEO_CHAPTER_CREATE/UPDATE/DELETE`。

### 6.3 自动化
- 后端：`uv run pytest apps/api/tests/ -k "video_chapter or video_tracker_propagate"`。
- 前端：`pnpm --filter web test`（确认现有 video workbench 测试无回归 + 给新 hook 加 happy-path）。
- 类型：`pnpm --filter web typecheck` + `uv run mypy app`（如启用）。
