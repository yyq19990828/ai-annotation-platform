# v0.9.41 · 图片工作台 Wave α 基础稳态

> 范围：ROADMAP/2026-05-12-image-workbench-optimization.md 中 **Wave α** 五条（I3 + I7 + I8 + I16 + I17）。Wave β/γ/δ/ε 留待后续 minor 版本。
>
> 后续版本可重命名为 `2026-05-13-v0.9.41-image-workbench-wave-alpha.md`（遵循 CLAUDE.md §6 命名规范）。

## Context

图片工作台已经是项目最成熟的一块（Konva 五层、Viewport / Minimap / Blurhash、rAF 节流、SAM 集成），但有几处低悬的甜果一直没收：

- `selectedIds` 每次 render 都 `new Set(...)`，下游 memo 失效（[ImageStage.tsx:141-143](apps/web/src/pages/Workbench/stage/ImageStage.tsx)）。
- 通用 hook（`useViewportTransform` / `Minimap` / `polygonGeom`）没有 `stage/shared/`，视频侧已经在 import 图片侧路径，依赖方向反了。
- longtask 上报只在 `useFrameClock.ts:244-256` 单点，没抽成 hook，BugReport 也没附 viewport 诊断。
- 顶点拖动每次都 push history、发字段级 PATCH，offline queue 已就绪但前端没 dirty tracker，序列化代价大。
- 渲染参数（imageSmoothing / 亮度 / 顶点大小）散在源码，用户无法调；后端 `User.preferences` 字段不存在。

本版本目标：把"看不见但每天都在累积成本"的基础设施一次到位，为 Wave β 性能优化和 Wave γ 形状能力扩展铺路。**纯前端 + 一条后端迁移**，无新 UI 范式。

## 实施清单

### I3 · selectedIds / memo 稳定引用

**文件**：[apps/web/src/pages/Workbench/stage/ImageStage.tsx](apps/web/src/pages/Workbench/stage/ImageStage.tsx):141-143、490-644

- L141-143：`selSet` 改为基于内容签名的稳定引用 —— 当 `selectedIds` 数组顺序无关地等价时返回上次的 Set 实例。实现：`useMemo` 内 sort+join 作为 key，外层 ref 缓存上次 key→Set。
- `currentShapes` 衍生（filteredShapes / visibleShapes）走稳定 selector：抽出 `selectVisibleShapes(shapes, selSet, toolState)`，避免每次渲染 Konva children 全量重建。
- L490 (bg)、L609 (historical-canvas)、L644 (overlay) 已是 `listening={false}`，**无需改**；但 L498 (ai) 和 L528 (user) 在 HandTool / 只读模式下也应关闭 listening：在工具状态切换处加 `listening={toolState !== 'hand' && !readOnly}`。

**验收**：Chrome Performance 录拖动场景，React commit 时间下降 ≥30%。

### I8 · 观测接入

- 新建 `apps/web/src/pages/Workbench/stage/shared/useWorkbenchPerf.ts` —— 复刻 [useFrameClock.ts:243-258](apps/web/src/pages/Workbench/stage/useFrameClock.ts) 的 longtask 监听，输出 `{ longTaskCount, longTaskMaxMs, lastLongTaskAt }`。图片 + 视频 stage 顶层调用，挂载到现有 diagnostics 收集器。
- 将 longtask 从 dev-only 改为带采样的生产可用（采样率 5%，可由 I17 配置覆盖）。
- BugReport：在 `BugReportDrawer` 收集快照点（搜 `recent_console_errors` 相邻代码）追加 `viewport_snapshot: { scale, tx, ty, longTaskStats, hitTestMs }`。
- 基准 fixture 推迟到 Wave β（与视频 R7 同步设计），本版本仅打通采集链路。

### I16 · State 脏标记 + 增量序列化

后端 PATCH `/annotations/{id}` 已支持 `exclude_unset=True`（[tasks.py:844-858](apps/api/app/api/v1/tasks.py)），offline queue（[offlineQueue.ts](apps/web/src/pages/Workbench/state/offlineQueue.ts)）也已就绪。本条只在前端补 dirty 追踪：

- 新增 `apps/web/src/pages/Workbench/state/useDirtyTracker.ts`：
  - `markDirty(annotationId, field)` / `getDirtyFields(annotationId)` / `clear(annotationId)`
  - 内部 `Map<string, Set<keyof AnnotationUpdatePayload>>`。
- 修改 `useAnnotations` 的 setShape 路径：调用 `markDirty` 而非每次构造全量 payload；flush 时按 dirty fields 选字段调 `AnnotationUpdatePayload`。
- 拓展 [useAnnotationHistory.ts:9-22](apps/web/src/pages/Workbench/state/useAnnotationHistory.ts) 的 kind 联合：新增 `'polygonVertexBatch'`，payload `{ id, vertexPatches: Array<{idx, from, to}> }`。配套 `commitPolygonGeometry` 在拖动结束（mouseup）时打包整次拖动为一条 batch。
- ImageStageShapes.tsx 顶点拖动：拖动中只 ref 本地，松开才走 `onCommitPolygonGeometry`（避免每帧 push history）。
