# v0.9.1 — Grounded-SAM-2 backend embedding cache (M1 / Declarative Feather)

> 计划文件名沿用 plan-mode 默认路径；按 CLAUDE.md §6 规范在落地 commit 时改写为 `2026-05-07-v0.9.1-declarative-feather.md` 即可。

## Context

v0.9.0 (`01f8d45`) 已经把 grounded-sam2-backend 容器化并跑通三种 prompt 同端点分流，实测 RTX 4060 上 bbox / text / point 的首次延迟分别是 2408 / 3559 / 1602 ms。其中 SAM 2 `set_image()` 计算 image embedding 是大头；同一张图被连续点击（工作台交互式精修的典型动作）时这一段重复花费完全可省。

v0.9.1 (M1) 的目标是把 SAM 2 的 image embedding 用 LRU 缓存起来，让同图二次操作从 ~1.6s 降到 < 50ms，并补 Prometheus 观测和缓存统计端点，为后续 M2 工作台 `S` 工具的体验铺路。范围严格限定在 backend 容器内部，不动平台 API、不改协议契约。

ROADMAP 锚：[`ROADMAP/0.9.x.md`](../../ROADMAP/0.9.x.md) §4 v0.9.1。

## 设计要点

### 1. 缓存键
- `cache_key = sha1(url_path + "|" + sam_variant)`
- `url_path` 取自 `file_path`：HTTP/HTTPS 走 `urllib.parse.urlsplit().path`（**剥掉 query string**，避免 MinIO presigned signature TTL 滚动导致缓存失效）；本地路径直接用绝对路径。
- 拼上 `sam_variant`（`tiny` / `small` / `base_plus` / `large`）保证切大模型时不会命中过期 embedding。
- DINO 端不缓存（每次 prompt caption 不同，命中率低；ROADMAP §3.5 已决）。

### 2. 缓存范围
**三种 prompt 全覆盖**：point / bbox / text 三条路径里 SAM 的 `set_image()` 都接缓存。
- `predict_point` / `predict_bbox` 命中时还可以**跳过 `_fetch_image()`** —— 因为 SAM `predict()` 只需要 embedding + `(w, h)` 做坐标反归一化；把 `(w, h)` 一同存进缓存条目即可。
- `predict_text` 命中时仍需要原图给 GroundingDINO 跑 `dino_predict`（caption→boxes），所以省 `set_image()`、不省 fetch。

### 3. 缓存值
SAM2ImagePredictor 在 `set_image()` 之后把状态写到 `_features` / `_orig_hw` / `_is_image_set` / `_is_batch` 几个实例属性。缓存值是 `{features: dict, orig_hw: tuple[int,int], wh: tuple[int,int]}`，命中时把这些字段写回 predictor 实例后直接调 `predict()`。`features` 里的 tensor 已在 GPU，`deepcopy` 对 GPU tensor 不友好；存的就是引用，配合 LRU 容量上限保证不会无限增长。

### 4. 容量
- 默认 `EMBEDDING_CACHE_SIZE=16`（4060 / 8GB）；env 可调到 32（3090+ / 24GB）。
- 单条 SAM 2 tiny embedding ≈ 256×64×64 fp16 ≈ 4 MB；16 条 ≈ 64 MB GPU，相对 SAM 模型本体可忽略。
- 文档要写明：切到 `large` 变体时单条 embedding 体积放大 ~6×，需要把 cache size 调小。

### 5. Prometheus 接入
风格对齐 `apps/api/app/observability/metrics.py`（raw `prometheus_client`）。Backend 内新建 `observability.py` 模块，定义：

| metric | 类型 | labels | 含义 |
|---|---|---|---|
| `embedding_cache_hits_total` | Counter | `prompt_type` | 命中次数（按 prompt 类型分） |
| `embedding_cache_misses_total` | Counter | `prompt_type` | 未命中次数 |
| `embedding_cache_size` | Gauge | — | 当前缓存条目数 |
| `inference_latency_seconds` | Histogram | `prompt_type`, `cache` (`hit`/`miss`) | 端到端 `/predict` 延迟 |

新增 `GET /metrics` 端点（`generate_latest()`）。

### 6. `/cache/stats`
人类可读的 JSON：
```json
{ "size": 12, "capacity": 16, "hits": 187, "misses": 32, "hit_rate": 0.854, "variant": "tiny" }
```
不进 ml-backend-protocol（backend 内部端点）。

## 验证

### 单元测试（无 GPU 即可跑）
- `pytest apps/grounded-sam2-backend/tests/test_embedding_cache.py -v`
- 覆盖：put/get、LRU 淘汰顺序、cache_key 在变体切换时隔离、URL query string 不影响 key、并发 put 在 lock 下安全。

### 集成验证（需 GPU 容器）
1. `docker compose --profile gpu up grounded-sam2-backend`，等 `/health` 返回 `loaded: true`。
2. 用 `curl` 对同一 `file_path` 连续 5 次 point 请求：
   ```bash
   for i in {1..5}; do
     time curl -s -X POST localhost:8001/predict \
       -H 'content-type: application/json' \
       -d '{"task":{"id":"t1","file_path":"<minio_url>"},"context":{"type":"point","points":[[0.5,0.5]],"labels":[1]}}' \
       | jq '.inference_time_ms'
   done
   ```
   **预期**：第 1 次 `inference_time_ms ≤ 500`（4060），第 2-5 次 ≤ 50。
3. `curl localhost:8001/cache/stats` → `hits=4 misses=1 size=1`。
4. `curl localhost:8001/metrics | grep embedding_cache` → 看到 hits/misses Counter 增长正确。
5. 切到不同 `file_path` → 看到 misses 增长 size→2；继续到 17 张图 → size 稳在 16，最早的被 evict（用 cache_stats 验证 size 上限）。
6. 切 `SAM_VARIANT=small` 重启容器后再请求同 file_path → 第 1 次仍 miss（变体隔离生效）。

## 风险 & 注意
- **GPU OOM**：若运维误把 cache size 调到 64+ 配 `large` 变体，单缓存可吃掉 ~1.5 GB。文档要给容量公式 `≈ size × variant_embedding_bytes`，并在 `/cache/stats` 里同时回 `gpu_mem_estimate_mb`。
- **SAM 2 内部属性变动**：`_features` 等是 vendor 内部 API。`scripts/sync_vendor.sh` 升级 commit 时必须人肉回归；在 `embedding_cache.py` 顶部加注释提醒，单测里加一条「snapshot 后 restore 再 predict 结果与首次一致」的契约测试（需 GPU，标 `@pytest.mark.gpu`，CI 默认 skip）。
- **并发**：FastAPI 单 worker 单进程，但同一 worker 里两个 predict 异步交错可能踩 SAM predictor 单实例状态。EmbeddingCache 加锁是惯性保险；真正关键的是 predictor 调用本身需保持串行（v0.9.0 已是这样：`_predictor` 是单例，predict 内部同步调用），M2 加并发前再观察。
- **plan 文件改名**：合并前按 CLAUDE.md §6 改 `docs/plans/2026-05-07-v0.9.1-declarative-feather.md`。
