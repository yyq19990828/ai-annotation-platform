# v0.9.2 — 工作台 SAM `S` 工具 + 文本入口（M2 / Luminous Canvas）

## Context

v0.9.0 / v0.9.1 已经把 grounded-sam2-backend 容器化 + embedding LRU 缓存铺好。后端 `POST /projects/{pid}/ml-backends/{bid}/interactive-annotating` + 前端 `mlBackendsApi.interactiveAnnotate` 在 v0.8.6 已就位。剩下的全部缺口都在工作台前端：标注员没有任何入口去触发交互式 SAM；DINO 阈值仍是 backend env 全局值，不同业务图无法在 UI 调。

v0.9.2 (M2) 的目标是把 SAM 工具从「能跑」变成「能用」：标注员选 `S` 工具，**点击 / Alt+点击 / 拖框** 就能让后端在 ~1.6s（首次）/ <50ms（命中缓存）内返回 polygon 候选；右侧 AI 助手在 S 模式下显出**文本提示**输入框，输入 "person" 这类英文 prompt 一键全图召回。返回的 polygon 以**紫虚线**叠加到 Konva 画布，`Enter` 接受 / `Esc` 取消 / `Tab` 切候选。

ROADMAP 锚：[`ROADMAP/[archived]0.9.x.md`](../../ROADMAP/[archived]0.9.x.md) §4 v0.9.2。

代号：`luminous-canvas`（流光画布——交互式 SAM 在 Konva 上以紫虚线点亮的视觉记忆点）。

## 决策（§5 拍板）

1. **中→英翻译**：v0.9.2 **不做**。后端没有 LLM client，引入会挤压主线 6 天预算。文本输入框旁加一行 hint「英文 prompt 召回最佳（如 `person` / `car`）」。后续如客户反馈强烈，单独开 micro-feature 走平台 LLM 网关。
2. **`box_threshold` / `text_threshold` 项目级 override**：v0.9.2 **一并做**。Project 表新增两列（默认 0.35 / 0.25）+ ProjectSettings UI 数值字段 + `/predict` 协议透传。这两个值是「特定业务图（车牌 / 商品 / 卫星）必调」的旋钮，做完 SAM 工具刚好赶在客户落地前把旋钮交到运营手里。

## 范围

### 后端

#### `apps/api`（Project 阈值字段）

- `0048_project_dino_thresholds.py`：
  ```sql
  ALTER TABLE projects ADD COLUMN box_threshold  REAL NOT NULL DEFAULT 0.35;
  ALTER TABLE projects ADD COLUMN text_threshold REAL NOT NULL DEFAULT 0.25;
  ```
  CHECK 约束：`box_threshold BETWEEN 0 AND 1`、`text_threshold BETWEEN 0 AND 1`。
- `db/models/project.py`：加两个 `Mapped[float]` 字段。
- `schemas/project.py`：`ProjectUpdate` / `ProjectOut` 都加上；`Annotated[float, Field(ge=0, le=1)]`。
- `api/v1/ml_backends.py::interactive_annotating`：从 `project` 取 `box_threshold` / `text_threshold`，**注入到 `body.context`**（仅当客户端未显式覆盖时；UI 默认不传，由后端兜底）。
- `schemas/ml_backend.py::InteractiveRequest.context` 已是 `Dict[str, Any]`，无需改 schema。

#### `apps/grounded-sam2-backend`（接受 context 阈值 override）

- `main.py::_run_prompt`：
  - text 路径多读 `ctx.get("box_threshold")` / `ctx.get("text_threshold")`，向 `predictor.predict_text` 透传。缺省回退到 `BOX_THRESHOLD` / `TEXT_THRESHOLD` env。
  - point / bbox 不受影响（没有 DINO）。
- `predictor.py::predict_text`：函数签名加 `box_threshold: float | None = None` / `text_threshold: float | None = None`，None 时用 `self.box_threshold` / `self.text_threshold`。
- `tests/`：补一个 `test_threshold_override.py`，断言 ctx 给的阈值优先于 env。

### 前端

#### 1. 注册 `S` 工具

- `pages/Workbench/stage/tools/SamTool.ts`：
  - `id: "sam"`、`hotkey: "S"`、`label: "SAM 智能"`、`icon: "sparkles"`、`cursor: "crosshair"`。
  - `onPointerDown`：space pan / readOnly 走 pan；`evt.shiftKey` 不再额外处理（SAM 选区由 useInteractiveAI 接管）；返回 `{ kind: "draw", ... }` 让 stage 走拖框流程；松手后由 ImageStage 把几何送到 `useInteractiveAI`（而非 onCommitDrawing 落库）。
  - **对外语义切换**：`tool === "sam"` 时，ImageStage 的 `onCommitDrawing` 改派发到 SAM 路径（看下方 ImageStage 改动）；点击（pointerDown 后立即 up，无拖动距离）= positive point；Alt+click = negative point；拖框 = bbox prompt。
- `pages/Workbench/stage/tools/index.ts`：注册到 `TOOL_REGISTRY` + `ALL_TOOLS`，加到 `ToolId` union。
- `pages/Workbench/state/useWorkbenchState.ts`：`Tool` union 加 `"sam"`。
- `pages/Workbench/state/hotkeys.ts`：
  - `HOTKEYS` 加一条 `{ keys: ["S"], desc: "SAM 智能工具", group: "draw", actionType: "setTool" }`。
  - `RESERVED_LETTERS` 加 `s/S`。
  - `dispatchKey`：在 `b/p/v` 旁加一行 `if (e.key === "s" || e.key === "S") return { type: "setTool", tool: "sam" }`，`HotkeyAction.setTool.tool` union 加 `"sam"`。

#### 2. `useInteractiveAI` hook（state/）

```ts
useInteractiveAI({ projectId, taskId, mlBackendId })
  → {
    runPoint: (pt: [number, number], polarity: 1 | 0) => void,
    runBbox:  (bbox: [number, number, number, number]) => void,
    runText:  (text: string) => void,
    candidates: PendingCandidate[],   // 紫虚线 polygon 队列
    activeIdx: number,                 // Tab 切换光标
    accept: (idx?: number) => void,
    cancel: () => void,
    cycle: (dir: 1 | -1) => void,
    isRunning: boolean,
  }
```

- 路由：所有 prompt 类型都走 `mlBackendsApi.interactiveAnnotate(projectId, mlBackendId, { task_id, context: { type, ... } })`。
- 防抖：`runPoint` 用 80ms debounce 合并连续点击（轻击 / 多点同图场景）。`runBbox` / `runText` 不防抖（一次完整动作）。
- 失败：`pushToast({ msg: "SAM 推理失败", sub: 错误信息.slice(0,80), kind: "error" })`，candidates 不动。
- mlBackendId 缺失（项目未绑定）：toast "项目未绑定 ML Backend，无法使用 SAM" + 不发请求。

#### 3. 紫虚线 polygon 叠加 + 接受/取消交互

- `stage/PendingPolygonsOverlay.tsx`（新增，~80 行）：
  - 接 `candidates`、`activeIdx`、`vp`、`imgW`、`imgH`，渲染 `<Konva.Line points={...} stroke="#a855f7" dash={[6, 4]} closed />`。
  - 当前候选 stroke 加粗（width=2.5）；其它候选半透（opacity=0.5）。
  - 顶点角标（小圆点）+ 候选序号 chip（左上角，紫色，e.g. `1/3`）。
- 接受流：`Enter` → 复用 `ClassPickerPopover`（锚到候选 polygon AABB），onPick 调 `createAnnotation` 创建 polygon 标注（geometry: `{ type: "polygon", points }`），落库后清队列；onCancel 仅退出当前候选回到候选列表。
- 取消流：`Esc` → `cancel()` 清空 candidates。
- Tab：`cycle(1)`；Shift+Tab：`cycle(-1)`。
- 这些键位接到 `useWorkbenchHotkeys`：S 模式且 candidates.length > 0 时优先吃 Enter / Esc / Tab，绕过 polygon 多边形闭合 / 任务取消等已有分支。

#### 4. AI 助手「文本提示」入口

- `AIInspectorPanel.tsx` 顶部「一键预标 + 全部采纳」按钮组下方，**仅 tool === "sam" 时**渲染：
  ```
  ┌ SAM 文本提示 ───────────────┐
  │ [输入框: e.g. person]       │
  │ [找全图] (loading 转圈)     │
  │ 英文 prompt 召回最佳        │
  └─────────────────────────────┘
  ```
- onClick「找全图」→ `runText(text)`。空文本 disabled。
- props 加 `tool: ToolId` + `onRunSamText: (text: string) => void` + `samRunning: boolean`。

#### 5. ProjectSettings 阈值面板

- `pages/ProjectSettings/sections/ModelIntegrationSection.tsx`（或现有 AI 段）加两个数值输入：
  - 「DINO box 阈值」 默认 0.35，范围 [0, 1]，step 0.05。
  - 「DINO text 阈值」 默认 0.25，范围 [0, 1]，step 0.05。
  - 旁注：「特定业务图（车牌 / 商品）召回不足时降低；噪声多时升高」。
- `api/projects.ts` ProjectUpdate payload + Project 类型加字段。

## 风险

| 风险 | 缓解 |
|---|---|
| Konva 拖框松手与 SAM 路径双触发（已有 BboxTool 的 onCommitDrawing 走 ClassPicker） | tool 分发：`tool === "sam"` 时 ImageStage 的 `onCommitDrawing` 不进 `s.setPendingDrawing`，而是 `runBbox`；BboxTool / SamTool 共用同一 stage drag 路径只是终点不同。 |
| 防抖把首次 click 也吃掉 | 第一次 click 立即触发；后续 80ms 内合并为最后一次（leading + trailing 取舍：leading=true）。 |
| Project 表新字段对老项目兼容 | `server_default="0.35"` / `"0.25"` 让所有历史行自动得到值；schema `Field(ge=0, le=1)` 防越界。 |
| AIInspectorPanel S 模式条件渲染破坏现有快照测试 | 默认 tool="box" 不渲染，旧测试不受影响；新加测试覆盖 S 模式分支。 |
