# v0.9.5 — Async Oasis（M4 + M5 + 工具栏 UX + 截图回填 + 类别英文 alias + 多 chip 收口）

> 版本代号 **Async Oasis**：M4 主线是异步批量预标（Celery + WS 进度），v0.9.5 是 v0.9.x 系列在 v0.10.x SAM 3 接入前的收尾绿洲，把 SAM 接通后暴露的零散问题一次性收齐。
>
> 计划日期：2026-05-08（开始 v0.9.4 phase 3 已落次日）
> 预估：~11 工作日 / ~2.2 周（用户拍板「主线 + 多 chip 全打包」）

---

## Context

为什么这版要做：

1. **M4 主线（产品价值）** — `/ai-pre` 仍是 PlaceholderPage，是 v0.9.x 系列对外承诺的最后一块功能拼图（文本批量预标），不收口的话 v0.9.x 整条线半个收尾。后端 70% 骨架已就位（Celery `batch_predict` task `apps/api/app/workers/tasks.py:104-109` + Redis pub/sub `_publish_progress` + WS `/ws/projects/{project_id}/preannotate` `apps/api/app/api/v1/ws.py:48-67` + `MLBackendClient.predict()` `apps/api/app/services/ml_client.py:44-76`），只缺触发端点 + 前端页面 + batch 状态机扩展。
2. **M5 运维收口** — backend `/health` 当前仅 `torch.cuda.is_available()` 布尔（`apps/grounded-sam2-backend/main.py:73-80`），生产部署没显存数值；ADR-0012/0013 + deploy.md GPU 章节缺位。
3. **工具栏 UX 重构 + 截图回填同窗口** — ROADMAP §C.2 明确「建议与 v0.9.5 截图自动化回填同窗口避免双拍」。9 处 sparkles 共用识别度归零，工具栏 UX 仍是 v0.5.3 基线；截图 14 张目前 0/14 实跑（`apps/web/e2e/screenshots/scenes.ts` 20 个 scene 配置就位）。两条同窗口可共享一次截图回填，错过这窗口下次得重拍。
4. **中→英翻译开关 → 类别英文 alias（用户决策）** — 不引入运行时 LLM 依赖，把双语映射推到数据层：`Project.classes_config[name]` 加可选 `alias` 字段，SAM 文本输入框升级为 Combobox 从 `project.categories` 拉已配置 alias 下拉。优雅、零外部依赖、覆盖率高于词典、demo 场景直接可用。
5. **打包零散 chip** — cost 单条透传（`PredictionMeta.total_cost` 字段已就位 `apps/api/app/db/models/prediction.py:46-52`、AdminDashboard 已聚合，前端零拉一下）+ `text_output_default` 项目级字段（v0.9.4 phase 2 用 sessionStorage 兜底，转持久化）+ GeneralSection AI 绑定 UX 解耦。

预期结果：v0.9.x 系列在此版本收尾干净，进入 v0.10.x SAM 3 接入时无 v0.9.x 残留尾巴。

## 风险

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| Postgres enum ADD VALUE 不能在事务中跑 | 高 | 0049 迁移失败 | 迁移 op 用 `op.execute("COMMIT")` + `ALTER TYPE`，alembic 文档标准做法 |
| WS 频道名跨 worker 不一致 | 中 | 进度推不到前端 | 用 `task_id`（Celery uuid）拼频道而非 batch_id；前端订阅时同 uuid |
| Combobox 在中文输入法 IME 下 Enter 误提交 | 中 | UX 卡顿 | onKeyDown 检查 `e.nativeEvent.isComposing`，已有现存模式可复用 |
| 批量预标跑 100+ 张 LRU cache 击穿 | 低 | 同图重 set_image 重复推理 | v0.9.1 LRU(16) 在 batch 内充分；> 16 张时按图序 LRU 自然 evict 可接受 |
| 截图回填触发 fixture 兜不住的 race | 中 | 部分 PNG 仍空白 | seed.py prepare 钩子先用 sql 落定数据再 trigger，scene 间加 `awaitNetworkIdle` |
