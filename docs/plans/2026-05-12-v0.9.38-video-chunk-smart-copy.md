# v0.9.38 · Video Chunk Smart-Copy

## 目标

本版本落地视频后端帧服务的非 AI P0：chunk smart-copy 与前端 R5.3 解码诊断底座。范围限定在 FastAPI + Celery + PostgreSQL + MinIO 现有架构内，不改 tracker adapter、ML Backend client 或真实 SAM video backend。

## 范围

1. `VideoChunk` 持久化生成模式与诊断信息，API 向前兼容地返回新增字段。
2. `ensure_video_chunks` 在 codec 支持且 chunk 起始帧 keyframe 对齐时优先尝试 ffmpeg stream copy。
3. smart-copy 失败时自动 fallback 到现有 H.264 baseline fragmented MP4 重编码。
4. 文档、OpenAPI 和测试同步 chunk 诊断契约。

## 非目标

- 不实现真实 SAM 2 / SAM 3 video predictor。
- 不改 AI tracker job、adapter registry、GPU worker 或 `ml_client`。
- 不做 timetable compact / sparse。
- 不引入 DASH / HLS / adaptive bitrate。

## 验收

- eligible chunk 写入 `generation_mode="smart_copy"` 和 codec/keyframe/byte offset 诊断。
- 非 keyframe 或不支持 codec 的 chunk 继续走 transcode，并暴露 fallback reason。
- smart-copy 失败后 chunk 仍可通过 transcode 生成 ready。
- 旧 pending/ready/failed chunk 行不需要迁移数据即可继续被 API 返回。
