# v0.10.x — SAM 3 接入（与 Grounded-SAM-2 并存）

> 状态：**草案 / 待对齐**。前置依赖：v0.9.x Grounded-SAM-2 已上线，详见 [`0.9.x.md`](./[archived]0.9.x.md)。
>
> 目标：在 v0.9.x grounded-sam2-backend 之外**新增** sam3-backend，作为「高精度 / 单模型 / GPU 充裕场景」的并存选项。**不下线 grounded-sam2-backend**。
>
> 范围边界：v0.10.x **不做**视频追踪 UI、不做训练 / Active Learning。视频 predictor 留给 v0.11+。

---

## 0. TL;DR

- v0.9.x 已经把协议、工作台 `S` 工具、`/ai-pre` UI、embedding 缓存、mask→polygon、成本卡片、健康检查全部落到位 —— v0.10.x **复用 ~80%**，只新增 sam3-backend 服务 + exemplar prompt + 路由策略 + AB 对比工具。
- 两个 backend 并存策略：**按 prompt 类型 + 项目偏好路由**。grounded-sam2 兜底（更快、笔记本能跑），sam3 当首选（更准、需充裕 GPU）。
- SAM 3 只发了 848M 一档（无 tiny/large 之分）；4060 显存吃紧（FP16 ~6-7GB），建议生产部署在 3090 或 A100 上；4060 笔记本继续走 grounded-sam2。
- 切成 4 个里程碑：M0 sam3-backend 容器化 → M1 工作台支持 exemplar + 路由策略 UI → M2 AB 对比工具（同任务跑两 backend，平台显示对比）→ M3 运维并存收口。
- 估时 ~3 周。

## 1. SAM 3 能力盘点

### 1.1 模型尺寸（**只有一档**）

| 维度 | 情况 |
|---|---|
| 参数规模 | **仅 848M**（无 SAM 2 那种 tiny / large 多档） |
| checkpoint | ~3.2GB |
| 官方仓 | `facebook/sam3`（2025-11）+ `facebook/sam3.1`（2026-03，仅 checkpoint）|
| HuggingFace 拉权重 | 需 `HF_TOKEN` |
| 子模型（共享 backbone） | `Sam3Model` / `Sam3VideoModel` / `Sam3TrackerModel` / `Sam3TrackerVideoModel` |
| 社区量化 | HF 上有 ~10 个 int8/int4，可降到 ~2GB；精度未官方背书 |

### 1.2 能力（vs Grounded-SAM-2）

| 能力 | grounded-sam2 (v0.9.x) | sam3 (v0.10.x) |
|---|---|---|
| point / bbox prompt | ✅ SAM 2.1 直接做 | ✅ SAM 3 visual prompt |
| 文本 prompt（找全图实例）| ✅ DINO → SAM 2 复合链 | ✅ SAM 3 PCS 单模型一步出 |
| **exemplar prompt（图像示例）** | ❌ 不支持 | ✅ SAM 3 PCS |
| 复合错误率 | DINO + SAM 复合 | 无（单模型）|
| 推理延迟（同硬件）| DINO+SAM 串行 | SAM 3 单次 forward，但模型大 |
| 4060 笔记本可用 | ✅ 200-500ms | ⚠️ FP16 显存 6-7GB，单图勉强、批量不行 |
| 3090 工作站 | ✅ 100-200ms | ✅ 200-400ms |
| A100 生产 | ✅ 50-100ms + 高并发 | ✅ 30-60ms（H200 上 30ms / 100+ 对象）|

### 1.3 软件依赖
- Python 3.12+ / PyTorch 2.7+ / CUDA 12.6+（与 v0.9.x grounded-sam2 的 3.10 / 2.3 / 11.8 不兼容 → 各跑各的容器）
- License：单独看 `facebook/sam3` 的 LICENSE，**不与 SAM 2 自动等价**，商用前逐条核对
- 默认走 facebookresearch/sam3 + 自建 wrapper（不用 Ultralytics 的 AGPL 集成）

## 3. 架构设计要点

### 3.1 部署拓扑（双 backend 并存）

```
┌─────────────────┐
│  apps/api       │  根据 prompt.type + project 偏好路由
│   MLBackendClient│
└──┬──────────────┘
   │
   ├──────▶ ┌──────────────────────────────────┐
   │        │ grounded-sam2-backend (v0.9.x)    │  GPU 11.8 / Python 3.10
   │        │ point / bbox / text              │
   │        │ 4060 / 3090 都能跑               │
   │        └──────────────────────────────────┘
   │
   └──────▶ ┌──────────────────────────────────┐
            │ sam3-backend (v0.10.x 新)         │  GPU 12.6+ / Python 3.12
            │ point / bbox / text / exemplar   │
            │ 推荐 3090 / A100                 │
            └──────────────────────────────────┘
```

- 两个 backend 同时挂在同一项目下，分别有独立的 `ml_backends` 行
- 路由策略详见 §3.3
- 任一 backend 故障不影响另一个

### 3.2 协议补丁（向后兼容）

`context.type` 在 v0.9.x `text` 基础上加 `"exemplar"`：

```json
// v0.9.x 已有
{ "type": "point", ... }
{ "type": "bbox",  ... }
{ "type": "text",  "text": "ripe apples" }

// v0.10.x 新增
{ "type": "exemplar", "bbox": [x1,y1,x2,y2] }   // 取图中已有框作为视觉示例
```

`exemplar` 只能由 sam3-backend 处理；apps/api 根据项目挂载情况判断该 prompt 是否可用，前端在 sam3 未挂时**禁用** Shift+拖框入口。

### 3.3 路由策略

apps/api 收到交互式 `/predict` 时按下表决定路由目标：

| Prompt type | 项目挂的 backend | 路由到 |
|---|---|---|
| point / bbox | 都挂了 | grounded-sam2（默认更快）|
| point / bbox | 只挂 grounded-sam2 | grounded-sam2 |
| point / bbox | 只挂 sam3 | sam3 |
| text | 都挂了 | **按项目偏好**（ProjectSettings 选 default text backend）|
| text | 只挂一个 | 那个 |
| exemplar | 挂了 sam3 | sam3 |
| exemplar | 没挂 sam3 | **400 错误**（前端入口本应已禁用）|

「默认按项目偏好」是为了让管理员能选「精度优先 = sam3」或「速度优先 = grounded-sam2」。

### 3.4 image embedding 缓存

- sam3-backend 复用 v0.9.1 同款 LRU 设计
- cap 按硬件分档：3090 默认 cap=24，A100 默认 cap=64
- 缓存与 grounded-sam2 的缓存**互不共享**（embedding 来自不同模型，不能跨）

### 3.5 GPU 资源策略

| 场景 | 建议 |
|---|---|
| 团队仅 4060 笔记本 | 只挂 grounded-sam2，不部署 sam3 |
| 有 1 张 3090 工作站 | grounded-sam2 部 dev / 备机；sam3 部生产工作站 |
| 有 A100 生产服务器 | sam3 主推（高并发 + 高精度）；grounded-sam2 兜底 |
| 多 GPU 集群 | 两 backend 各占独立 GPU；nginx LB 多副本 sam3 |

> ❗️**待用户给数据**：上线时同时在线标注员数？打算给 sam3 几张卡？

### 3.6 不做的事
- ❌ 不下线 grounded-sam2-backend
- ❌ 不接 SAM 3 视频 predictor（`<VideoStage>` 还没建）
- ❌ 不做 SAM 3 微调端点 / Active Learning
- ❌ 不做"自动选最优 backend"智能路由（v0.10.x 是规则路由，自动学习留 v0.11+）

## 5. 开放问题

> v0.10.x 启动前必须解决：

1. **GPU 部署位置**：sam3-backend 部到哪台机？3090 工作站还是 A100？决定 cap 和并发。
2. **License 取向**：facebookresearch/sam3 weights 商用条款（与 SAM 2 不自动等价）—— 出方案前抠 LICENSE。
3. **HF_TOKEN 管理**：放 `.env` 还是 secrets manager？
4. **mask 输出保留**：未来要做 image-seg 工作台时，预测时是否同时写库 `predictions.mask_rle`？还是届时回填？建议**届时回填**（避免现在多写一列冗余存储）。
5. **AB 对比模式的"discarded" 数据保留时长**：永久保留（审计 / 后续训练）还是 30 天 GC？建议永久保留，加 `is_discarded` flag。

## 6. 风险

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| sam3 在 4060 显存 OOM | 高 | 笔记本不能用 | 文档明确 sam3 不部 4060；4060 用户走 grounded-sam2 |
| facebook/sam3 license 商用受限 | 低 | 项目重选 | 出方案前抠 LICENSE；保留切回 grounded-sam2 路径（v0.9.x 已上线，protocol 不变）|
| Python 3.12 / CUDA 12.6 部署门槛 | 中 | 用户难自部署 | 单独 `Dockerfile.sam3`；deploy.md 写明 |
| sam3 vs grounded-sam2 在 SOP 上输给后者 | 低 | 投资浪费 | M2 AB 对比工具就是为了避免盲目相信 sam3 |
| 路由表 bug 把请求发错 backend | 中 | 失败/慢 | M1 单测覆盖 7 种 case + 前端禁用 exemplar 入口当 sam3 未挂 |
