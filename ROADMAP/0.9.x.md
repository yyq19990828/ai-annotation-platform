## 0. TL;DR

- v0.9.x 的 backend 是 `apps/grounded-sam2-backend/`，内部链路是 **GroundingDINO（文本 → boxes）+ SAM 2.1（box / point → mask）**。
- 你的硬件全段（4060 笔记本 / 3090 工作站 / A100 生产）都能舒服跑全链；笔记本 4060 单图 200-500ms，3090 100-200ms。
- 协议层面**只在 v0.9.x 一次性把 `context.type` 扩到 `"text"`**（`"exemplar"` 留给 v0.10.x 与 SAM 3 一起上）。
- 切成 6 个里程碑：M0 backend 容器化 → M1 embedding 缓存 → M2 工作台 `S` 工具（含文本入口）→ M3 mask→polygon 调参 → M4 `/ai-pre` 文本批量预标 UI → M5 运维收口。
- v0.10.x 接 SAM 3 时，本期写的 `apps/grounded-sam2-backend/` **不下线**，作为另一个可选 backend 永久存在；项目管理员按需选哪个。

## 1. Grounded-SAM-2 能力盘点

### 1.1 链路构成

```
text prompt: "ripe apples"
    ↓
GroundingDINO（T 或 1.5）→ N 个 bounding box + score
    ↓
SAM 2.1（tiny ~ large）→ 每个 box 出 mask
    ↓
mask → polygon（cv2.findContours + shapely.simplify）
    ↓
返回 N 个 polygonlabels
```

`point` / `bbox` 类 prompt 跳过 GroundingDINO，直接喂给 SAM 2.1 image_predictor。

### 1.2 模型尺寸（vendor 时一次定下）

| 组件 | 默认变体 | checkpoint | 备选 |
|---|---|---|---|
| 检测 | GroundingDINO-T (Swin-T backbone, ~172M) | ~700MB | GroundingDINO-B / Grounding DINO 1.5 Pro（更准但需 API key） |
| 分割 | SAM 2.1 tiny (38.9M) | ~155MB | small / base+ / large（精度逐升、速度逐降） |
| 显存合计（FP16）| ~3-4GB | | tiny + DINO-T 在 4060 上 OK |

**默认配置**：`DINO-T + SAM 2.1 tiny`（速度优先），运维环境变量可切 large。

### 1.3 性能参考

| 硬件 | text 全链单图 | 缓存命中点击 |
|---|---|---|
| 4060 8GB | 200-500ms | < 50ms |
| 3090 24GB | 100-200ms | < 30ms |
| A100 40GB | 50-100ms + 高并发批量 | < 20ms |

### 1.4 License / 依赖
- GroundingDINO Apache 2.0；SAM 2 Apache 2.0；Grounded-SAM-2 自身 Apache 2.0 → 商用无忧。
- Python 3.10+ / PyTorch 2.3+ / CUDA 11.8+（与 v0.10.x SAM 3 的 3.12 / 2.7 / 12.6 不兼容 → 必须独立服务进程）。
- HuggingFace 拉权重不需 token。

### 1.5 限制
- **复合错误**：DINO 漏检 → SAM 没框可分；DINO 误检 → 多余 polygon。需 box_threshold / text_threshold 调参。
- **不支持 exemplar prompt**（图像示例）：留给 v0.10.x SAM 3。
- 文本提示**英文为主**：训练语料以英文 noun phrases 为主，中文需中→英翻译层（M4 处理）。

## 3. 架构设计要点

### 3.1 部署拓扑

```
┌─────────────────┐    HTTP /predict    ┌──────────────────────────────────┐
│  apps/api       │────────────────────▶│  grounded-sam2-backend (新)       │
│  (FastAPI 3.11) │                     │  FastAPI + PyTorch 2.3 + CUDA 11.8│
│   MLBackendClient│◀────────────────────│  GroundingDINO + SAM 2.1         │
└─────────────────┘                     │  + LRU embedding cache (SAM 侧)   │
                                        │  GPU profile (4060+ / 3090 / A100)│
                                        └──────────────────────────────────┘
```

- 新服务 `apps/grounded-sam2-backend/`，独立 docker-compose service，`profiles: ["gpu"]`
- vendor `IDEA-Research/Grounded-SAM-2` 到 `apps/grounded-sam2-backend/vendor/`，固定 commit hash（避免上游 README 改了 demo 脚本结构）
- 复用 echo-backend 样板的 FastAPI 骨架

### 3.2 vendor 策略

```
apps/grounded-sam2-backend/
├── pyproject.toml          # Python 3.10 + 依赖锁
├── Dockerfile              # 基于 pytorch/pytorch:2.3.1-cuda11.8-cudnn8-runtime
├── main.py                 # FastAPI 4 端点
├── predictor.py            # 封装 vendor 的推理调用
├── mask_to_polygon.py      # cv2.findContours + shapely
├── embedding_cache.py      # LRU
├── vendor/
│   └── grounded-sam-2/     # git submodule 或 vendored copy（固定 commit）
├── checkpoints/            # 启动时下载，挂 volume
└── README.md
```

> ❗️**待拍板**：submodule vs vendor copy？我倾向 vendor copy（写一次脚本同步），上游变动可控。

### 3.3 协议扩展

`context.type` 加 `"text"` 一种，不动既有字段：

```json
// 既有
{ "type": "point", "points": [[x,y]], "labels": [1] }
{ "type": "bbox",  "bbox": [x1,y1,x2,y2] }

// v0.9.x 新增
{ "type": "text", "text": "ripe apples" }

// v0.10.x SAM 3 时再加
// { "type": "exemplar", "bbox": [...] }
```

返回 `result[]` 数组（多实例时多 polygon）；前端 `S` 工具已能渲染 N 条。

> 协议契约文档同步 `docs-site/dev/ml-backend-protocol.md` §2.2 加 `text` 类型 + 注一句「`exemplar` 待 v0.10.x SAM 3」。

### 3.4 mask → polygon：放后端

cv2.findContours + shapely.simplify(tolerance, preserve_topology=True)；目标 50-200 顶点；M3 跑 IoU 评估调 tolerance。

抽到 `apps/_shared/mask_utils/`（Python 包），v0.10.x sam3-backend 直接 import 复用。

### 3.5 image embedding 缓存

```
Key:   sha1(file_path) + sam2_variant
Value: SAM 2 image embedding tensor
Size:  LRU cap = 16（4060 显存预算保守）/ 32（3090+）
```

- 缓存只缓存 SAM 2 侧（点/框 prompt 命中即用）
- GroundingDINO 端不缓存（每次文本 prompt 不同，缓存命中率低，不值得）
- `/cache/stats` 端点 + Prometheus metrics

### 3.6 不做的事
- ❌ 不在 v0.9.x 实现 exemplar prompt（推迟 v0.10.x SAM 3）
- ❌ 不做视频 / Active Learning / 训练闭环
- ❌ 不做运行时多变体共存 UI（运维改 env 重启即可）
- ❌ 不集成 Florence-2 / DINO-X 等 Grounded-SAM-2 的可选高级模块（v0.9.x 只走 DINO-T + SAM 2.1 tiny 这条主链）

## 5. 开放问题

1. **vendor 形态**：git submodule vs vendored copy？默认 vendor copy（写脚本一键同步上游）。
2. **DINO 变体**：默认 DINO-T 还是 DINO-B？建议 DINO-T（4060 友好），运维 env 切 DINO-B。
3. **SAM 2.1 变体**：默认 tiny 还是 small？建议 tiny。
4. **中→英翻译**：经 LLM 还是查词典？建议小型 LLM（你已有平台已用 LLM 集成；走相同 client）；用户可关闭。
5. **box_threshold / text_threshold 默认**：DINO 默认 0.35 / 0.25 在通用场景 OK，业务图（如车牌、商品）需 fine-tune；做成 ProjectSettings 项目级覆盖。

## 6. 风险

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| DINO 复合错误（漏检 / 误检）| 中 | 文本预标质量低 | M4 验收同时跑 5 类业务图 sample；调 threshold |
| upstream Grounded-SAM-2 重构 | 中 | vendor 同步成本 | 固定 commit + 同步脚本 |
| polygon 简化复杂形状失真 | 中 | 标注员重画 | M3 评估脚本 + tolerance 可调 |
| 中文 prompt 召回率低 | 高 | 项目方失望 | M2 中→英翻译开关 + 文档建议英文 |
| 笔记本 4060 显存吃紧（开多任务）| 低 | 不影响生产 | dev 文档建议笔记本只跑 tiny |
