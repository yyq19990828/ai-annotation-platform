# 视频追踪标注

视频任务在同一个工作台里打开，左侧队列、顶部提交、右侧属性与评论仍沿用图片工作台。画布区域会切换为视频播放器，时间轴悬浮在画布底部。

## 基本操作

1. 打开 `video-track` 项目的任务。
2. 用悬浮时间轴、`←` / `→` 或 `,` / `.` 定位到目标帧。
3. 暂停视频，在当前帧拖拽绘制 bbox；系统会创建一条对象轨迹。
4. 选中这条轨迹后，跳到其它帧继续画框或拖动框体，系统会把它保存为同一条轨迹的关键帧。若当前帧还没有框，画布会显示最近关键帧的虚线参考框，也可以点击「复制到当前帧」先生成当前帧关键帧再微调。
5. 如果目标在当前帧消失或被遮挡，在右侧轨迹面板点击「消失」或「遮挡」。
6. 标完后点击「提交质检」。

## 播放与逐帧

| 操作 | 说明 |
|---|---|
| `Space` | 播放 / 暂停 |
| `←` / `→` | 后退 / 前进 1 帧 |
| `,` / `.` | 后退 / 前进 1 帧 |
| `Shift + ←` / `Shift + →` | 后退 / 前进 10 帧 |
| 时间轴拖动 | 直接跳到指定帧 |

播放时画布只读，暂停后才能编辑，避免拖框和播放状态互相抢焦点。时间轴会在鼠标进入画布时显示，编辑拖拽时自动隐藏。

## 轨迹与关键帧

v0.9.17 起，视频任务默认保存为 `video_track`：

- 一条轨迹对应一个对象，轨迹列表会显示类别和 `track_id`。
- 同一对象在不同帧上的框会保存为 `keyframes[]`，不会展开成每一帧一条 annotation。
- 两个有效关键帧之间会显示线性插值框，插值框用虚线区分。
- 当前帧不在两个有效关键帧之间时，选中轨迹会显示最近关键帧的参考框；拖动参考框或点击「复制到当前帧」会在当前帧创建关键帧。
- 如果某段目标消失，给当前帧标记「消失」后，插值不会跨过该帧。
- 「遮挡」用于标记目标仍存在但可见性差，画布会用虚线状态展示。
- 单个关键帧的新增、移动、消失和遮挡标记支持 `Ctrl + Z` / `Ctrl + Shift + Z` 撤销重做。

轨迹列表支持：

| 操作 | 说明 |
|---|---|
| 显隐 | 临时隐藏 / 显示某条轨迹，只影响当前工作台视图 |
| 锁定 | 防止误拖动或误追加关键帧，只影响当前工作台视图 |
| 重命名 | 修改该轨迹的类别名 |
| 删除 | 选中轨迹后按 `Delete` 删除整条轨迹 |

## 质量提示

工作台会在画布左下角提示基础质量问题：

- 同一轨迹关键帧间隔过大。
- 当前帧存在极小框。
- 当前帧同类别框高度重叠。

这些提示不会阻止保存，但提交前应尽量处理。

## 当前版本边界

当前仍不支持视频 AI tracker、SAM video predictor 或 polygon track。视频导出支持 Video Tracks JSON，可选择只导出关键帧或展开逐帧结果。

v0.9.16 保存的旧 `video_bbox` 会继续在对应帧显示，但新建视频标注会保存为 `video_track`。
