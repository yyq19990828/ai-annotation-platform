# 批次与分配

## 什么是批次

批次（Batch）= 项目内一组任务的逻辑分组。常见用途：

- **按数据来源分批**：第一批由 A 公司提供，第二批由 B 公司
- **按时间分批**：每月新进的数据
- **按难度分批**：简单数据先给新人做

## 创建批次

1. 进入项目 → 「批次」标签页 → 「新建批次」
2. 填写：
   - 批次名
   - 来源数据筛选条件
   - 截止日期
3. 系统计算后展示「将包含 N 条任务」，确认后创建

## 分配人员

批次创建后默认状态 `draft`，需要分配标注员/审核员才能进入 `active` 状态：

- **手动分配**：勾选成员，按权重均分
- **按规则**：例如「优先分配给本组内」「按历史通过率」

## 状态机

```
draft → active → in_progress → review → completed → exported
                       ↓
                   cancelled
```

| 状态 | 含义 | 谁能看到 |
|---|---|---|
| draft | 已创建未分配 | 项目管理员 |
| active | 已分配未开始 | 项目管理员 + 标注员 |
| in_progress | 标注中 | 全部 |
| review | 审核中 | 项目管理员 + 审核员 |
| completed | 已完成 | 全部 |
| exported | 已导出 | 全部 |
| cancelled | 已取消 | 仅项目管理员 |

## 多选批量操作（v0.7.3）

批次列表支持多选 → 批量操作：暂停 / 恢复 / 重新分配 / 取消。
